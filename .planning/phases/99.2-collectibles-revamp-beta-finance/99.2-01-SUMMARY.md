# Phase 99.2 Plan 01: Foundation (helpers + CSS) Summary

**Single-source collectible urgency classifier + last-payment helper in finance.js, and the dedicated collectibles CSS block (scorecard chips, 6px progress bars, 6 urgency left-borders, ≤768px card rules) — foundation every Wave 2/3 plan consumes.**

## Performance

- **Duration:** ~12 min
- **Tasks:** 2
- **Files modified:** 2

## Accomplishments
- `getCollectibleUrgency(coll)` is the ONLY place the overdue threshold is defined (D-08b); returns `{ tier, daysOverdue, daysUntilDue, isOverdue }` over 6 tiers. `deriveCollectibleStatus()` now delegates its overdue branch to it — status and tier can never disagree.
- `getCollectibleLastPayment(coll)` returns the newest non-voided payment record (D-08a), drives the future Last Payment column.
- New `/* COLLECTIBLES REVAMP β (Phase 99.2) */` block in views.css: 4 chip accents, 6px progress bar (+is-overdue/is-paid/is-normal fills), the 6 urgency left-border tiers shared by `<tr>` and `.fc-card`, relative-due/last-payment text classes, the Show-N-completed toggle, and a ≤768px block adding `#collectibles-section` to the hide-tables list + the `fc-card-list` container.
- Zero visible behavior change yet (helpers unused, classes unreferenced) — intentional foundation staging for Waves 2–3.

## Task Commits

1. **Task 1: shared urgency classifier + last-payment helper** - `63a27db` (feat)
2. **Task 2: dedicated collectibles CSS block** - `8a0ebaa` (feat)

## Files Created/Modified
- `app/views/finance.js` - added `getCollectibleUrgency()` + `getCollectibleLastPayment()` at module scope; refactored `deriveCollectibleStatus()` overdue branch to delegate
- `styles/views.css` - added the Phase 99.2 collectibles CSS block (desktop rules global, mobile rules in a ≤768px media query)

## Decisions Made
- Day-granularity local-midnight overdue computation (more correct than the old time-of-day `new Date(due) < new Date()`); return strings unchanged so no downstream consumer breaks.
- Tier precedence ordering fixed so a fully-paid collectible is always `paid`, never `critical`/`overdue`.
