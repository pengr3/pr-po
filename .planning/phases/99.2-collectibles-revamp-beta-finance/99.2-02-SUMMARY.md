# Phase 99.2 Plan 02: Scorecard + 5-col table + mobile cards Summary

**4-chip reactive scorecard + 10→5 column collectibles redesign (urgency left-borders, 6px progress bars, relative-due, last-payment, fully-paid hide toggle, recomputed pagination) + ≤768px card list — all consuming the Plan 01 helpers/CSS.**

## Performance

- **Duration:** ~20 min
- **Tasks:** 3 (committed together — coupled)
- **Files modified:** 1 (app/views/finance.js)

## Accomplishments
- **Scorecard (SC1/D-02):** `renderCollectiblesScorecard(displayed)` aggregates Total Invoiced / Cash Collected / Outstanding / Overdue over the exact visible (filtered + post-hide) array the table paginates; the Overdue chip is omitted at ₱0 and its outstanding total uses the shared `getCollectibleUrgency` (D-08b).
- **5-col table (SC3/D-05):** thead collapsed 10→5; each `<tr>` gets the per-tier 4px urgency left-border via `coll-urgency-${urg.tier}`; columns are Project·Tranche (name + tranche% + dept chip + COLL id), Collection Progress (6px bar + "₱X of ₱Y · Z%"), Due·Urgency (relative string with ‼/⚠), Last Payment (non-voided newest), Actions (fully-paid → View History only).
- **Hide toggle + pagination (D-05b/D-08c):** fully-paid rows hidden by default behind a "Show N completed" toggle; `renderCollectiblesTable`, `changeCollectiblesPage`, and the scorecard all recompute over the post-hide set. `window.toggleCollShowCompleted` registered + torn down.
- **Mobile (SC4/D-06):** `buildCollectibleCard()` mirrors the 73.1 `.fc-card` anatomy with the urgency border + 5-col content; `#collectiblesCardList` host populated alongside `tbody`.
- **CSV non-regression (D-07):** `getDisplayedCollectibles()` byte-unchanged; the hide lives in the render path so the 13-col CSV still exports fully-paid rows.
