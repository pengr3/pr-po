### 2. Unified collapsible approved-not-filed banner (component + listener, event-driven + transform)

**Analog:** `renderPendingBillingBanner()` L1607 (the existing single-section banner) + its listener `brUnsub` L1592 + `collectiblesData` listener L1539 + `openCreateCollectibleModal(preselectKey)` L1755.

**Current state — single section, pending only.** The banner host already exists at L3702:


**Existing pending listener — ONLY subscribes to `status=='pending'`** (L1592-1601). This is the FIRST thing to change: sub-section #2 needs `status=='approved'` rows too.
```javascript
const brUnsub = onSnapshot(
    query(collection(db, 'billing_requests'), where('status', '==', 'pending')),   // L1593 — too narrow
    (snapshot) => {
        pendingBillingRequests = [];

        renderPendingBillingBanner();
```

**Change:** widen to load both states. Either drop the `where` and split client-side into `pendingBillingRequests` + `approvedBillingRequests`, or run two queries. Prefer one un-filtered (or `in ['pending','approved']`) query + client split to keep a single listener in `listeners[]`.

**Reconciliation (sub-section #2) — the must-have correctness logic (D-04).** Approved-with-no-COLL = approved billing request whose `project_code + tranche_index` has no matching doc in `collectiblesData`. The reconcile is a client-side `.find()` (spike 027b):
```javascript
// approved billing request `req` is an ORPHAN (show in sub-section #2) when:
const hasColl = collectiblesData.some(c => {
    const code = c.department === 'services' ? c.service_code : c.project_code;
    const reqCode = req.department === 'services' ? req.service_code : req.project_code;
    return code === reqCode && c.tranche_index === req.tranche_index;
});
// show when !hasColl
```
Note the dept-aware code selection mirrors `getDisplayedCollectibles()` L1339 and `approveBillingRequest()` L1655-1658.

**"File COLL-xxx" button → reuse the existing preselect bridge** (do NOT invent a new modal path). `approveBillingRequest()` L1655-1659 already builds the exact preselectKey shape:
```javascript
const isService = req.department === 'services';
const preselectKey = isService
    ? 'services:' + (req.service_code || '') + ':' + req.tranche_index
    : 'projects:' + (req.project_code || '') + ':' + req.tranche_index;

```
`openCreateCollectibleModal` parses that 3-segment key at L1766-1771. Reuse verbatim for the sub-section #2 button.

**Banner markup to extend** — current single block at L1635-1643 (chevron + label + count pill + collapsible body). Restructure into one outer `<div>` with two independently-collapsible sub-sections: "Awaiting Your Review" (amber, pending) + "Approved — File as Collectible" (blue, orphans). Existing approve/reject row markup (L1621-1633) is the template for sub-section #1 rows.

**Listener teardown:** `brUnsub` is pushed to `listeners[]` (L1601) and torn down by `destroy()`. The widened query stays in the same array — no teardown change needed. All `escapeHTML` on user strings is already the convention (L1619-1627).

**LANDMINE (D-04):** Reconciliation correctness is a MUST-HAVE — an orphan must NEVER be silently absent. This banner is the *only* recovery path for the Phase 99 orphan state (approve flips status before COLL exists). Do NOT make approve atomic (explicitly deferred).

**4px urgency left-border (D-05a):** belongs in the new CSS block (per-tier classes), applied via a class on each `<tr>`. NOT inline. Spike 027c cites Spike 005's 4px left-border accent as the established precedent.

## Shared Patterns

### Overdue / urgency helper (single source of truth — D-08b, the #1 invariant)
**Source:** `deriveCollectibleStatus()` L114 currently inlines `coll.due_date && new Date(coll.due_date) < new Date()`.
**Apply to:** scorecard Overdue chip (#1), row urgency tiers (#3), mobile card border (#4), relative-due string (#3).
**Action:** Extract ONE helper (e.g. `getCollectibleUrgency(coll)` returning `{ tier, daysOverdue, daysUntilDue }`) used by all four. The 6 tiers (D-05a): critical (30+ overdue) / overdue (1-29) / near-due (≤7 to due) / partial (paid>0, not overdue) / healthy / fully-paid. Do NOT duplicate the threshold logic. `deriveCollectibleStatus()` should consume the same helper for its `isOverdue` branch so status and tier never disagree.
```javascript
// existing single overdue check to centralize — L114
const isOverdue = coll.due_date && new Date(coll.due_date) < new Date();
```

### Non-voided payment rule (D-08a / D-09)
**Source:** repeated at L111-113 (deriveStatus), L1409-1411 (table row), L2632-2634 (CSV).
**Apply to:** Collected chip, progress bar, Last Payment column, balance — everywhere money-received is summed.
```javascript
(coll.payment_records || []).filter(r => r.status !== 'voided').reduce((s, r) => s + (parseFloat(r.amount) || 0), 0)
```
Last Payment (#3) must ALSO filter `status !== 'voided'` before picking the newest by `date`.

### Window-function registration + teardown (CLAUDE.md SPA contract)
**Source:** `attachWindowFunctions()` L373-397 (register) + `destroy()` L4653-4675 (delete).

**Apply to:** merged "Project · Tranche" col (#3) and the mobile card (#4) — call `getDeptBadgeHTML({ department: coll.department })`.
