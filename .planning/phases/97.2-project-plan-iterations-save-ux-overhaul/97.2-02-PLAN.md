    Change this to an async function that also cleans up the Firestore auto-snapshot:
      async function dismissUndoToast() {
          clearTimeout(_undoToastTimer);
          document.getElementById('iterUndoToast')?.remove();
          if (_autoSnapId) {
              const snapId = _autoSnapId;
              _autoSnapId = null;
              try {

    - The try/catch is silent — cleanup is best-effort; do not surface an error to the user.
    - deleteDoc(doc(db, 'project_iterations', snapId)) is the same pattern already used in undoIterRestore() at line 3565.
