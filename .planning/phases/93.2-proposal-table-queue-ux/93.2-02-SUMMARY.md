## Decisions Made

- `'active_only'` replaces `null` as the default filter value — ensures users see draft/pending_internal/pending_client/for_revision proposals on first load without needing to interact
- Active-stage tile clicks set `'active_only'` (not the tile's own key) per plan D-02 — the four active stages are treated as a group, so clicking any one shows all four
- Closed tile toggle goes to `'active_only'` (not `null`) on second click — `null` is now an unreachable state in the normal UX flow
- `safePage = Math.min(Math.max(1, page), totalPages)` applied before slice — out-of-range page numbers from onclick attributes produce valid results rather than errors (T-93.2-05)
