**1. Rewrite `showProcurementTimeline()` to group PRs with their child POs:**

Instead of building a flat `timelineItems` array and passing it to `createTimeline()`, build custom timeline HTML directly in the function. The `createTimeline()` component only supports flat lists — nesting requires custom HTML.

**Grouping logic:**
```javascript
// Group POs by pr_id for parent-child pairing
const posByPR = {};
pos.forEach(po => {
    const prId = po.pr_id || '_unlinked';
    if (!posByPR[prId]) posByPR[prId] = [];
    posByPR[prId].push(po);
});
```

**Timeline HTML structure:**
Build a `<div class="timeline">` with the following items in order:

1. **MRF Created entry** (always first):
   - Title: `MRF Created: ${mrf.mrf_id}`
   - Date: `formatTimestamp(mrf.created_at) || 'N/A'`
   - Description: `Requestor: ${mrf.requestor_name} | ${deptLabel}: ${getMRFLabel(mrf)}`
   - Status class: `completed`

2. **For each PR, render the PR as a parent item with its child POs indented below:**
   ```html
   <div class="timeline-item ${prStatusClass}">
       <div class="timeline-item-title">Purchase Request: ${pr.pr_id}</div>
       <div class="timeline-item-date">${formatTimestamp(pr.date_generated) || 'N/A'}</div>
       <div class="timeline-item-description">Supplier: ${pr.supplier_name} | Amount: ₱${formatCurrency(pr.total_amount)}</div>
       <!-- Child POs -->
       <div class="timeline-children">
           ${(posByPR[pr.pr_id] || []).map(po => `
               <div class="timeline-child-item ${poStatusClass}">
                   <div class="timeline-item-title">Purchase Order: ${po.po_id}</div>
                   <div class="timeline-item-date">${formatTimestamp(po.date_issued) || 'N/A'}</div>
                   <div class="timeline-item-description">Supplier: ${po.supplier_name}</div>
                   <div class="timeline-procurement-status">
                       <span class="status-badge ${getStatusClass(po.procurement_status || 'Pending Procurement')}">
                           ${po.procurement_status || 'Pending Procurement'}
                       </span>
                   </div>
               </div>
           `).join('')}
       </div>
   </div>
   ```

   PR status class logic:
   - `pr.finance_status === 'Approved'` → `'completed'`
   - `pr.finance_status === 'Rejected'` → `'rejected'`
   - Otherwise → `'pending'`

   PO status class logic:
   - `po.procurement_status === 'Delivered' || po.procurement_status === 'Processed'` → `'completed'`
   - `po.procurement_status === 'Procuring' || po.procurement_status === 'Processing'` → `'active'`
   - Otherwise → `'pending'`

3. **For each TR, render as standalone item (no children):**
   - Title: `Transport Request: ${tr.tr_id}`
   - Date: `formatTimestamp(tr.date_submitted) || 'N/A'`
   - Description: `Amount: ₱${formatCurrency(tr.total_amount)}`
   - Status: same PR logic based on `tr.finance_status`

4. **Handle orphan POs** (POs not linked to any PR — `posByPR['_unlinked']`):
   Render them as standalone timeline items after all PRs. This handles edge cases where POs exist without a valid pr_id.

5. **Procurement Status per PO:**
   Each PO child item includes a `<div class="timeline-procurement-status">` showing the current `po.procurement_status` as a styled badge using `getStatusClass()`. This is the current status only, not a history.

6. **Replace the `createTimeline(timelineItems)` call** with direct HTML injection:
