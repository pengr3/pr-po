**Before:** Gate `proposal.status !== 'pending_client'` → error. Single _applyProposalStateTransition call with `newStatus: null, newProjectStatus: null`.

**After:** Gate `!== 'pending_client' && !== 'for_revision'` → error. `isResend = (status === 'for_revision')`. Two branches: isResend passes `newStatus: 'pending_client', newProjectStatus: 'Proposal Under Client Review'`; else passes `newStatus: null, newProjectStatus: null`.

### New functions: openRequestRevisionModal + confirmRequestRevision

Inserted between submitLoss and openClientApprovedModal. Mirrors Mark-as-Loss sub-modal shape exactly:
- Modal id: `proposalRequestRevisionModal`, z-index: 1001
- Optional comment textarea (no minLength validation — D-11 specifies optional)
- Confirm btn-primary (not btn-danger, unlike Loss) calling `window.confirmRequestRevision`
- confirmRequestRevision calls `_applyProposalStateTransition({ newStatus: 'for_revision', newProjectStatus: 'For Revision', auditAction: 'REVISION_REQUESTED' })`
