## Task 1 — Helper + Cache (D-06/D-07)

Four sub-edits made to `app/proposal-modal.js`:

**1A — Module state (line 81):**
Added `let _parentDocCache = null;` after `_modalProjectsLoaded`. Shape: `{ collection: 'projects'|'services', doc: {id, personnel_user_ids, ...} } | null`.

**1B — Helper function (after _fetchProposalDoc, before _loadModalDropdownData):**
Added `_isCallerAttachedToProposalParent(proposal)` — synchronous (no async), reads `_parentDocCache` and `window.getCurrentUser()`. Returns `{ canApprove, canDrive }`.

Role + parent_collection truth table:

| Role | parent_collection | canApprove | canDrive |
|------|-------------------|-----------|---------|
| super_admin | any | true | true |
| operations_admin | projects | true | true |
| operations_admin | services | false | false |
| services_admin | services | true | true |
| services_admin | projects | false | false |
| operations_user (assigned) | projects | false | true |
| operations_user (not assigned) | projects | false | false |
| services_user (assigned) | services | false | true |
| services_user (not assigned) | services | false | false |
| finance_user / procurement_user | any | false | false |
| _parentDocCache = null | any | false | false (defensive default-deny) |

**1C — openProposalModal preload:**
After `currentProposal = proposal;`, one `getDoc` fetches the parent project/service doc into `_parentDocCache`. Wrapped in try/catch; sets null on error (defensive default-deny).

**1D — closeProposalModal clear:**
`_parentDocCache = null;` added after `currentProposal = null;`.

## Task 2 — renderProposalActionButtons Refactor (D-05/D-06/D-07/D-08/D-09)

**Before (single flag):**
```javascript
const canApprove = ['super_admin', 'operations_admin'].includes(cu?.role);
// draft/for_revision: Submit gated to canApprove; Edit unconditional
// pending_internal: Approve/Reject gated to canApprove
// pending_client: all three buttons gated to canApprove
```

**After (dual flags):**
```javascript
const { canApprove, canDrive } = _isCallerAttachedToProposalParent(proposal);
// draft/for_revision: Submit+Edit gated to canDrive
// pending_internal: Approve/Reject gated to canApprove
// pending_client: Mark Sent/Client Approved/Mark as Loss gated to canDrive
```

Key behavior changes:
- **draft/for_revision Submit:** was `canApprove`-only → now `canDrive` (D-05: assigned ops/services user can submit their own proposal)
- **draft/for_revision Edit:** was unconditional → now `canDrive` (D-08: non-attached users no longer see Edit)
- **pending_client all buttons:** was `canApprove` → now `canDrive` (core G2 fix: assigned ops/services user can drive the client-facing lifecycle)
- **pending_internal Approve/Reject:** unchanged — `canApprove` only (D-09)
