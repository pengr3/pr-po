## Q3 — Multi-category model + storage + seed list (re-asked)

## Q4 — Column position

**Options presented:**
- After Supplier Name (recommended)
- Before Actions (rightmost data column)
- After Phone

**User selected:** After Supplier Name. Final column order: **Supplier Name | Categories | Contact Person | Email | Phone | Actions**.

---

## Q5 — Legacy / blank rows UX

**Options presented:**
- Em-dash + 'Show uncategorized only' toolbar control (recommended)
- Em-dash, no special filter
- Force category on Edit (block save)

**User selected:** Em-dash + 'Show uncategorized only'. Edit form does NOT enforce ≥1 category — users can save legacy rows without categorizing them, supporting a gradual manual cleanup pass.

---

## Q6 — Search / filter on Categories

**Options presented:**
- Extend search to match categories (recommended minimal)
- Search + category filter dropdown
- No change to search/filter

**User selected:** Extend search only. `applySupplierSearch()` matches the term against `supplier_name`, `contact_person`, OR any string in `supplier.categories`. No dedicated category filter dropdown added to the toolbar this phase.

## Claude's discretion items (decided without asking)
- **Firestore Security Rules** (D-08): verified that `firestore.rules:354-362` permits supplier writes with no field-level restriction, so the new `categories` field needs no rules update. Called out explicitly so the planner does not invent unnecessary deploy work.
- **Field naming** (`category` → `categories`): the ROADMAP success criterion #1 says singular `category`. The user's "suppliers fall onto different, numerous categories" reframing supersedes this — the schema is plural `categories: string[]`. Flagged in CONTEXT.md Specifics so the planner doesn't try to "fix" the criterion.
- **Permission gate** (`procurement` vs `procurement_suppliers`): left as a planner decision because the existing supplier edit gates still use `canEditTab('procurement')` despite Phase 91 introducing the finer-grained `procurement_suppliers` key. Worth a conscious choice during planning rather than a silent inheritance.
