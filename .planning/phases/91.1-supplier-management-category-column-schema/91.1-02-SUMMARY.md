## Decisions Made

- **`_categoryPillState.get(editStateId) || []` instead of `getCategoryPillState(editStateId)` inside the renderer.** The Plan 01 helper returns a defensive spread-copy; inside a hot render loop the Map handle is identical-shape data and avoids the redundant allocation. The renderer is the only call site that legitimately reads the module-private Map directly — every other consumer goes through the helper.
- **`Array.isArray(supplier.categories) ? supplier.categories : []` guard on the edit-row seed.** Matches D-01 read-side semantics (undefined / null / [] all = "uncategorized"). Without this, an `Array(undefined)` or string-typed `categories` from a malformed Firestore doc would propagate into the pill control and either crash the render or paint phantom pills.
- **No >=1 guard on `saveEdit`.** Load-bearing per D-02: legacy suppliers must be saveable while operations team manually encodes categories post-deploy. The contrast against `addSupplier`'s explicit `categories.length === 0` branch is grep-able and reviewer-visible — both validators live at their respective call sites, no shared helper.
- **`clearAddForm` uses clear-then-set sequence, not just clear.** `clearCategoryPillState` deletes the Map entry *and* hides any open dropdown; if a user clicked Cancel mid-typing into "Add new...", we want both the data slot and the dropdown chrome to reset. The follow-up `setCategoryPillState([])` recreates an empty slot so the next form-open finds a usable slot (no `undefined` re-init dance inside `toggleAddForm`). `rerenderCategoryPillControl` repaints the chrome to the empty state.
- **`toggleAddForm` initializes pill state on every open, not just first open.** Idempotent guarantee — even if a previous close path skipped `clearAddForm` (e.g., a click that bypassed the toggle button), the next open starts clean.
- **Inline-edit row gets the Categories cell at column position 2, not last.** Plan 03 will add the *display-row* Categories column at position 2 — keeping the edit row's cell in the matching position avoids a column re-shuffle when Plan 03 ships. This is the "coordinate now, avoid re-work later" call from the plan.
