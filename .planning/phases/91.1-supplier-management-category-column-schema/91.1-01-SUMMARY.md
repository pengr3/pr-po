## Decisions Made

- **Pill state in a `Map<containerId, string[]>`** rather than a single global selectedCategories array ‚Äî required because Plans 02/03 will render the Add form's control and one inline-edit row's control simultaneously on the same page
- **Reuse existing `.personnel-pill` / `.pill-input-container` / `.pill-dropdown` CSS classes verbatim** rather than introducing a parallel `.category-pill` class ‚Äî preserves the single-source-of-truth styling rule called out in the plan's interface contract and D-CONTEXT gotchas
- **`localeCompare` with `{ sensitivity: 'base' }`** for the known-categories sort ‚Äî defensive; current seed is title-case so behavior is unchanged today, but keeps future user-typed `welding` vs `Welding` grouped if they ever diverge
- **`Array.isArray` + `typeof === 'string'` + `trim().length > 0` filter** inside `getKnownCategories` ‚Äî closes the T-91.1-03 hole where a malformed Firestore doc carrying `null` / `''` inside `categories` could otherwise leak into the dropdown
- **Single-shot document click-away listener registered via `setTimeout(...,0)`** for dropdown close ‚Äî `setTimeout` deferral avoids the listener catching the originating click that opened the dropdown
- **`escapeHTML` alone is sufficient** for category strings interpolated into single-quoted onclick attributes (verified by reading `app/utils.js`: single quotes become `&#39;`, so the surrounding `'...'` delimiters remain unambiguous ‚Äî no extra `.replace(/'/g, ...)` chain needed)
