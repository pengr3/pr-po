**buildLifecycleTrack(project)**
- Reads `project.project_status` (with `|| 'For Inspection'` fallback for legacy docs)
- Implements the same curIdx logic as spike.html buildTrack():
  - status === 'Loss': curIdx = 4 (Client Approved node shown as s-loss)
  - status === 'For Revision': curIdx = index of 'Proposal Under Client Review' in LC_STAGES
  - else: LC_STAGES.findIndex(sg => sg.status === status)
- Iterates LC_STAGES to build node HTML using class names exactly: `stage-node ${nodeClass}`, `stage-circle ${circClass}`, `stage-check`, `stage-label`, `stage-chip ${chipCls}`
- nodeClass: isPast → 's-done-node', isRevNode → 's-revision-node', isCurrent → 's-current-node', else ''
- circClass: isPast → 's-done', isRevNode → 's-revision', isCurrent → 's-current', else 's-future'
- chipCls/chipTxt: isPast → 'chip-done'/'DONE', isRevNode → 'chip-revision'/'REVISION', isCurrent → 'chip-here'/'← HERE', Completed → 'chip-end'/'END', gated → 'chip-gap'/'GATE', else 'chip-end'/'—'
- After STAGES loop: if status === 'Loss', appends the red badge div (same HTML as spike.html lines 596-598)
- Connector between nodes: `<div class="connector ${isPast ? 'done' : ''}"></div>`
- All label/emoji text is static (no user data) — no escapeHTML needed
- Returns HTML string

- Reads `project.project_status` with `|| 'For Inspection'` fallback
- Determines isActive = ['For Inspection','Client Approved','For Mobilization','On-going','For Revision'].includes(status)

    <div class="lc-body" id="lcBody"><!-- Plan 03 fills this --></div>
