<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser client → service_tasks writes | service-plan.js issues create/update/delete; authorization enforced by the Plan-01 rules block (not client code) |
| router param → view | the `#/services/{code}/plan` hash segment is attacker-controllable; the view must tolerate an unknown/empty service_code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-105-05 | Tampering | service-plan.js task writes | mitigate | All writes go through the Plan-01 two-tier rules; the view relies on rules (not UI gating) for authorization — UI gating is advisory only |
| T-105-06 | Information Disclosure | clientless / unknown service_code | mitigate | clientless block returns early with "No service code"; the onSnapshot is scoped `where('service_id','==',currentService.id)` so an attacker cannot enumerate other services' tasks via the param |
| T-105-07 | Denial of Service | renderGantt during drag | accept | mirrors the shipped project plan's `_ganttBarDragging` re-render guard; no new surface introduced (parity) |
| T-105-08 | Repudiation | task created_by | mitigate | service_tasks docs carry `created_by` (mirrored from project schema D-20) for audit attribution |
| T-105-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs — Frappe Gantt + html2pdf already CDN-loaded in index.html (reused). N/A. |
</threat_model>
