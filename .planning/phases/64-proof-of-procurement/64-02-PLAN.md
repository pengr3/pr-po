**Important note about Finance calling showProofModal:** The `showProofModal` and `saveProofUrl` functions live in procurement.js and are registered on `window` during procurement.js init. However, the Finance view is a separate module — if the user navigates directly to Finance without visiting Procurement first, `window.showProofModal` would be undefined.

To handle this, add a fallback inline in finance.js. At the top of the Finance PO table rendering function, add a local `showFinanceProofModal` that either delegates to `window.showProofModal` if available, or creates a simple prompt-based fallback:

```javascript
// Proof URL helper — delegates to procurement's modal if available, otherwise uses prompt
window.financeShowProofModal = window.financeShowProofModal || function(poId, currentUrl) {
    if (window.showProofModal) {
        window.showProofModal(poId, currentUrl, false, null);
        return;
    }
    // Fallback: simple prompt
    const url = prompt(currentUrl ? 'Update Proof URL:' : 'Enter Proof URL (https://):', currentUrl || '');
    if (url === null) return;
    if (!url.startsWith('https://')) {
        showToast('URL must start with https://', 'error');
        return;
    }
    const isFirstAttach = !currentUrl;
    const poRef = doc(db, 'pos', poId);
    const updateData = { proof_url: url, updated_at: new Date().toISOString() };
    if (isFirstAttach) {
        updateData.proof_attached_at = serverTimestamp();
    } else {
        updateData.proof_updated_at = serverTimestamp();
    }
    updateDoc(poRef, updateData).then(() => {
        showToast(isFirstAttach ? 'Proof URL attached' : 'Proof URL updated', 'success');
    }).catch(err => {
        console.error('[Finance] Error saving proof URL:', err);
        showToast('Failed to save proof URL. Please try again.', 'error');
    });
};
```

Register `window.financeShowProofModal` in Finance's `init()` and clean up in `destroy()`.
