**Important note on showProofModal availability:** mrf-records.js is used in the My Requests tab under the MRF Form view. `window.showProofModal` is registered by procurement.js when that view is active. Since My Requests is under `#/mrf-form`, procurement.js may NOT be loaded. However, `window.showProofModal` is a global that persists once set. If it is not available, the click will silently fail (no crash — calling undefined as function would crash, but the onclick is a string that evaluates `window.showProofModal(...)` which returns undefined if the function doesn't exist). To be safe, wrap modal calls with a guard:

Replace bare `window.showProofModal(` calls in the proof indicators with:
```javascript
onclick="if(window.showProofModal){window.showProofModal('${po.docId}','',false,null,'')}else{window.open('','_blank')}"
```

Actually, the simpler and correct approach: `window.showProofModal` is defined in procurement.js which registers it on `window` during its `init()`. When user is on My Requests (mrf-form view), procurement.js is NOT loaded. So proof indicators in My Requests should handle this gracefully. Use `typeof window.showProofModal === 'function'` check. For the empty/remarks states where the only action is opening the modal, show a tooltip "Open Procurement > PO Tracking to attach proof" if showProofModal is unavailable. For filled state, `window.open` works regardless since it just opens the URL.
