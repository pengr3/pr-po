<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client journal write ‚Üí Firestore service subcollections | Untrusted user text (note/issue/progress) crosses into Firestore documents and is later rendered back to other users |
| journal success path ‚Üí parent service doc | The `last_activity_at` bump crosses from the (broadly-permitted) subcollection write into the (role-restricted) parent-doc write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-104-06 | Tampering (XSS via stored journal text) | _renderFeedEntry / _renderIssueRow / _renderProgressCard | mitigate | All user text + names rendered through `escapeHTML` (already imported service-detail.js:8); copied builders already wrap user fields ‚Äî KEEP every escapeHTML call intact during the copy. |
| T-104-07 | Denial of Service / Repudiation | D-14 last_activity_at bump on non-team poster | mitigate | Bump is fire-and-forget, un-awaited, un-batched, `.catch`-swallowed, and success-path-only ‚Äî a denied parent write (enforced by Plan 01 rules T-104-03) never rolls back or surfaces on the journal entry. |
| T-104-08 | Information Disclosure | journal listeners read all subcollection docs | accept | `allow read: if isActiveUser()` per Plan 01 (T-104-05) ‚Äî matches Phase 101 projects precedent; operational content, not sensitive PII. Accepted. |

No HIGH-severity unmitigated threat. Stored-XSS (T-104-06) mitigated by preserving escapeHTML on the copy; D-14 cross-write (T-104-07) mitigated by the fire-and-forget contract.
