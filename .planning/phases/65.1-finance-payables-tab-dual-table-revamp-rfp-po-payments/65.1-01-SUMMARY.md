# Phase 65.1 Plan 01: Dual-Table Payables HTML + RFP Processing Table Summary

## One-liner

Replaced single Payables table with dual-table layout — `renderRFPTable()` renders a flat per-RFP list (no chevron/history rows) and `renderPOSummaryTable()` stub scaffolds the PO-grouped table; split filter state into 4 per-table variables.

## What Was Built

### Task 1: Split filter state and replace Payables HTML with dual-table layout

- Replaced `let payablesStatusFilter = ''` and `let payablesDeptFilter = ''` with 4 per-table vars
- Replaced the single `<section id="payables-section">` block with a dual-table layout:
  - **Table 1** (RFP Processing): `id="rfpTableBody"`, 11 columns, own filter bar (`rfpStatusFilter`, `rfpDeptFilter`) calling `window.filterRFPTable()`
  - **Table 2** (PO Payment Summary): `id="poSummaryTableBody"`, 9 columns, own filter bar (`poSummaryStatusFilter`, `poSummaryDeptFilter`) calling `window.filterPOSummaryTable()`
  - Visual divider (border-top) between tables
- Updated window registrations to `filterRFPTable`, `filterPOSummaryTable`, `togglePOExpand`
- Added destroy() cleanup: deletes all 6 payables window functions, resets `rfpsData` and all 4 filter state vars

### Task 2: New functions + updated initPayablesTab

- Added `const statusBadgeColors` at module level (line 44) — shared by both render functions
- Added `filterRFPTable()` — reads `rfpStatusFilter`/`rfpDeptFilter` from DOM, calls `renderRFPTable()`
- Added `filterPOSummaryTable()` — reads `poSummaryStatusFilter`/`poSummaryDeptFilter`, calls `renderPOSummaryTable()`
- Added `togglePOExpand(poId)` — toggles `po-expand-${poId}` row and `po-chevron-${poId}` chevron
- Replaced `renderPayablesTable()` with `renderRFPTable()`:
  - Flat list, no chevron column, no payment history expand rows (D-02)
  - Overdue row tint `background-color:#fef2f2` (D-04)
  - Record Payment button on non-Fully-Paid rows (D-03)
  - Sort by PO Ref asc then tranche_percentage asc (D-20)
  - Empty state: `colspan="11"` with context-aware message
- Added `renderPOSummaryTable()` stub — returns placeholder "PO summary loading..." row
- Updated `initPayablesTab()` onSnapshot: `renderRFPTable(); renderPOSummaryTable();`
- Eliminated all old function names: `renderPayablesTable`, `filterPayables`, `togglePaymentHistory`
