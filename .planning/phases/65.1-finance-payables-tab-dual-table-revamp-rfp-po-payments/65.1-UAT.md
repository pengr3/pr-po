### 1. Dual-table layout renders
expected: Finance > Payables tab shows two distinct sections: "RFP Processing" heading (bold) at top, then a horizontal divider, then "PO Payment Summary" heading below. Each section has its own Status and Department filter dropdowns directly above its table.
result: pass

### 2. Table 1 — flat RFP list, no chevrons
expected: Table 1 (RFP Processing) shows one row per RFP/tranche. There are NO expand chevrons on the left — it's a flat list. Rows are sorted by PO Ref first, then by tranche percentage (so all tranches under the same PO appear together in order). Columns: RFP ID | Supplier | PO Ref | Project/Service | Tranche | Amount | Paid | Balance | Due Date | Status | Actions.
result: issue
reported: "pass, however lets fix the groupings and sorting. Paid and unpaid items are mixed up — within a PO, fully-paid tranches appear before unpaid/overdue ones within the same PO group"
severity: major

### 3. Table 1 — overdue tint and Record Payment button
expected: An RFP with a past due date that is not Fully Paid has a red-tinted row background. Fully-Paid rows have no tint. Non-Fully-Paid rows show a "Record Payment" button in the Actions column. Fully-Paid rows show no button.
result: pass

### 4. Table 1 filters work independently
expected: Selecting a Status or Department in Table 1's filter bar filters only Table 1 rows. Table 2 (PO Payment Summary) is completely unaffected. Clearing Table 1 filters restores all Table 1 rows.
result: pass

### 5. Table 2 — one row per PO with aggregated totals
expected: Table 2 (PO Payment Summary) shows one row per unique PO ID, sorted alphabetically. Each row shows the correct aggregate totals: Total Amount = sum of all RFP amounts for that PO, Total Paid = sum of all non-voided payments, Remaining = Total Amount minus Total Paid. The "Current Active Tranche" column shows the label of the first unpaid tranche (or "Fully Paid" if all tranches are paid).
result: pass

### 6. Table 2 — expand/collapse PO rows
expected: Each PO row in Table 2 has a ▶ chevron on the left. Clicking it reveals a sub-table showing all RFPs/tranches under that PO (columns: RFP ID, Tranche, Amount, Paid, Balance, Due Date, Status, Actions). Clicking again collapses it. Chevron rotates from ▶ to ▼ when expanded. Multiple POs can be expanded simultaneously.
result: pass

### 7. Table 2 — overall status badge
expected: The "Overall Status" badge in Table 2 is correctly derived: "Fully Paid" (green) if all tranches are paid, "Overdue" (red) if any tranche is overdue and unpaid, "Partially Paid" (blue) if some payments exist but not fully paid, "Pending" (yellow) if no payments at all. A fully-paid PO shows green even if some tranches have passed their due dates.
result: issue
reported: "Overall status shall be able to track that a PO has remaining payments to be made. Not based on active RFPs because a PO with progress billing, that have not yet billed are marked as fully paid even tho we know that there are future payments to be made."
severity: major

### 8. Table 2 filters work independently
expected: Selecting a Status or Department in Table 2's filter bar filters only Table 2 PO rows. Table 1 (RFP Processing) is completely unaffected.
result: pass

### 9. Record Payment from Table 2 sub-rows
expected: Expanding a PO row in Table 2 and clicking "Record Payment" in a sub-table RFP row opens the payment modal correctly (same modal as Table 1). After recording a payment, BOTH Table 1 and Table 2 update in real-time without a page reload.
result: pass
