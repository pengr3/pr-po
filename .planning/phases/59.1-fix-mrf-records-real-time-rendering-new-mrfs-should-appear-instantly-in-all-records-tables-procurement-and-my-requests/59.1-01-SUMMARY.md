# Phase 59.1 Plan 01: Fix MRF Records Real-Time Rendering Summary

**One-liner:** Remove historical-status server filter from loadPRPORecords and bust cache after MRF create so Pending/Approved MRFs appear in Records tab immediately.

## What Was Built

Three targeted changes to `app/views/procurement.js`:

1. **Query fix** — `loadPRPORecords()` previously used `where('status', 'in', historicalStatuses)` which excluded `Pending` and `Approved` MRFs entirely. Replaced with a plain `getDocs(mrfsRef)` so all MRFs are fetched. The existing `filterPRPORecords()` already handles client-side filtering, so removal of the server-side restriction is safe.

2. **Cache invalidation** — `saveNewMRF()` now sets `_prpoRecordsCachedAt = 0` immediately after `addDoc()` succeeds. Previously, the 5-minute cache was never invalidated after creating an MRF, so switching to the Records tab within 5 minutes would serve the pre-create snapshot and hide the new record.

3. **Filter dropdown expansion** — Added `Pending` and `Approved` as selectable options in the `histStatusFilter` dropdown so users can filter the newly visible statuses.
