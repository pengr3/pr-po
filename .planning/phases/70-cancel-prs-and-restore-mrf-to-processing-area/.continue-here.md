<decisions_made>

- **Right-click trigger only** — no action bar button; consistent with existing PO ID right-click pattern for RFPs
- **Two-path recall (Option B)**:
  - All PRs Pending/Rejected → simple cancel (delete PRs, restore MRF)
  - Any PR Finance-Approved → check POs: if all Pending Procurement → force recall (void POs to 'Cancelled', delete all PRs, restore MRF)
- **Hard block if PO has procurement progress** — Procuring/Procured/Delivered = no cancel, error toast with PO IDs
- **Safety net at context menu level** — blockedMRFIds Set checked before menu renders; blocked MRFs produce no context menu at all (UX safety net, not just error handling)
- **Void = procurement_status: 'Cancelled'** (not deletion) — preserves audit trail
- **refreshBlockedMRFs efficiency** — single `where('mrf_id', '==', ...)` + `where('procurement_status', 'in', [...])` compound query per MRF; N reads per snapshot cycle instead of N×M; server-side filtering means zero payload for non-blocked MRFs
- **Finance-approval recall flow → BACKLOG.md** — Phase 70 is immediate unblocking only; full Finance-gated recall is future work
</decisions_made>
