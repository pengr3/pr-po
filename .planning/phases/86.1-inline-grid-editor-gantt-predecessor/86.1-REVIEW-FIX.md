## Fixed Issues

### CR-01: Gantt bar drag never recomputes parent dates

**Files modified:** `app/views/project-plan.js`
**Commit:** 8d59a23
**Applied fix:** Replaced the empty `.then()` stub in `handleGanttDateChange` with an `async` callback that patches the local `tasks` array with the new dates for the dragged child task, then calls `recomputeParentDates(t.parent_task_id)` if the task has a parent. This ensures parent summary bar dates are updated in Firestore immediately after any Gantt drag, not left stale until an unrelated grid-cell edit triggers recompute.

### WR-01: ISO date-only strings parsed as UTC, off-by-one in UTC+ timezones

**Files modified:** `app/views/project-plan.js`
**Commit:** 851bfe5
**Applied fix:** In `computeDurationDisplay`, replaced `new Date(start)` + `setHours(0,0,0,0)` with `new Date(start + 'T00:00:00')` for both start and end. In `addDays`, replaced `new Date(isoDate)` with `new Date(isoDate + 'T00:00:00')`. Appending `T00:00:00` forces the Date constructor to parse the string as local midnight rather than UTC midnight, eliminating the off-by-one date shift for users in UTC+ timezones.

### WR-02: Unescaped task ID in CSS attribute selector in initGanttDragLink

**Files modified:** `app/views/project-plan.js`
**Commit:** 1b2e834
**Applied fix:** Wrapped `ganttDragState.fromTaskId` with `CSS.escape()` in the `querySelector` template literal inside the SVG `mousemove` handler. This prevents a `SyntaxError` crash if a task ID ever contains CSS selector special characters (quotes, brackets, backslashes).
