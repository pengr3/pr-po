**Three targeted UAT-diagnosed bug fixes in project-plan.js: bindGridEvents-before-innerHTML eliminates double-write on Name/new-row; re-entry guard with synchronous value clear blocks second handleNewRowCommit invocation; parseDuration accepts bare integers and defaults start_date to today when empty**

## Accomplishments
- renderTaskGrid non-empty path: `bindGridEvents(container)` moved before `container.innerHTML` so old capture-phase blur handler is removed before DOM replacement fires blur on destroyed input nodes — eliminates double-write on Name cell edit (UAT gap 1/test 2)
- handleNewRowCommit: re-entry guard `if (!input.value.trim()) return;` at top + `input.value = ''` clear before first await — second invocation from surviving blur exits immediately, creating exactly one task (UAT gap 2/test 4)
- parseDuration: added `mBare` path accepting bare positive integers as days; `n > 0` guard rejects zero/negative per T-86.1-05-01 threat register
- handleGridCellBlur duration branch: removed `!t.start_date` early-return; `effectiveStart` defaults to `formatDateISO(new Date())` (local time) when `t.start_date` empty; writes `{start_date, end_date}` when start was defaulted, `{end_date}` otherwise (UAT gap 3/test 3)
