# Phase 86.1 Plan 06: D-Q4 Subtree Guard + Resources Free-Text Summary

Closes two UAT-diagnosed defects: (1) drag-over highlighting was shown on descendant rows before the drop was eventually blocked at handleRowDrop, and (2) the Resources column used an app-user pill picker instead of a plain free-text input.

## Decisions Made

- **isDescendant module extraction**: Three identical inline `const isDescendant = (candidateId, ancestorId) => {...}` closures removed from showTaskContextMenu, gridIndentTask, and handleRowDrop. Single module-scope `function isDescendant(...)` declared before renderTaskGrid. All existing call sites continue to resolve via scope chain.
- **_gridDragOverHandler subtree guard**: Added `if (isDescendant(row.dataset.taskId, _draggedTaskId)) return;` before `e.preventDefault()` — descendant rows now receive no blue highlight and no drag-over event during a parent drag (D-Q4 visual fix). handleRowDrop's check unchanged as belt-and-suspenders.
- **Resources free-text**: Cell replaced from `<span class="tg-resource-names">` to `<input class="tg-input tg-res-input" data-col="resources">`. Value sourced from `t.resources || ''` (Firestore string field). handleGridCellBlur now has a `resources` branch that writes `{resources, updated_at}`.
- **New task payloads**: Both handleNewRowCommit docData and gridInsertRowAbove setDoc now include `resources: ''` alongside `assignees: []`.
- **openAssigneePicker deleted**: 66-line function removed entirely. bindGridEvents `.tg-resource-names` click block also removed. destroy() no longer references `taskAssigneePicker`.
- **normalizePersonnel import preserved**: Still present at line 8 per plan instruction — referenced in Plan 04 comment within init(); not removed.
