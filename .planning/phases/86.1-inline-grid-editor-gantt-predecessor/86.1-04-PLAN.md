    2. **Insert `initGanttDragLink()` function** verbatim from PATTERNS.md "SVG overlay pattern" excerpt (lines 729-839). Place AFTER renderGantt() (so it appears below the function that calls it). The function:
       - Queries `document.querySelector('#ganttPane svg')` — the Frappe SVG root
       - For each `.bar-wrapper`, registers:
         - `mouseenter` → if not `.parent-summary-bar` (D-07), create or show a circle `.gantt-link-handle` at the bar's right-edge midpoint via `bar.getBBox()` coordinates
         - `mouseleave` → if no drag in progress, remove the handle
         - `mousedown` on the handle → preventDefault + stopPropagation, read `barWrapper.dataset.id` as fromTaskId, create rubber-band SVG `<line>` with class `.gantt-rubber-band` + dashed stroke, set `ganttDragState = { fromTaskId, svgLine }`
       - On the SVG root:
         - `mousemove` → if dragging, transform `e.clientX/Y` to SVG coords via `getScreenCTM().inverse()`, update line's x1/y1 (from-bar right-edge) and x2/y2 (cursor)
         - `mouseup` → if dragging:
           1. Clear ganttDragState and remove svgLine
           2. Resolve drop target via `e.target.closest('.bar-wrapper')` — null/empty space → cancel
           3. Read `toTaskId` from data-id; reject if same as fromTaskId or missing
           4. Look up target task in `tasks[]`; if not found → cancel
           5. Build `existingDeps`; reject silently if `existingDeps.includes(fromTaskId)` (already linked)
           6. Build `newDeps = [...existingDeps, fromTaskId]`
           7. Run `detectDependencyCycle(newDeps, toTaskId)` — if truthy, toast `This dependency would create a cycle: ${cycleNames.join(' → ')}. Link rejected.` (verbatim copy from PATTERNS.md) and return without writing
           8. `await updateDoc(doc(db, 'project_tasks', toTaskId), { dependencies: newDeps, updated_at: serverTimestamp() })` — onSnapshot will refresh

    3. **Wire initGanttDragLink call inside renderGantt()** — at the end of renderGantt(), AFTER the existing `renderTodayLine()` call and the `__ganttInitialScrollDone` block, add:
       ```javascript
       // Phase 86.1 Plan 04 — drag-to-link overlay
       initGanttDragLink();
       ```
       This runs every time renderGantt fires (every snapshot), re-binding handlers on the freshly-rendered SVG. The handlers don't accumulate because each call queries `.bar-wrapper` elements on the new SVG; the old SVG is replaced wholesale by Frappe on rebuild and refresh-in-place keeps the same elements (re-binding via addEventListener is safe because Frappe re-creates the DOM elements on rebuild and addEventListener on already-bound elements is idempotent for the same handler).

    4. **Defensive de-duplication on bar-wrapper handlers** — Plan 04 should NOT add the same listener twice if Frappe's `gantt.refresh()` keeps the same DOM elements. To prevent duplicate handlers, use a sentinel attribute:
       ```javascript
       svg.querySelectorAll('.bar-wrapper').forEach(barWrapper => {
           if (barWrapper.dataset.linkBound === '1') return;  // already wired
           barWrapper.dataset.linkBound = '1';
           // ...mouseenter / mouseleave / mousedown bindings...
       });
       ```
       The svg-level `mousemove` and `mouseup` listeners ALSO need de-duplication. Use a different sentinel:
       ```javascript
       if (svg.dataset.linkBound !== '1') {
           svg.dataset.linkBound = '1';
           svg.addEventListener('mousemove', /* ... */);
           svg.addEventListener('mouseup', /* ... */);
       }
       ```
       This is a deviation from PATTERNS.md (which doesn't show de-dup); document in the SUMMARY.


## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → SVG mouse events | User-supplied mouse coordinates control rubber-band line position; user controls drop target via mouseup target |
| SVG DOM → Firestore | Drop event resolves a task_id from `barWrapper.dataset.id` and writes it to dependencies[]; data-id is set by Frappe Gantt from the task spec we provide |
| Frappe Gantt internals → our code | We mutate Frappe's rendered SVG by appending overlay elements; if Frappe's structure changes between v1.2.2 and a future upgrade, our queries may silently no-op |

Server-side rules don't enforce dep validity. v4.1 hygiene candidate. |
