## CRITICAL

### CR-01: Gantt bar drag never recomputes parent dates

**File:** `app/views/project-plan.js` — `handleGanttDateChange`
**Confidence:** 95

`handleGanttDateChange` writes `start_date`/`end_date` to Firestore after a Gantt drag but never calls `recomputeParentDates`. The inline comment "Plan 04's parent-recompute helper will fire from the snapshot callback" is incorrect — the `onSnapshot` callback only calls `renderTaskGrid()`, `renderGantt()`, and `checkAndToastFsViolations()`. None of those trigger `recomputeParentDates`. The grid-cell blur path in `handleGridCellBlur` does call it, but Gantt drag bypasses that path entirely.

**Effect:** Dragging any child task bar leaves its parent's Firestore `start_date`/`end_date` permanently stale. The parent summary bar and grid locked-date display show wrong dates until an unrelated grid-cell edit accidentally triggers recompute.

**Fix — add parent recompute after the `updateDoc` write in `handleGanttDateChange`:**

```javascript
.then(async () => {
    if (t.parent_task_id) {
        const idx = tasks.findIndex(x => x.task_id === t.task_id);
        if (idx >= 0) tasks[idx] = { ...tasks[idx], start_date: newStart, end_date: newEnd };
        await recomputeParentDates(t.parent_task_id);
    }
})
```

## WARNINGS

### WR-01: ISO date-only strings parsed as UTC, displayed in local time — off-by-one in UTC+ timezones

**File:** `app/views/project-plan.js` — `addDays`, `computeDurationDisplay`
**Confidence:** 85

`new Date('2026-05-06')` (date-only ISO string) is parsed as UTC midnight per spec. `formatDateISO` reads it back with local-time methods (`getFullYear`, `getMonth`, `getDate`). In UTC+8, this is 2026-05-05T16:00 local, so `getDate()` returns 5, not 6.

**Effect:** Every duration-based end-date computation produces a date one day earlier than intended for users in UTC+ timezones. Affects Duration column display and Firestore `end_date` writes from duration editing.

**Fix — force local-midnight parse in both functions:**

```javascript
function addDays(isoDate, n) {
    const d = new Date(isoDate + 'T00:00:00');
    d.setDate(d.getDate() + n);
    return formatDateISO(d);
}

function computeDurationDisplay(start, end) {
    if (!start || !end) return '';
    const s = new Date(start + 'T00:00:00');
    const e = new Date(end   + 'T00:00:00');
    const days = Math.max(1, Math.round((e - s) / 86400000) + 1);
    return `${days}d`;
}
```

### WR-02: Unescaped task ID in CSS attribute selector in `initGanttDragLink`

**File:** `app/views/project-plan.js` — `initGanttDragLink`
**Confidence:** 82

```javascript
const fromWrapper = svg.querySelector(`.bar-wrapper[data-id="${ganttDragState.fromTaskId}"]`);
```

`ganttDragState.fromTaskId` is unescaped in the CSS selector template literal. If a task ID ever contains CSS selector special characters (quotes, brackets, backslashes — possible if a project code has such chars), `querySelector` throws `SyntaxError` and silently crashes the mousemove handler mid-drag.

**Fix:**

```javascript
const fromWrapper = svg.querySelector(`.bar-wrapper[data-id="${CSS.escape(ganttDragState.fromTaskId)}"]`);
```

## INFO

### IR-01: Inline delete-confirm `<span>` appended as direct child of `<tr>` — invalid HTML

**File:** `app/views/project-plan.js` — `gridDeleteRow`
**Confidence:** 80

```javascript
row.appendChild(confirm); // <span> as direct child of <tr> is invalid HTML
```

Only `<td>` and `<th>` are valid direct children of `<tr>`. Browsers silently re-parent the span outside the table, which can cause the confirm widget to be invisible or misplaced.

**Fix:**

```javascript
const nameCell = row.querySelector('.tg-name');
(nameCell || row).appendChild(confirm);
```
