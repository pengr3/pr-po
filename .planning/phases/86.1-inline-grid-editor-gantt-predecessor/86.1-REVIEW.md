## WARNING

### WR-01: handleNewRowCommit silently discards task name on project_code error path

**File:** `app/views/project-plan.js` — `handleNewRowCommit`
**Confidence:** 85

The re-entry guard (Plan 05 Fix B) clears `input.value = ''` synchronously before checking `currentProject?.project_code`. On the error path — when the project has no code yet — the toast fires but the user's typed task name is already gone. They must re-type the task name after dismissing the warning.

**Fix:** Restore `input.value = name` in the `project_code` guard branch before returning:
```javascript
if (!currentProject?.project_code) {
    input.value = name; // restore so user doesn't lose their text
    showToast(`This project doesn't have a project code yet.`, 'warning');
    return;
}
input.value = ''; // clear only after guard passes
```

### WR-02: isDescendant O(depth × n) linear scan fires in the hot dragover path (~60 Hz)

**File:** `app/views/project-plan.js` — `isDescendant` (module scope, line ~187)
**Confidence:** 80

`isDescendant` uses `tasks.find()` (O(n)) for each ancestor step. The `_gridDragOverHandler` calls `isDescendant` on every `dragover` event, which fires at ~60 Hz during a drag. For a project with 200 tasks and 10-level hierarchy this is ~2,000 linear scans per second.

**Fix:** Build a Map once inside `isDescendant` to reduce cost from O(depth × n) to O(n + depth):
```javascript
function isDescendant(candidateId, ancestorId) {
    const byId = new Map(tasks.map(t => [t.task_id, t]));
    let cur = byId.get(candidateId);
    while (cur?.parent_task_id) {
        if (cur.parent_task_id === ancestorId) return true;
        cur = byId.get(cur.parent_task_id);
    }
    return false;
}
```

### IR-01: parseDuration silent failure gives no user feedback for invalid formats

**File:** `app/views/project-plan.js` — `handleGridCellBlur` duration branch
**Confidence:** 80

When the user types an unrecognized format (e.g. "3.5", "abc") in the Duration cell, `parseDuration` returns `null` and the handler returns silently. The cell reverts to the original value on the next snapshot with no explanation. Users may be confused why their input was ignored.

**Fix:** Add a toast when `parseDuration` returns null for non-empty input:
```javascript
const days = parseDuration(rawValue);
if (days === null) {
    if (rawValue) showToast('Enter duration as days (e.g. "3d") or weeks (e.g. "2w").', 'warning');
    return;
}
```
