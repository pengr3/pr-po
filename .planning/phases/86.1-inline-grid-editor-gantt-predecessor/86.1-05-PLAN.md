<task type="auto">
  <name>Task 1: Fix double-write — call bindGridEvents before innerHTML assignment</name>
  <files>app/views/project-plan.js</files>
  <action>
Root cause: In the non-empty path of renderTaskGrid() (around line 295), container.innerHTML is set BEFORE bindGridEvents(container) is called. When innerHTML is replaced, the focused input's DOM node is destroyed, triggering a blur event that fires the OLD capture-phase handler (_gridInputHandler) still registered on the container — causing handleGridCellBlur or handleNewRowCommit to run a second time.

Fix A — reorder the two lines in renderTaskGrid() non-empty path:

BEFORE (lines ~295-309):
```javascript
    container.innerHTML = `
      <table class="task-grid">
        ...
      </table>`;

    bindGridEvents(container);
```

AFTER (swap the two operations):
```javascript
    bindGridEvents(container);

    container.innerHTML = `
      <table class="task-grid">
        ...
      </table>`;
```

The call `bindGridEvents(container)` at the top of that function removes the old _gridInputHandler, _gridChangeHandler, _gridContextMenuHandler, _gridDragStartHandler, _gridDragOverHandler, _gridDropHandler, _gridDragEndHandler via removeEventListener before the innerHTML replacement destroys DOM nodes and fires blur. The new handlers are registered after, then innerHTML is set, so they apply to the new DOM.

Fix B — defense-in-depth re-entry guard in handleNewRowCommit:

At the very top of handleNewRowCommit (line 475), BEFORE reading input.value, add:
```javascript
async function handleNewRowCommit(input) {
    if (!input.value.trim()) return;          // re-entry guard: second call finds empty value → exits
    const name = input.value.trim();
    input.value = '';                          // clear synchronously before first await
    ...rest unchanged...
```

This means: the first invocation reads the name, clears input.value, and proceeds. If the blur fires again (before the snapshot updates tasks[]), the second invocation sees input.value === '' and returns immediately.

Apply BOTH fixes. Fix A is the primary structural fix; Fix B is defense-in-depth that prevents any surviving race path.

The empty-state path in renderTaskGrid() (sorted.length === 0, around line 217) already calls bindGridEvents(container) before setting container.innerHTML — do NOT change that path.

<task type="auto">
  <name>Task 2: Fix Duration UX — accept bare integers and default Start to today</name>
  <files>app/views/project-plan.js</files>
  <action>
Two sub-fixes in the same file, both in the duration handling code path.

Sub-fix 2a — parseDuration accepts bare positive integers:

Current (line 826):
```javascript
    const m = str.trim().match(/^(\d+)\s*([dw])$/i);
    if (!m) return null;
    return m[2].toLowerCase() === 'w' ? parseInt(m[1], 10) * 7 : parseInt(m[1], 10);
```

Replace with:
```javascript
    const s = str.trim();
    // Accept "3d", "2w" (with optional space), OR a bare positive integer treated as days
    const mSuffix = s.match(/^(\d+)\s*([dw])$/i);
    if (mSuffix) return mSuffix[2].toLowerCase() === 'w' ? parseInt(mSuffix[1], 10) * 7 : parseInt(mSuffix[1], 10);
    const mBare = s.match(/^(\d+)$/);
    if (mBare) {
        const n = parseInt(mBare[1], 10);
        return n > 0 ? n : null;
    }
    return null;
```

Sub-fix 2b — default start_date to today if empty when Duration is entered:

Current (col === 'duration' branch in handleGridCellBlur, lines 420-426):
```javascript
    } else if (col === 'duration') {
        // Parse "3d" / "2w" (D-03)
        const days = parseDuration(rawValue);
        if (days === null || !t.start_date) return;
        const endDate = addDays(t.start_date, days - 1); // inclusive: 1d => same day
        if (endDate === t.end_date) return;
        updateData = { end_date: endDate, updated_at: serverTimestamp() };
```

Replace with:
```javascript
    } else if (col === 'duration') {
        // Parse "3d" / "2w" / bare integer (D-03)
        const days = parseDuration(rawValue);
        if (days === null) return;
        // Default start_date to today if empty
        const effectiveStart = t.start_date || formatDateISO(new Date());
        const endDate = addDays(effectiveStart, days - 1); // inclusive: 1d => same day
        if (effectiveStart === t.start_date && endDate === t.end_date) return;
        updateData = effectiveStart !== t.start_date
            ? { start_date: effectiveStart, end_date: endDate, updated_at: serverTimestamp() }
            : { end_date: endDate, updated_at: serverTimestamp() };
```

Note: formatDateISO(new Date()) produces today as 'YYYY-MM-DD' in local time because formatDateISO uses getFullYear/getMonth/getDate (not UTC). This is the desired behavior — same as addDays already does. Verify formatDateISO is already defined in this file (it is; used by addDays at line 844).
