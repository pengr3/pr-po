## What Was Built

### Task 1: Context Menu + Hierarchy Operations (commit c2530ab)

Lines added to `app/views/project-plan.js` (+234 lines net):

**`showTaskContextMenu(event, taskId)`** (lines 472â€“545):
- Removes any existing `#taskGridContextMenu` before creating a new one (prevents DOM accumulation, T-86.1.2-07)
- Computes `rowIdx`, `isFirstRow`, `rowAbove` from `document.querySelectorAll('.tg-row:not(.tg-empty-row)')`
- Inline `isDescendant()` helper: walks parent chain via `tasks[]` to detect descendant relationship
- `canIndent`: false when first row OR row above is a descendant of this task
- `canOutdent`: `!!t.parent_task_id`
- `hasChildren` + `childCount` computed for Delete Row label
- Menu at `position:fixed; z-index:10000` with 4 items + outside-click dismissal (10ms one-shot)
- `escapeHTML(taskId)` in every onclick attribute string (T-86.1.2-01 / T-86.1.2-08)

**`gridIndentTask(taskId)`**:
- Sorts tasks by row_order, finds row above â†’ becomes new parent
- Cycle guard: rejects if rowAbove is a descendant of the task being indented
- `updateDoc({ parent_task_id: newParent.task_id, updated_at })` + local patch
- `recomputeParentDates(oldParentId)` if old parent existed, then `recomputeParentDates(newParent.task_id)` â€” both envelope updates

**`gridOutdentTask(taskId)`**:
- `newParentId = oldParent?.parent_task_id || null` (moves one level up)
- `updateDoc({ parent_task_id: newParentId, updated_at })` + local patch
- `recomputeParentDates(oldParentId)` + `recomputeParentDates(newParentId)` if new parent exists

**`gridInsertRowAbove(taskId)`**:
- Allocates new task_id via 5-attempt collision-retry (mirrors handleNewRowCommit pattern)
- `toShift = tasks.filter(x => x.row_order >= targetOrder)` â†’ writeBatch CHUNK=450 incrementing each by 1
- `setDoc` new task at `targetOrder` with `parent_task_id` inherited from displaced row

**`gridDeleteRow(taskId)`** (D-05 no modal):
- `hasChildren` check via `tasks.some(x => x.parent_task_id === taskId)`
- Parent task: appends inline `<span class="tg-delete-confirm">` to the row with Yes/No buttons; Yes â†’ `deleteTaskNow(taskId)`, No â†’ `confirm.remove()`
- Leaf task: calls `deleteTaskNow(taskId)` directly

**Window registrations** in `init()`: `window.gridIndentTask/Outdent/InsertRowAbove/DeleteRow`
**Destroy() teardowns**: all 4 deleted + `document.getElementById('taskGridContextMenu')?.remove()`

**Rule 1 Fix (closeDeleteTaskConfirm bug):** `closeDeleteTaskConfirm()` in `deleteTaskNow` replaced with `document.querySelector('.tg-delete-confirm')?.remove()` â€” the modal function was deleted in Plan 01, leaving a dead reference.

### Task 2: HTML5 Drag-and-Drop Row Reordering (commit 7e39417)

Lines added to `app/views/project-plan.js` (+129 lines net):

**Module-scope drag state** (after existing handler refs):
- `_gridDragStartHandler`, `_gridDragOverHandler`, `_gridDropHandler`, `_gridDragEndHandler` â€” refs for idempotent re-bind in bindGridEvents + cleanup in destroy()
- `_draggedTaskId` â€” tracks currently-dragged task_id across event callbacks

**`bindGridEvents()` extended** (drag handlers at end of function):
- Idempotent: removes previous handlers before adding new ones (same pattern as context menu handler)
- `dragstart`: blocked if `e.target` is INPUT or has `.tg-input` class (preserves inline editing); sets `_draggedTaskId`, adds `.tg-dragging`
- `dragover`: adds `.tg-drag-over` visual hint on target row; clears others
- `drop`: calls `handleRowDrop(_draggedTaskId, targetTaskId)`
- `dragend`: clears `.tg-dragging` / `.tg-drag-over` from all rows, nulls `_draggedTaskId`

**`handleRowDrop(draggedTaskId, targetTaskId)`** (D-Q4 validation):
- Constraint 1: target is dragged task's parent â†’ toast "Can't move a task above its parent. Outdent it first."
- Constraint 2: target is a descendant of dragged â†’ toast "Can't drop a task into its own subtree."
- Computes new order: removes dragged from sorted array, splices at target position, calls `commitRowOrderReorder()`

**`commitRowOrderReorder(reorderedTaskIds)`**:
- CHUNK=450 writeBatch loop assigning `row_order: globalIdx + 1` (1-based) to each task
- `updated_at: serverTimestamp()` on every updated doc (satisfies tier-1 Firestore rule requirement)

**destroy() teardown** extended:
- Removes all 4 drag event listeners from gridContainer
- Nulls all 4 handler refs + `_draggedTaskId`
