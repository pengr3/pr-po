## Deviations from Plan
**1. Cue id scheme.** Plan suggested `cue-${rfp.rfp_id || random}`. Because the same RFP renders in BOTH the RFP table and the PO-summary sub-table, an rfp_id-keyed id would produce duplicate DOM ids (invalid HTML; getElementById returns the first → tap-toggle targets the wrong tooltip). Used a monotonic module counter (`_feeCueSeq`) for guaranteed uniqueness per render. Correctness improvement; all acceptance greps still pass.
