## Deviations from Plan
**1. TR blocked Submit also gets `data-locked="1"` (beyond the plan's stated DF + all-tranches scope).** Necessary correctness fix: the new `validateRFPFees → setRFPSubmitEnabled(true)` path would otherwise re-enable TR's "RFP already exists" Submit (which must stay disabled). The server-side TR guard still blocks the write regardless, but the button should reflect the locked state.
**2. `.field-err` inserted after `.fee-row`** (full-width line below the row) rather than after `.fee-amount-wrap` (which would inject a 4th cell into the 3-column grid). Cleaner, no layout break; acceptance greps unaffected.
