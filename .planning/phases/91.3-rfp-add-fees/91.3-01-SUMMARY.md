# Phase 91.3 Plan 01: Shared RFP Fee-Math Helpers Summary

**Four legacy-safe fee helpers + one field-name constant exported from `app/utils.js` — the Wave-1 contract every downstream plan consumes for fee math.**

## LOCKED Field Shape (consumed by Plans 02–06 — identical contract)

| Field | Type | Meaning |
|-------|------|---------|
| `amount_requested` | number | **BASE** RFP amount — UNCHANGED meaning, always recoverable |
| `transfer_fee` | number | optional supplementary fee; `0`/absent when not applied |
| `cash_out_fee` | number | optional supplementary fee; `0`/absent when not applied |
| `misc_fees` | `Array<{label:string, amount:number}>` | zero-or-more arbitrary fee lines; `[]`/absent when none |
| `total_with_fees` | number | persisted grand total = base + transfer + cash-out + Σ misc; ABSENT on legacy docs |

Legacy RFP (none of the fee fields present) → `getRFPTotal === amount_requested` exactly as today.

## Exported API

```javascript
export const RFP_FEE_FIELDS = ['transfer_fee', 'cash_out_fee', 'misc_fees', 'total_with_fees'];
export function getRFPFees(rfp)        // → {transfer, cashOut, misc[], feesTotal}
export function getRFPTotal(rfp)       // → number (fee-inclusive grand total, legacy-safe)
export function getRFPFeeBreakdown(rfp)// → {base, lines[], feesTotal, grandTotal, hasFees}
```
