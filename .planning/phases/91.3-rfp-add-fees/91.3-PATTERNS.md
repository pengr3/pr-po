### C. `finance.js` — payables/status compute (swap to fee-inclusive total)

**`derivePOSummary(rfpList, poTotalAmount)` — `:889`.** Currently sums `amount_requested` for the fallback total and uses PO `total_amount` as authoritative:
```javascript
const totalAmount = (poTotalAmount != null && poTotalAmount > 0)
    ? poTotalAmount
    : regularRFPs.reduce((s, r) => s + (r.amount_requested || 0), 0);
const totalPaid = regularRFPs.reduce((s, r) => s + (r.payment_records || [])
    .filter(p => p.status !== 'voided').reduce((ps, p) => ps + (p.amount || 0), 0), 0);
const remaining = totalAmount - totalPaid;
```
**⚠ Subtle:** when `poTotalAmount` is passed it WINS over the RFP sum. Fees are NOT in the PO `total_amount`, so the planner must add `Σ fees` on top of `poTotalAmount` (or sum the fee-inclusive RFP totals) for `Total payable` to reflect fees (D-16/sketch-004). This is the trickiest single edit — the authoritative-PO-total branch silently drops fees if untouched.

**`deriveRFPStatus(rfp)` — `:34` (finance) and `:550` (procurement, duplicate logic):**
```javascript
if (totalPaid >= rfp.amount_requested && rfp.amount_requested > 0) return 'Fully Paid';
```
Swap `rfp.amount_requested` → fee-inclusive total in BOTH copies (they are duplicated across the two view files — keep them in sync). Legacy-safe: `(feeInclusiveTotal(rfp))` must fall back to `amount_requested` when no fee fields.

**Analog to replicate:** the existing `(rfp.amount_requested || 0)` null-coalescing IS the legacy-safe template — wrap fee reads identically (`rfp.transfer_fee || 0`). Recommend a single shared helper (e.g. `rfpGrandTotal(rfp)`) defined once and reused by every consumer to avoid drift between the duplicated derive functions.

## Shared Patterns

### Window-function handler binding (CLAUDE.md mandatory)
**Source:** `procurement.js:1505` (`window.showAltBank = showAltBank;`), `:1536` (`window.toggleRFPBankFields = ...`)
**Apply to:** every new fee handler (`addMiscFeeRow`, `removeMiscFeeRow`, `revealTransferFee`, `recomputeRFPTotal`, `toggleFeeCue`, etc.). Any function named in an inline `onclick`/`oninput` MUST be on `window`.

### Legacy-safe field reads (D-12)
**Source:** the `(rfp.amount_requested || 0)` idiom used at ~18 finance sites
**Apply to:** all fee reads — `(rfp.transfer_fee || 0)`, `(rfp.cash_out_fee || 0)`, `(rfp.misc_fees || [])`. Recommend one shared `rfpGrandTotal(rfp)` helper so the duplicated derive functions can't drift.

### Currency rendering
**Source:** `formatCurrency()` (`app/utils.js:36`) — already imported in both views
**Apply to:** every fee echo, running total, table amount, tooltip line. Add `font-variant-numeric:tabular-nums` + `₱` prefix per UI-SPEC.

### Inline-styled modal markup (no framework)
**Source:** all 3 RFP modals + `openRecordPaymentModal` — `.modal`/`.modal-content`/`.modal-header`/`.modal-body`/`.modal-footer` + inline styles + `.form-control`/`.btn`/`.btn-outline`/`.btn-primary`
**Apply to:** the sectioned overhaul + breakdown card. Prefix new classes `fee-`/`rfp-` to avoid collision (UI-SPEC §Component Inventory).

### Read-form-by-CSS-class (CLAUDE.md DOM rule)
**Source:** `generatePR()` / `saveNewMRF()` loops over `#lineItemsBody tr` reading `.item-name`, `.item-qty` via `querySelector` (`:3257`, `:3875`, `:6219`)
**Apply to:** reading Misc-fee rows back in the submit handlers — `querySelectorAll('#miscFeesBody tr')` + `.fee-misc-label`/`.fee-misc-amount`, **never** data attributes.
