## Accomplishments

- Rewrote `initPanelResize()` to mutate `ganttPane.style.left` and `planDivider.style.left` instead of `splitPane.style.gridTemplateColumns` — task grid columns never shrink on drag
- Added `_paneDividerPct = 35` module-scope variable for persistent drag state; reset in `destroy()`
- Rewrote split-pane CSS: `.plan-split-pane` is now `position:relative + overflow:hidden + display:flex`; `.gantt-pane` is `position:absolute` at `z-index:10`; `.plan-divider` is `position:absolute` at `z-index:20`

## Files Created/Modified

- `styles/views.css` - `.plan-split-pane` position:relative/overflow:hidden/display:flex; `.plan-divider` position:absolute/z-index:20; `.task-grid-rail` position:relative/flex:0 0 100%; `.gantt-pane` position:absolute/z-index:10; mobile breakpoint preserves single-pane layout

## Decisions Made

- Curtain approach: task-grid-rail declared as `flex: 0 0 100%; min-width: 100%` so it fills the full container width. Gantt pane overlaps from `left: 35%` as absolute-positioned overlay. No `gridTemplateColumns` mutation on drag.
- `_paneDividerPct` module-scope state is the source of truth for the current divider position. Applied on every `initPanelResize()` call so the position persists within a session (until destroy resets it to 35).
- `destroy()` resets `_paneDividerPct = 35` for clean re-entry — next view-enter always starts at 35%.
- Mobile `@media (max-width: 768px)`: `.gantt-pane { display: none }` and `.task-grid-rail { position: relative; flex: 1 1 auto; min-width: 0 }` preserves single-pane mobile behavior.
- The 2px left-offset on the divider (`calc(pct% - 2px)`) centers the 4px divider bar on the panel boundary — its visual center lands exactly at the percentage boundary.
