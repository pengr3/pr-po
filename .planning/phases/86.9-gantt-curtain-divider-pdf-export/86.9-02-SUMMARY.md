## Implementation Details

### exportGanttPDF() function (project-plan.js)

- Derives `projectName` from `currentProject?.project_name || projectCode || 'Project'`
- Iterates `tasks[]` (module-scope snapshot array) to build summary table rows
- Captures `document.getElementById('ganttPane').innerHTML` as the Gantt snapshot
- Injects `<div id="gantt-print-frame">` into `document.body` with full print content
- Calls `window.print()` then removes frame after 800ms timeout
- Shows `showToast('Opening print dialog — choose "Save as PDF" to download.', 'info')`

### Export button (render() HTML)

```html
<button type="button" class="plan-export-btn" onclick="window.exportGanttPDF()" title="Export Gantt as PDF">Export</button>
```

Positioned after closing `</div>` of `.zoom-pill-group`, before `<label class="plan-toolbar-toggle"`.

### Window registration and destroy() cleanup

- `init()`: `window.exportGanttPDF = exportGanttPDF;` (after `window.toggleCriticalPath`)
- `destroy()`: `delete window.exportGanttPDF;` (after `delete window.toggleCriticalPath`)

### CSS (views.css)

- `.plan-export-btn`: 16px border-radius pill, 1.5px blue border, transparent background, fills blue on hover — matches zoom-pill aesthetic
- `@media print` block: hides all `body > *:not(#gantt-print-frame)`, shows `#gantt-print-frame`
- Print styles for `.pdf-header`, `.pdf-gantt-section`, `.pdf-table-section`, `.gantt-print-snapshot`, `.pdf-summary-table`
