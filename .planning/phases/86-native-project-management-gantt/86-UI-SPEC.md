## Spacing Scale

Phase 86 uses the existing CLMC spacing convention (rem-based; `1rem = 16px`)
and only references multiples-of-4. Source: `styles/main.css:104-140`,
`styles/components.css` button + card paddings, `app/views/project-detail.js`
inline styles.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`0.25rem`) | Chevron-to-label gap in left rail tree; pill-search inline padding |
| sm | 8px (`0.5rem`) | Gap inside `.btn`; horizontal padding on `.btn-sm`; gap between filter chips; gap inside Highlights sub-cells |
| md | 16px (`1rem`) | Default card-body padding; modal section gap; row gap on filter toolbar; gap between summary stats and Highlights block |
| lg | 24px (`1.5rem`) | `.card-body` padding (Status & Assignment card precedent at `project-detail.js:453`); padding around split-pane; bottom margin between cards |
| xl | 32px (`2rem`) | `.container` page padding; padding-bottom of plan view full-screen surface |
| 2xl | 48px (`3rem`) | Reserved — empty-state vertical padding inside the plan-view right pane when no tasks exist |
| 3xl | 64px (`4rem`) | Not used in Phase 86 |

**Exceptions / declared specifics:**

- **Split-pane gutter:** 1px solid `var(--gray-200)` (`#dadce0`) divider between left rail and right Gantt pane. No additional padding on the gutter itself.
- **Frappe Gantt internal row height:** library default (~38 px) — accepted as-is. Do NOT override unless tasks visibly clip; revisit only on overflow.
- **Milestone diamond size:** 14 px (CONTEXT Specifics lean) — fits inside Frappe's default row height with 12 px breathing room.
- **Today vertical line:** 2 px wide — readable across all zoom modes, doesn't compete with Frappe's tick marks.
- **Parent summary bar:** 4 px tall (CONTEXT Specifics lean) — visually distinct from the default ~24 px child bar without crowding the row.
- **Left rail row height:** 32 px (8-px multiple, slightly denser than the 38-px Gantt rows because rail rows carry only text; CSS will leave whitespace so each rail row aligns vertically with its corresponding Gantt row via `aria-rowindex` synchronization or padding tweak).

Inherits the CLMC `body` stack (`'Segoe UI', Tahoma, Geneva, Verdana, sans-serif`).
Sizes match the existing `.text-*` utilities at `styles/main.css:147-150` and the

|------|------|--------|-------------|-------------------|
| Body | 14 px (`0.875rem` / `.text-sm`) | 400 | 1.5 | Task name in left rail; Gantt bar label; description preview; toolbar control labels; modal form inputs |
| Label | 14 px (`0.875rem`) | 600 (`.font-semibold`) | 1.5 | Form field labels in Add/Edit Task modal; toolbar control captions ("Zoom:", "Filter:"); Highlights sub-cell titles ("Most recent accomplishment", "Next milestone", "Ongoing milestone") |

**Exception:** Frappe Gantt renders its own internal SVG text (date axis labels, bar progress label) at the library default — accepted as-is to avoid forking the library. We do NOT theme the SVG `<text>` nodes.

| Destructive | `#ef4444` (today line, parent-detach warning, RFP-style danger) and `#ea4335` (`--danger` for `.btn-danger`) | Today line; FS-violation arrow highlight; "Delete Task" confirm button (uses existing `.btn-danger`) |
| Warning | `#f59e0b` (CONTEXT lean) | **Milestone diamond fill** — only place this color appears in Phase 86 |

**Accent (`#1a73e8`) reserved for — explicit, exhaustive list:**

1. Primary CTA buttons: `+ Add Task` (toolbar), `Open Plan` (summary card), `Save` (Add/Edit Task modal)
2. Default Gantt task bar fill (leaf tasks only)
3. Active state of zoom toolbar pill (`Day` | `Week` | `Month` — selected pill gets `background: var(--primary); color: white`)
4. The big numeric display on summary card ("`{N} tasks`" and "`{Y}% complete`" use `color: var(--primary)`)
5. Hover/focus ring on form inputs inside modals (existing `.form-control:focus` pattern)
6. Highlights link target on summary card: clicking "Next milestone: {task name}" deep-links into the plan view at that task — link uses `color: var(--primary); text-decoration: underline`

Accent is **NEVER** used for: task name text, parent summary bar, milestone diamond, today line, secondary buttons, table header backgrounds, FS-dependency arrows (default), or non-active zoom pills.

**Destructive (`#ef4444` / `--danger`) reserved for — explicit list:**

1. Today vertical line on Gantt (CONTEXT D-09 lean)
2. Highlighted FS-dependency arrow when D-10 violation occurs
3. `.btn-danger` class on "Delete Task" confirm button (existing rule at `components.css:206`)
4. "Delete this task and its `{N}` subtasks?" modal title color (existing modal-header `.text-danger` pattern)
5. Toast `type='error'` background (existing `showToast` utility — no new styling)

**Warning (`#f59e0b`) reserved for — exactly one element:**

1. Milestone diamond SVG fill (CONTEXT D-08 + Specifics lean) — overrides the default Frappe `.bar` rectangle via `.milestone-marker` custom class

## Interaction Contract

Locked by CONTEXT decisions; restated here for the executor.

| Surface | Interaction | Behavior |
|---------|-------------|----------|
| Toolbar | Click `+ Add Task` | Opens `#taskFormModal` in Add mode; pre-fills `start_date = today`, `end_date = today + 7d`. If `project.project_code` is null, button is disabled with tooltip pointing to clientless block copy. |
| Left rail row | Click row body | Selects row (highlight) and scroll-syncs the Gantt pane to that task's bar (smooth scroll, x-axis only). |
| Left rail row | Click chevron | Toggle expand/collapse (children rows show/hide; Gantt rows for that subtree show/hide in lockstep). State persists in module-scope, NOT in Firestore. |
| Left rail row | Click task name | Opens `#taskFormModal` in Edit mode pre-populated with the task. |
| Left rail row | Click `Delete` icon button | Opens `#deleteTaskConfirmModal`. If task has children, copy switches to "Delete Task and Subtasks?" variant. Cascades on confirm (children-first delete, parent last — Phase 82 D-05 cascade pattern). |
| Gantt task bar | Drag bar body | Reschedules task (preserves duration). On mouseup → single Firestore `updateDoc` of `{start_date, end_date}` + parent recompute (CONTEXT D-11). FS-dep check fires per CONTEXT D-10 → toast if violated. |
| Gantt task bar | Drag bar edge | Resizes task. Same write path as bar-body drag. |
| Gantt task bar | Drag progress handle | Updates progress 0–100. Single `updateDoc` of `{progress}`. Permission check at JS layer (and at rule layer per D-18). |
| Gantt task bar | Hover | Frappe default tooltip (task name + dates + progress) — accepted as-is, no theming this phase. |
| Gantt parent bar | Any drag attempt | **Disabled** via `.parent-summary-bar { pointer-events: none }` on resize/move handles (CONTEXT D-11). |
| Gantt milestone marker | Hover / drag | Frappe milestone tasks have `start_date == end_date`; drag reschedules the single date. Click-to-edit opens modal in Edit mode. |
| Toolbar zoom pills | Click `Day` / `Week` / `Month` | `gantt.change_view_mode('Day' | 'Week' | 'Month')`. Default = Week (CONTEXT D-07). State persists in module scope (NOT URL-synced this phase). |
| Toolbar `Filters` | Click | Toggles a filter panel (slides down from toolbar) with two controls per CONTEXT D-21: date range (From + To native `<input type="date">`) and personnel multi-select. Apply re-derives the visible task list + re-renders the Gantt with the filtered set. Clear button resets both filters. |
| Today line | Render-time | Auto-scroll the Gantt pane horizontally so today is centered (or left-aligned if project starts in the future). One-shot on `init()`, not on every re-render (CONTEXT D-09). |
| Add/Edit modal save | Click `Save Task` | Validates: name non-empty, end_date >= start_date, dependencies don't form a cycle (CONTEXT D-13). On cycle detected → error toast with the cycle path. On success → `addDoc` or `updateDoc`, then re-fetch parent (if any) and `updateDoc` parent dates. Modal closes; toast `type='success'`: `Task saved.` |
| Delete confirm | Click `Delete` / `Delete All` | Children-first cascade `deleteDoc`s; parent last; then re-fetch and recompute the grandparent's dates. Toast `type='success'`: `Task deleted.` |

## Accessibility Contract

| Concern | Mitigation |
|---------|------------|
| Keyboard focus on left-rail tree | Each row gets `tabindex="0"`; `Enter` opens edit modal; `ArrowUp/Down` moves focus; `ArrowRight/Left` expands/collapses (standard ARIA tree pattern). |

### Non-Blocking Recommendations (carry into planning, not blockers)

1. **Dual-red destructive palette.** Existing `--danger: #ea4335` (Material, used by `.btn-danger` via `main.css :root`) coexists with new overlays at `#ef4444` (today line, FS-violation arrow, parent-detach warning). Both reds will appear on screen simultaneously. Phase 86 reuses existing button class as-is and applies CONTEXT lean to new overlays — do NOT attempt to unify in this phase (would touch the whole app). Future hardening phase could consolidate `:root` to the Tailwind palette.
2. **Left-rail / Gantt-row vertical alignment.** Spec calls for 32 px rail rows lining up with Frappe's ~38 px rows. Executor should validate alignment in browser; simplest fix is to override Frappe's row height to 32 px rather than padding the rail. Capture as a first-executor decision, not a UI-SPEC defect.
3. **Mobile parity carved out.** UI-SPEC explicitly scopes mobile (< 768 px) Gantt out of MVP — list-only with a "Show Timeline" toggle. Planner must NOT promise full mobile Gantt parity in success criteria.
4. **Frappe Gantt SVG a11y limitation.** Accepted MVP limitation; keyboard users edit via the left rail + modal. A "View tasks as table" fallback could be a future polish phase if user feedback flags it.
5. **Subresource Integrity on CDN script.** SRI is not currently applied to the existing Chart.js include either. Not a Phase 86 blocker; could be a separate hardening phase covering both Chart.js and Frappe Gantt.
