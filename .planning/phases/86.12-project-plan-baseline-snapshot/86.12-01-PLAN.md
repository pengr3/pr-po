<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Firestore | Unauthenticated or under-privileged users attempting to write baseline docs |
| Browser → Firestore | Any user attempting to overwrite (update) an existing baseline doc |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86.12-01 | Tampering | baselines subcollection write | mitigate | `allow create` restricted to super_admin + operations_admin; `allow update: if false` makes docs immutable |
| T-86.12-02 | Information Disclosure | baselines read | accept | All active users can read baselines (consistent with project read access); no PII in task date snapshots |
| T-86.12-03 | Elevation of Privilege | saveBaseline() client-side call | accept | Write is still gated by Firestore security rules even if the function is called by an unauthorized user; rule enforcement is server-side |
</threat_model>
