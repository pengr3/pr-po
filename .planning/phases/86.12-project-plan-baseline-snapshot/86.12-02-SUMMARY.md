# Phase 86.12 Plan 02: Baseline Visual Overlay Summary

**Gantt SVG overlay delivering dashed outline rects and slip badge pills; slip summary row showing behind/ahead/on-track counts above the chart**

## Performance

- **Duration:** ~15 min
- **Started:** 2026-05-29T00:20:00Z

- **Tasks:** 2
- **Files modified:** 2

## Accomplishments
- Added `#baselineSlipSummary` div (hidden by default) immediately above `plan-split-pane` in `render()` HTML
- Added CSS rules in `views.css` after Phase 86.11 bar-status section:
  - `.gantt-baseline-outline` — dashed grey stroke (`#94a3b8`, `stroke-dasharray: 4 3`), no fill, pointer-events none
  - `.gantt-slip-badge` and `.gantt-slip-badge-bg` — SVG text/rect pill elements
  - `.baseline-slip-summary` — flex row with `.slip-behind` (red), `.slip-ahead` (green), `.slip-label` (muted)
- Implemented `dateToX(dateStr)` — converts YYYY-MM-DD to SVG x using `gantt.gantt_start` anchor + `ganttXPerDay()` (mirrors `renderTodayLine()` math); returns 0 on null gantt or parse error
- Implemented `injectBaselineOverlay()`:
  - Early returns if `_baselineData` absent, `gantt` null, or SVG not in DOM
  - Cleans up prior `.gantt-baseline-outline`/`.gantt-slip-badge`/`.gantt-slip-badge-bg` elements before re-injecting
  - For each task with a baseline entry: reads `.bar` rect y/height from Frappe SVG, injects dashed outline rect at baseline date range
  - Computes `diffDays` (current end vs baseline end); injects red `+Nd` or green `-Nd` pill badge for non-zero drift; skips badge for on-track tasks
  - Per-task try/catch prevents one bad task from aborting the rest
  - Does NOT touch `.bar` fill, `.bar` class, or `bar-status-*` class attributes (T-86.12-04 safe)
- Implemented `renderSlipSummary()`:
  - Hides `#baselineSlipSummary` when `_baselineData` is null
  - Counts `behind`/`ahead`/`onTrack` from `tasks[]` vs `_baselineData.tasks`
  - Sets `innerHTML` with `escapeHTML` on baseline label (T-86.12-05 mitigated)
  - Sets `display:flex` when baseline is loaded
- Wired `injectBaselineOverlay()` + `renderSlipSummary()` into `renderGantt()` after step 5e `applyCriticalPathStyles()`
- Wired same calls into `setGanttZoom()` after `applyCriticalPathStyles()` — ensures overlay survives zoom SVG rebuilds

## Decisions Made
- Badge positioned at `curX2 + 4` (right of current bar end) rather than baseline end — user instantly sees where the bar is now and how much it has slipped
- `diffDays === 0` → outline only, no badge — keeps chart clean for on-track tasks
- Per-task try/catch wraps the inner loop body: a single malformed date string silently skips that task (T-86.12-06 mitigated) while the rest of the overlay still renders
- `escapeHTML()` applied to `_baselineData.label` in `renderSlipSummary()` innerHTML (T-86.12-05 mitigated)
- New functions placed after `ganttXPerDay()` and before `renderTodayLine()` to keep all SVG overlay helpers co-located
