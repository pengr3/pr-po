# Phase 93.1 Plan 02: Unified Proposals Table with Filterable Scorecards Summary

Five scorecard tiles and a unified single-table view replace the per-stage card loop on the Home Proposals sub-tab; clicking a tile filters the table client-side without a Firestore round-trip.

## Tasks Completed

| 2 | Wire helpers into _loadHomeProposalsTab, init(), and destroy() | 13a9566 | app/views/home.js |

## What Was Built

Modified `app/views/home.js`:

**Task 1 — New helper functions:**

1. `_homeProposalStatusFilter = null` added as module-level state (alongside `_homeProposalsCache`).
2. `_renderHomeProposalScorecards(proposals, activeFilter)` — renders 5 tiles (one per `STAGE_ORDER` entry) with D-12 border-left colors, `data-status`, `scorecard-label`, `scorecard-count`, and `project-scorecard-card--active` when matching `activeFilter`.
3. `_renderHomeProposalTable(proposals)` — 7-column table (Status | PROP-ID | Title | Project | Client | Amount | Age). Each `<tr>` has `cursor:pointer`, `onmouseenter`/`onmouseleave` hover, and `onclick` calling `window.openProposalModal && window.openProposalModal(id)`. All user-data fields pass through `escapeHTML()`; amount uses `formatCurrency()` (numeric, no escaping needed); status uses `getProposalStatusBadge()` (escapes internally); age uses `renderAgeBadge()` (escapes internally).
4. `_rerenderProposalTable()` — reads from `_homeProposalsCache` (pre-scoped, no Firestore), applies `_homeProposalStatusFilter`, replaces `#homeProposalTableSection` innerHTML, and syncs active-class on scorecard tiles.

**Task 2 — Wiring:**

1. `_loadHomeProposalsTab`: `_homeProposalStatusFilter = null` reset added at top. `const active = scoped.filter(...)` removed; queue block changed to `scoped.filter(p => p.status === 'pending_internal')`. Stage-loop replaced by `<div id="homeProposalTableSection">` containing scorecards + unified table. `renderStageGroupCard` removed from the proposals.js import.
2. `init()`: `window.handleHomeProposalScorecardClick` registered with toggle logic (null if same key clicked again, else set key) then calls `_rerenderProposalTable()`.
3. `destroy()`: `delete window.handleHomeProposalScorecardClick` added.

None — plan executed exactly as written.

## Threat Surface Scan

No new network endpoints, auth paths, file access patterns, or schema changes. All user-data fields in `_renderHomeProposalTable` pass through `escapeHTML()` per T-93.1-02. Scorecard tile labels use `escapeHTML(stage.label)` per T-93.1-03 (labels are from the `STAGE_ORDER` constant but escaping is applied by project convention). `_rerenderProposalTable` reads from `_homeProposalsCache` which is the pre-scoped `filterProposalsForUser` output — no privilege bypass (T-93.1-04 accepted).
