## Queue + Unified Table Layout

| Option | Description | Selected |
|--------|-------------|----------|
| Keep queue + table separate | Queue card at top (Approve/Reject), unified table below. Two surfaces coexist. | ✓ |
| Fold into unified table | One table for everything; Approve/Reject in action column for pending_internal rows. Queue section disappears. | |
| Compact queue summary + table | Small banner with count + filter shortcut, then unified table. | |

**User's choice:** Keep queue + table separate

## Queue Row Click Target

| Option | Description | Selected |
|--------|-------------|----------|
| Full row click | Clicking anywhere on row (except Approve/Reject buttons) opens modal. cursor:pointer, hover highlight. | ✓ |
| Title link only | Proposal title becomes a link; clicking elsewhere does nothing. | |

**User's choice:** Full row click

## Status Scope

| Option | Description | Selected |
|--------|-------------|----------|
| Active stages only | 3 stages: pending_internal, pending_client, for_revision. Terminal states excluded. | |
| All 5 lifecycle stages | All stages including Client Approved and Loss. Full history visible. | ✓ |
| All active + Closed tile | 3 active + 1 "Closed" combined tile for Client Approved + Loss. | |

**User's choice:** All 5 lifecycle stages (initially selected "active only", then reconsidered when clarified that STAGE_ORDER has 5 stages; also clarified that "For Proposal" is a project status, not a proposal status — no scope impact on proposals collection)
