# Phase 92.1 — Scorecard Improvements: Context

**Gathered:** 2026-05-18
**Status:** Ready for planning
**Source:** Power-mode (requirements explicit in phase name + Phase 92 context)

---

## Phase Boundary

Two targeted improvements to the scorecard strip built in Phase 92:

1. **Color-coded status cards** — five statuses get distinct color treatment:
   - Loss = Red
   - Completed = Blue
   - On-going = Yellow
   - For Mobilization = Orange
   - Client Approved = Green
   All other statuses remain monochrome (white/neutral).

2. **Scorecard visibility scope fix** — scorecard counts must respect role-based project
   visibility. Currently `renderScorecards()` reads from `allProjects` (unscoped), so an
   `operations_user` sees inflated counts from projects outside their assignment scope.
   The fix: count from the same role-scoped subset that `applyFilters()` already builds
   (`scopedProjects`), but not from `filteredProjects` (Client/Search filters must not
   affect scorecard counts — cards are navigation, not filtered stats).

---

## Decisions

### D-01 — Color treatment: left border accent + subtle background tint

Apply a `3px left border` plus a very subtle background tint for each colored status.
Override hover and active states per-status so clicking a colored card uses the status
color (not the generic blue) for the highlight.

| Status | Border | Default bg | Hover bg | Active bg |
|---|---|---|---|---|
| Loss | `#ef4444` | `#fff5f5` | `#fee2e2` | `#fecaca` |
| Completed | `#1a73e8` | `#e8f0fe` | `#d2e3fc` | `#b3cef9` |
| On-going | `#eab308` | `#fefce8` | `#fef9c3` | `#fef08a` |
| For Mobilization | `#f97316` | `#fff8f3` | `#ffedd5` | `#fed7aa` |
| Client Approved | `#16a34a` | `#f0fdf4` | `#dcfce7` | `#bbf7d0` |

CSS implementation: `[data-status="Loss"]` attribute selector on `.project-scorecard-card`.
The colored-status hover/active rules have higher specificity than the generic
`.project-scorecard-card:hover` / `.project-scorecard-card--active`, so they win
automatically without needing `!important`.

### D-02 — Scorecard scope: pass `scopedProjects` to `renderScorecards()`

Change `renderScorecards()` signature to accept `baseProjects` parameter.
In `applyFilters()`, replace `renderScorecards()` with `renderScorecards(scopedProjects)`.
The function counts from `baseProjects` instead of the module-level `allProjects`.

This is the minimal change: one parameter added, one call-site updated, no new variables.
