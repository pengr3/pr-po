# Summary: Plan 91.4-02

## What was done

The Finance tab's "Project Expenses" and "Service Expenses" aggregate mappers were updated to include RFP fees. These mappers use server-side `getAggregateFromServer` for PO/TR totals but RFP fees live on RFP docs (not PO/TR docs), so a dedicated `getDocs` fetch was added to each mapper's `Promise.all`.

**Project expense mapper (~line 3763):**
- Added `const projectCode = project.project_code || ''`
- Extended `Promise.all([posAgg, trAgg])` → `Promise.all([posAgg, trAgg, rfpSnap])`
- rfpSnap = `getDocs(query(rfps, where('project_code','==',projectCode)))` with `Promise.resolve({docs:[]})` fallback when no project_code
- Added `let rfpFeesTotal = 0; rfpSnap.docs.forEach(d => { rfpFeesTotal += getRFPFees(d.data()).feesTotal; })`
- `totalExpense` now includes `rfpFeesTotal`

**Service expense mappers (one-time and recurring, ~line 4010 and ~4075):**
- Same pattern, keyed by `service_code`
- Both blocks were identical so `replace_all: true` applied the fix to both simultaneously

## Verification

`node --check app/views/finance.js` passed. `getRFPFees` was already imported in finance.js — no import change needed. `remainingBudget: budget - totalExpense` is auto-correct since `totalExpense` now includes fees. Legacy projects/services with no project_code or no RFPs get `rfpFeesTotal = 0`.
