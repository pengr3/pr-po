# Plan 91.4-02: Tier 2 — fee inclusion in finance.js expense dashboards

## Goal
The Finance tab's "Project Expenses" and "Service Expenses" tables show `totalExpense` derived from `sum('total_amount')` on POs and TRs. RFP fees live on RFP docs — not on PO/TR docs — so the server aggregate misses them. Fix: after each project/service's PO+TR aggregate resolves, also fetch its RFPs and add fee totals.

## Scope guard
- Only `refreshProjectExpenses` and `refreshServiceExpenses` functions modified
- Table columns and HTML unchanged — only the computed `totalExpense` value changes
- `_projectExpensesCachedAt` / `_serviceExpensesCachedAt` cache invalidation: unchanged (cache is already keyed by time; stale cache is invalidated on next load)
- Recurring service expenses (`refreshRecurringServiceExpenses` if it exists) — out of scope; recurring services don't typically have RFPs
- No changes to Firestore rules or schema

## Task 1 — project expense dashboard: add RFP fee fetch per project

**File:** `app/views/finance.js`

**Location:** Inside the `projectPromises` mapper (around lines 3756–3788), after the `Promise.all([posAgg, trAgg])` resolves.

**What to do:** Add a third parallel fetch for RFPs by `project_code`, sum fees, and add to `totalExpense`.

```js
// before (around line 3763–3786)
const [posAgg, trAgg] = await Promise.all([
    getAggregateFromServer(
        query(collection(db, 'pos'), where('project_name', '==', project.project_name)),
        { totalExpense: sum('total_amount'), poCount: count() }
    ),
    getAggregateFromServer(
        query(collection(db, 'transport_requests'), where('project_name', '==', project.project_name), where('finance_status', '==', 'Approved')),
        { transportTotal: sum('total_amount'), trCount: count() }
    )
]);

const totalExpense = (posAgg.data().totalExpense || 0) + (trAgg.data().transportTotal || 0);
```

```js
// after — add rfpSnap fetch in the same Promise.all; compute rfpFeesTotal
const projectCode = project.project_code || '';
const [posAgg, trAgg, rfpSnap] = await Promise.all([
    getAggregateFromServer(
        query(collection(db, 'pos'), where('project_name', '==', project.project_name)),
        { totalExpense: sum('total_amount'), poCount: count() }
    ),
    getAggregateFromServer(
        query(collection(db, 'transport_requests'), where('project_name', '==', project.project_name), where('finance_status', '==', 'Approved')),
        { transportTotal: sum('total_amount'), trCount: count() }
    ),
    projectCode
        ? getDocs(query(collection(db, 'rfps'), where('project_code', '==', projectCode)))
        : Promise.resolve({ docs: [] })
]);

let rfpFeesTotal = 0;
rfpSnap.docs.forEach(d => { rfpFeesTotal += getRFPFees(d.data()).feesTotal; });

const totalExpense = (posAgg.data().totalExpense || 0) + (trAgg.data().transportTotal || 0) + rfpFeesTotal;
```

**Note:** `getDocs` is already imported in finance.js. `getRFPFees` is already imported. The `project.project_code` field is on the project Firestore doc. If `project_code` is empty/missing (edge case on legacy projects), the fallback `Promise.resolve({ docs: [] })` means `rfpFeesTotal = 0` — safe and correct (no RFPs can exist without a project_code).

**Verify:** Grep confirms `rfpFeesTotal` appears in the project expense mapper. `node --check app/views/finance.js` passes.

## Task 2 — service expense dashboard: add RFP fee fetch per service

**File:** `app/views/finance.js`

**Location:** Inside the `servicePromises` mapper (around lines 3999–4034), after the `Promise.all([posAgg, trSnap])` resolves.

**What to do:** Add an RFP fetch to the existing `Promise.all`.

```js
// before (around line 4004–4020)
const [posAgg, trSnap] = await Promise.all([
    getAggregateFromServer(
        query(collection(db, 'pos'), where('service_code', '==', code)),
        { totalExpense: sum('total_amount'), poCount: count() }
    ),
    getDocs(query(collection(db, 'transport_requests'), where('service_code', '==', code)))
]);

let transportTotal = 0, trCount = 0;
trSnap.forEach(d => {
    if (d.data().finance_status === 'Approved') {
        transportTotal += (d.data().total_amount || 0);
        trCount++;
    }
});

const totalExpense = (posAgg.data().totalExpense || 0) + transportTotal;
```

```js
// after — add rfpSnap to Promise.all; compute rfpFeesTotal
const [posAgg, trSnap, rfpSnap] = await Promise.all([
    getAggregateFromServer(
        query(collection(db, 'pos'), where('service_code', '==', code)),
        { totalExpense: sum('total_amount'), poCount: count() }
    ),
    getDocs(query(collection(db, 'transport_requests'), where('service_code', '==', code))),
    code
        ? getDocs(query(collection(db, 'rfps'), where('service_code', '==', code)))
        : Promise.resolve({ docs: [] })
]);

let transportTotal = 0, trCount = 0;
trSnap.forEach(d => {
    if (d.data().finance_status === 'Approved') {
        transportTotal += (d.data().total_amount || 0);
        trCount++;
    }
});

let rfpFeesTotal = 0;
rfpSnap.docs.forEach(d => { rfpFeesTotal += getRFPFees(d.data()).feesTotal; });

const totalExpense = (posAgg.data().totalExpense || 0) + transportTotal + rfpFeesTotal;
```

**Note:** `code` guard (`code ? getDocs(...) : Promise.resolve({docs:[]})`) mirrors the `projectCode` guard in Task 1 — handles edge-case services with no `service_code`.

**Verify:** Grep confirms `rfpFeesTotal` appears in service mapper. `node --check app/views/finance.js` passes. `remainingBudget: budget - totalExpense` in the return object is auto-correct since `totalExpense` now includes fees.
