# Plan 91.4-01: Tier 1 — fee inclusion in expense-modal, project-detail, service-detail

## Goal
RFP fees (transfer_fee, cash_out_fee, misc_fees) are real project costs. Every view that shows a project/service expense total or remaining-payable figure must include them. This plan fixes the three files that already hold individual RFP objects in memory — no new Firestore reads required.

## Scope guard
- `app/utils.js` — read-only (getRFPTotal/getRFPFees already exist, no edits)
- `app/views/finance.js` — untouched (covered by Plan 02)
- Collectibles (`coll.amount_requested`) — untouched (fees concept doesn't apply to collectibles)
- No changes to Firestore schema or security rules

---

## Task 1 — expense-modal.js: swap amount_requested → getRFPTotal

**File:** `app/expense-modal.js`

**What to do:**

1. Edit the utils import (line 7):
   ```js
   // before
   import { formatCurrency, downloadCSV, escapeHTML } from './utils.js';
   // after
   import { formatCurrency, downloadCSV, escapeHTML, getRFPTotal } from './utils.js';
   ```

2. Edit line 85 inside the `rfpsForPayable.forEach` loop:
   ```js
   // before
   totalRequested += parseFloat(rfp.amount_requested || 0);
   // after
   totalRequested += getRFPTotal(rfp);
   ```

**Why:** `totalRequested` is displayed in the expense modal as the "Total Requested via RFPs" figure. Using `getRFPTotal` makes it fee-inclusive; legacy RFPs (no fee fields) return the base unchanged.

**Verify:** `node --check app/expense-modal.js` passes. Grep confirms `getRFPTotal` appears in import and on line ~85.

2. In the `rfpSnap.forEach` loop (around line 929-935), replace the existing body:
   ```js

   rfpSnap.forEach(d => {
       const rfp = d.data();
       rfpTotalRequested += parseFloat(rfp.amount_requested || 0);
       rfpTotalPaid += (rfp.payment_records || [])

           .filter(r => r.status !== 'voided')
           .reduce((s, r) => s + parseFloat(r.amount || 0), 0);
   });
   ```
