    **Part B — Split-panel mobile behavior:**

    In the `selectMRF()` function (line ~937), after the existing code that calls `renderMRFDetails()` and updates `.mrf-item.selected` classes, add:

    ```javascript
    // Mobile split-panel: reveal detail card and auto-scroll to it
    if (window.innerWidth <= 768) {
        const grid = document.querySelector('.dashboard-grid');
        if (grid) {
            grid.classList.add('mrf-selected');
            // Smooth scroll to detail card (second child)
            const detailCard = grid.querySelector('.card:nth-child(2)');
            if (detailCard) {
                detailCard.scrollIntoView({ behavior: 'smooth', block: 'start' });
            }
        }
    }
    ```

    Also handle the inverse: when `createNewMRF()` is called or the MRF list is refreshed (loadMRFs), ensure `.mrf-selected` is removed from `.dashboard-grid` if on mobile so the detail panel re-hides. Find the `createNewMRF()` function and add:
    ```javascript
    // Remove mobile selection state so detail card hides again if user navigates away
    const grid = document.querySelector('.dashboard-grid');
    if (grid) grid.classList.remove('mrf-selected');
    ```

    Actually, for `createNewMRF()`, the detail card should SHOW (user is creating a new MRF and needs to fill in the form). So for `createNewMRF()`, add `mrf-selected` class instead:
    ```javascript
    const grid = document.querySelector('.dashboard-grid');
    if (grid) grid.classList.add('mrf-selected');
    ```

    Read `createNewMRF()` to confirm it calls `renderMRFDetails()` or equivalent to show the form in the detail card.
