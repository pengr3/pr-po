  <action>
    Add a new `.table-scroll-container` class to components.css in the TABLES section (after the existing `.table-responsive` class at line ~439). This class is distinct from `.table-responsive` because it includes the right-edge scroll-indicator gradient and touch enhancements needed for production use.

    Add this block after the existing `.table-responsive` rule:

    ```css
    /* Scroll-container for data tables at narrow viewports.
       Wrap any <table> in this div to enable horizontal scroll with a
       right-edge fade indicator. Plans 02 and 03 apply this class. */
    .table-scroll-container {
        overflow-x: auto;
        -webkit-overflow-scrolling: touch;
        position: relative;
    }

    /* Right-edge fade gradient — signals the table continues to the right */
    .table-scroll-container::after {
        content: '';
        position: absolute;
        top: 0;
        right: 0;
        bottom: 0;
        width: 32px;
        background: linear-gradient(to left, rgba(255,255,255,0.9) 0%, rgba(255,255,255,0) 100%);
        pointer-events: none;
        /* Only visible when the container is actually scrollable.
           JS in Plans 02/03 adds class 'is-scrollable' when needed,
           but the gradient is always present and harmless when not needed. */
    }
    ```

    Note: Do NOT touch any existing rules. Append only.
  </action>

    TWO edits in this task.

    **Edit 1 — components.css:** Inside the existing `@media (max-width: 768px)` block (around line 1251), add modal footer button stacking AFTER the existing `.pagination-controls` rule. Do not create a new media block — add inside the existing one:

    ```css
    /* Stack modal footer buttons vertically on mobile */
    .modal-footer {
        flex-direction: column-reverse;
        gap: 0.5rem;
        padding: 1rem;
    }

    .modal-footer .btn {
        width: 100%;
        justify-content: center;
    }
    ```

    `column-reverse` is intentional: the primary action (Save/Confirm) stays visually first (bottom→top stacking means it renders at top in column-reverse), matching the desktop right-to-left order.
