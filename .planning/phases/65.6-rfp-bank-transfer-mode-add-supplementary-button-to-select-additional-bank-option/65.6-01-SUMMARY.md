# Phase 65.6 Plan 01: RFP Saved Bank Account Selector Summary

**Saved bank account dropdown added to PO and TR RFP modals — deduplicates bank details from rfpsData so users can one-click fill Bank Name, Account Name, and Account Number instead of retyping each time**

## Accomplishments

- Added `getSavedBankAccounts()` helper that filters rfpsData for Bank Transfer entries and deduplicates by composite key
- Added `selectSavedBank()` to fill rfpBankName, rfpBankAccountName, rfpBankDetails from saved selection
- Added `toggleSavedBankDropdown()` that builds and shows a dropdown of unique bank accounts
- Inserted "Select Saved Bank Account" button inside rfpBankFields in both openRFPModal and openTRRFPModal
- Updated `toggleRFPBankFields()` to remove the dropdown when switching away from Bank Transfer
- Registered both new functions on window in init; cleaned up in destroy

## Task Commits

1. **Task 1: Add getSavedBankAccounts helper and saved bank selector UI to both RFP modals** - `4be365e` (feat)

## Files Created/Modified

- `app/views/procurement.js` - Added 3 new helper functions, updated both RFP modal HTML, updated init/destroy registrations

## Decisions Made

- `getSavedBankAccounts` deduplicates by composite key `bank_name|bank_account_name|bank_details` — correctly treats same bank+account combo as one entry regardless of which PO/TR it came from
- `onmousedown` used in dropdown items (not `onclick`) — prevents blur-before-click race condition where input field blur fires before the click registers (established pattern in this codebase)
- Dropdown is rebuilt on each toggle call from current rfpsData — always shows latest accounts without stale data concerns
- When payment mode switches away from Bank Transfer, dropdown is removed from DOM entirely via `dropdown.remove()` — avoids orphaned elements
