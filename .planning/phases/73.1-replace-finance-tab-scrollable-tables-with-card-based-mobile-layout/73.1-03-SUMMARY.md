# Phase 73.1 Plan 03: Payables Tab Mobile Card Layout Summary

RFP Processing and PO Payment Summary tables converted to CSS dual-mode card layout — both fc-card-list sibling divs inserted in static render() HTML, populated by existing render functions alongside their tbody counterparts.

## What Was Built

### Task 1 — RFP Processing Card Layout

- `id="rfpCardList"` fc-card-list div inserted as sibling after `.table-scroll-container` in the RFP Processing card (static render() HTML)
- `id="poSummaryCardList"` fc-card-list div inserted between PO Summary `.table-scroll-container` and `#poSummaryPagination` (pagination stays visible on mobile per Pitfall 7)
- `buildRFPCard(rfp)` helper: derives status via `deriveRFPStatus`, computes totalPaid/balance, builds PO Ref as clickable link (po_id) or plain text (tr_id) or dash, selects Record Payment vs Manage Payments based on Fully Paid status, applies `fc-overdue` class for Overdue RFPs
- `renderRFPTable()` empty-state modified to populate `rfpCardList` with `fc-empty` div before returning
- `rfpCardList.innerHTML` populated at bottom of `renderRFPTable()` alongside tbody

### Task 2 — PO Payment Summary Card Layout

- `buildPOTrancheSubCard(rfp)` helper: renders individual RFP tranche rows as `fc-sub-card` divs with RFP ID, tranche, amount/paid/balance, due_date (raw), status badge, and Record Payment/Manage Payments button
- `buildPOSummaryCard(po)` helper: renders PO row as `fc-card` with expand/collapse sub-list; uses distinct IDs `po-card-expand-{safePoId}` and `po-card-chevron-{safePoId}` to avoid collision with desktop `po-expand-*` IDs
- `togglePOCardExpand(safePoId)` function: flips `display:block/none` on div element, updates chevron and button label between "Show Tranches" / "Hide Tranches"; registered on `window` in `attachWindowFunctions`
- `renderPOSummaryTable()` empty-state modified to populate `poSummaryCardList` before returning
- `poSummaryCardList.innerHTML` populated at bottom of `renderPOSummaryTable()` from same `pageEntries` (pagination applies to both)
