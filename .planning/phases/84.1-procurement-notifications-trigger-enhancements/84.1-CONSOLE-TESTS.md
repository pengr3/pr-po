# Phase 84.1 — Browser Console Test Harness

Faster than clicking through the UI. Paste blocks into DevTools Console while the app is loaded against your **dev** Firebase project. Each block is self-contained.

> **WARNING — environment check before you start.**
> All of these blocks write real notification documents to whatever Firebase project `app/firebase.js` is currently pointing at. Run them on dev only. The first block below verifies which project you're connected to.

---

## 0. Setup — paste this first, every session

Confirms you're on dev, loads the notifications module, and exposes a `T` object with the test helpers.

```js
(async () => {
  // 0a — confirm project
  const projId = window.db?._databaseId?.projectId || window.db?.app?.options?.projectId || 'unknown';
  console.log('%c Firebase project: ' + projId, 'background:#1a73e8;color:#fff;padding:2px 6px;border-radius:3px');
  if (projId === 'clmc-procurement') {
    console.warn('⚠ You are on PRODUCTION. Notifications written here are real. Stop and switch to dev unless you know what you are doing.');
  }

  // 0b — load module
  const notif = await import('/app/notifications.js');
  const fs = await import('https://www.gstatic.com/firebasejs/10.7.1/firebase-firestore.js');
  const u = window.getCurrentUser?.();
  if (!u) { console.error('No current user — sign in first.'); return; }
  console.log('Logged in as:', u.email, '| role:', u.role, '| uid:', u.uid);

  // 0c — expose helpers under window.T
  window.T = {
    notif, fs, db: window.db, me: u,
    types: notif.NOTIFICATION_TYPES,
    // Tail my recent notifications
    async myRecent(limit = 10) {
      const q = fs.query(
        fs.collection(window.db, 'notifications'),
        fs.where('user_id', '==', u.uid),
        fs.orderBy('created_at', 'desc'),
        fs.limit(limit)
      );
      const snap = await fs.getDocs(q);
      const rows = snap.docs.map(d => ({ id: d.id, ...d.data() }));
      console.table(rows.map(r => ({ type: r.type, message: r.message, read: r.read, created: r.created_at?.toDate?.()?.toISOString?.() ?? r.created_at })));
      return rows;
    },
    // Tail recent notifications for a specific UID (test recipient)
    async recentFor(uid, limit = 10) {
      const q = fs.query(
        fs.collection(window.db, 'notifications'),
        fs.where('user_id', '==', uid),
        fs.orderBy('created_at', 'desc'),
        fs.limit(limit)
      );
      const snap = await fs.getDocs(q);
      const rows = snap.docs.map(d => ({ id: d.id, ...d.data() }));
      console.table(rows.map(r => ({ type: r.type, message: r.message, read: r.read })));
      return rows;
    },
    // List active users by role (sanity check fan-out audience)
    async usersByRole(role) {
      const q = fs.query(
        fs.collection(window.db, 'users'),
        fs.where('role', '==', role),
        fs.where('status', '==', 'active')
      );
      const snap = await fs.getDocs(q);
      const rows = snap.docs.map(d => ({ uid: d.id, full_name: d.data().full_name, email: d.data().email }));
      console.table(rows);
      return rows;
    },
    // Get a doc by collection + composite-id field (e.g. 'prs', 'pr_id', 'PR-2026-014')
    async findByBusinessId(coll, idField, idValue) {
      const q = fs.query(fs.collection(window.db, coll), fs.where(idField, '==', idValue));
      const snap = await fs.getDocs(q);
      if (snap.empty) { console.warn(`No ${coll} doc with ${idField}=${idValue}`); return null; }
      const d = snap.docs[0];
      return { docId: d.id, ...d.data() };
    },
    // Verify all 6 new Phase-84.1 enum keys are loaded (not legacy bundle)
    enumCheck() {
      const need = ['MRF_SUBMITTED','PR_DECIDED','TR_DECIDED','RFP_PAID','PO_DELIVERED','PROJECT_COST_CHANGED'];
      const missing = need.filter(k => !notif.NOTIFICATION_TYPES[k]);
      if (missing.length) console.error('MISSING enum keys:', missing, '→ hard refresh (Ctrl+Shift+R) to bust cache');
      else console.log('%c All 6 Phase 84.1 enum keys present', 'color:#059669');
      return missing.length === 0;
    }
  };
  window.T.enumCheck();
  console.log('%c Ready. Test helpers available as window.T', 'color:#059669;font-weight:bold');
})();
```

After running, you should see the project name, your role, and a green "All 6 Phase 84.1 enum keys present" line. If any key is missing, hard-refresh (Ctrl+Shift+R) and re-run — the browser cached the old `notifications.js`.

## 1. Smoke tests — does the API write notifications at all?

These bypass the trigger logic and call helpers directly. Use to verify Firebase rules + the helper code work. Each block prints the doc ID and reflects in your bell within ~1 second.

### 1a. NOTIF-14 — broadcast to procurement role

```js
(async () => {
  const recipients = await T.usersByRole('procurement');
  console.log(`Will fan out to ${recipients.length} active procurement users (excludeActor=true skips you if you're procurement)`);
  const count = await T.notif.createNotificationForRoles({
    roles: ['procurement'],
    type: T.types.MRF_SUBMITTED,
    message: 'CONSOLE-TEST: New MRF MRF-9999-001 for "Test Codeless Project" needs processing',
    link: '#/procurement/mrfs',
    source_collection: 'mrfs',
    source_id: 'MRF-9999-001',
    excludeActor: false   // include yourself for testing
  });
  console.log(`createNotificationForRoles wrote ${count} notification(s)`);
})();
```

**Expected:** `count` should equal the number of active procurement users. If count is 0:
- You're seeing the deployed firestore.rules bug — `users.list` is still admin-only because rules haven't been deployed yet. Run `firebase deploy --only firestore:rules` first.
- Or there are zero active procurement-role users in the project.

## 2. Trigger tests — does the UI handler fire the right notification?

These call the actual `window.*` handlers the UI uses, so they exercise the trigger logic, not just the helper. **Pre-requisite:** be on the right page so the handler's local state is populated.

### 2a. NOTIF-15 PR Approved — invoke `approvePRWithSignature` programmatically

**Setup:** Be on Finance → Pending Approvals tab. Find a PENDING PR with `pr_creator_user_id` already stamped (Phase 84+). Then:

```js
(async () => {
  // 1. Look up a Pending PR you want to approve
  const pr = await T.findByBusinessId('prs', 'pr_id', 'PR-2026-001'); // <- replace ID
  if (!pr) return;
  console.log('PR creator UID:', pr.pr_creator_user_id || '(MISSING — legacy doc, NOTIF-15 will skip silently)');

  // 2. Approve it (this fires the live trigger)
  // Note: this requires the UI to already have set window.currentApprovalTarget OR you can call it with an ID directly
  // If approvePRWithSignature requires a signature dialog, this won't work headlessly — use the UI button.
  // But you can short-circuit it by calling the underlying updateDoc + manually invoking createNotification:

  await T.notif.createNotification({
    user_id: pr.pr_creator_user_id,
    type: T.types.PR_DECIDED,
    message: `PR ${pr.pr_id} has been Approved by Finance`,
    link: '#/procurement/records',
    source_collection: 'prs',
    source_id: pr.pr_id
  });
  console.log('Sent PR_DECIDED notification to', pr.pr_creator_user_id);
  await T.recentFor(pr.pr_creator_user_id, 3);
})();
```

### 2b. NOTIF-18 PO Delivered — verify fan-out for material PO

```js
(async () => {
  const po = await T.findByBusinessId('pos', 'po_id', 'PO-2026-001'); // <- replace
  if (!po) return;
  if (po.is_subcon) { console.warn('Subcon PO — out of NOTIF-18 scope'); return; }
  console.log('PO creator UID:', po.po_creator_user_id || '(MISSING — legacy doc)');

  // Resolve the MRF requestor
  const mrf = await T.findByBusinessId('mrfs', 'mrf_id', po.mrf_id);

  console.log('Requestor UID:', requestorUid || '(MISSING — legacy MRF)');

  const recipients = [po.po_creator_user_id, requestorUid].filter(Boolean);
  if (recipients.length === 0) { console.warn('No recipients — both fields missing'); return; }

  const count = await T.notif.createNotificationForUsers({
    user_ids: recipients,
    type: T.types.PO_DELIVERED,
    message: `PO ${po.po_id} for MRF ${po.mrf_id} has been Delivered`,
    link: '#/procurement/records',
    source_collection: 'pos',
    source_id: po.po_id,
    excludeActor: true
  });
  console.log(`Sent PO_DELIVERED to ${count} recipient(s)`);
})();
```

## 5. End-to-end UAT shortcut — fastest path

If you only have time for one pass, run this sequence on dev:

1. Block 0 (setup) — must succeed.
2. Block 1a (NOTIF-14 fan-out) — confirms rules are deployed.
3. Block 1b–1g — confirms each enum type writes a notification.
4. Open the bell — verify all 6 new types render with their icon/color (no fallback "?" badges).
5. Pick ONE flow per page-bound trigger and run it through the UI:
   - Submit an MRF as services_user → procurement bell pings (NOTIF-14)
   - Approve a PR as Finance → procurement creator's bell pings (NOTIF-15)
   - Reject a PR with reason → creator's bell shows reason text (NOTIF-15)
   - Record full-amount payment on an RFP → creator's bell pings (NOTIF-16)
   - Approve/reject a TR as Finance → creator's bell pings (NOTIF-17)
   - Mark a material PO Delivered → requestor + PO creator both bell-ping (NOTIF-18)
   - Edit a project's Budget → personnel bell-pings (NOTIF-19)
   - Reject an MRF with reason → bell shows reason (NOTIF-20)

Total time: ~15 minutes vs ~1 hour of pure UI clicking.

## Known gotchas

| Symptom | Cause | Fix |
|---|---|---|
| Bell never updates | You're on prod (or wrong project) | Verify `T.notif` setup printout, switch dev |
| `count: 0` from `createNotificationForRoles` for procurement | `firestore.rules` `users.list` not deployed yet | `firebase deploy --only firestore:rules` |
| Notification says `(?)` icon | Browser cached old `notifications.js` | Hard refresh (Ctrl+Shift+R) |
| NOTIF-15/16/17 "didn't fire" on real PR/TR/RFP | Legacy doc — `*_creator_user_id` is missing | See §3c — backfill or use fresh doc |
| `T` is undefined | You haven't run Block 0 yet (or hard-refreshed since) | Re-run Block 0 |
| NOTIF-14 with `excludeActor:true` got 0 recipients | You ARE the only procurement user, you got self-excluded | Test as a different role, or set `excludeActor:false` for the smoke test |
