## Decisions Made
- **Used `reason` local variable instead of `rejectedMrfSnap.rejection_reason` for the colon-suffix:** Both are equivalent post-updateDoc (the snapshot was captured before the updateDoc and updateDoc wrote `rejection_reason: reason.trim()`, so `rejectedMrfSnap.rejection_reason` is undefined at the read point — `reason` is the canonical source). Plan-supplied snippet uses `reason` directly; followed verbatim.
- **Defensive conditional `reason && reason.trim() ? ... : ''` (not just `reason ? ... : ''`):** The conditional checks both truthiness and non-empty trimmed content. In practice rejectMRF() already pre-validates `reason.trim()` is non-empty (lines 4226-4229) and returns early if not, so this is belt-and-suspenders defense. The conditional handles only theoretical bypass paths (programmatic invocation without prompt validation).
- **UAT scaffold as a separate `test(84.1)` commit:** Code change is one-line atomic; UAT scaffold is a 123-line test plan. Splitting the commits keeps each artifact reviewable independently and matches the parent's suggested commit pattern in the plan brief. UAT execution itself is the human checkpoint task — the scaffold is just preparation.
- **Pre-populated 8 regression tests in the UAT scaffold (not just the 7 new requirements):** The plan explicitly requires running Phase 84 UAT Tests 1-8 against the post-Phase-84.1 build to confirm no regression. Pre-populating them in the same UAT report keeps all 15 tests visible at once for the human tester. Alternatively the plan allows running 84-UAT.md and 84.1-UAT.md as separate sessions; chose unified for consolidation.
