<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| new user (unauthenticated→authenticated)→Firestore notifications | Newly registered user (pending, not yet active) writes to notifications collection during registration flow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84-07 | Spoofing | NOTIF-12 actor_id on notification | accept | actor_id is the new user's Firebase Auth UID — they are authenticated at fire time. Phase 83 Security Rule enforces actor_id == request.auth.uid. Cannot be spoofed. |
| T-84-08 | Denial of Service | REGISTRATION_PENDING fan-out to super_admins | accept | Super_admin count is very small (1-3 users). No quota risk. Notification write is fire-and-forget — a failed write does not block registration. |
| T-84-09 | Information Disclosure | Notification message contains fullName + email | accept | Only super_admins receive this — they already have access to user management and need this information to approve the account. |
</threat_model>
