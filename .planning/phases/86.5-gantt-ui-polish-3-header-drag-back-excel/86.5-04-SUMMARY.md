# Phase 86.5 Plan 04: Bidirectional Bar Resize — Leftward Reduces, Rightward Extends Summary

The empty-space mouseup branch added by Plan 86.5-03 in `initGanttDragLink()` is upgraded from extend-only to a symmetric resize: a signed `daysDelta` (positive = rightward, negative = leftward) is applied to `end_date`, clamped so it never falls below `start_date`, then written via the same `updateDoc(project_tasks)` + `recomputeParentDates` path Plan 03 established.

## What Changed

In `app/views/project-plan.js`, inside the `svg.addEventListener('mouseup', ...)` handler in `initGanttDragLink()`, the body of the `if (!toWrapper) { ... }` block (introduced by Plan 86.5-03) was replaced. Behaviorally:

1. **Same setup as Plan 03** — resolve `fromTask`, locate `.bar-wrapper[data-id]` and `.bar`, compute `barRightX = bbox.x + bbox.width`, transform `e.clientX/Y` into SVG space, and read `xPerDay2` from `gantt.config.column_width / gantt.config.step`.
2. **Removed leftward-cancel guard** — the line `if (svgPt2.x <= barRightX) return; // dragged leftward — cancel silently` is gone. Both directions are now in-band.
3. **Signed delta** — `const daysDelta = Math.round((svgPt2.x - barRightX) / xPerDay2);` (no `Math.max(1, ...)` floor; the sign is meaningful).
4. **Zero-delta short-circuit** — `if (daysDelta === 0) return; // no meaningful change` — drops within a sub-day pixel window of the right edge become no-ops.
5. **Apply + clamp** — `let newEndDate = addDays(fromTask.end_date, daysDelta);` — `addDays` uses `setDate(getDate() + n)` (line 977) so it accepts negative `n` correctly. Then the clamp: `if (newEndDate < fromTask.start_date) newEndDate = fromTask.start_date;` — string compare on ISO `YYYY-MM-DD` is lexicographically chronological, so this is a real chronological clamp without parsing.
6. **Idempotency guard** — `if (newEndDate === fromTask.end_date) return; // clamp produced no change` — when the clamp pins the result back to the existing `end_date`, skip the write entirely.
7. **Strengthened guard** — the early exit added `!fromTask.start_date` alongside the existing `!fromTask.end_date` because the clamp dereferences `start_date`.
8. **Same write path** — `updateDoc(doc(db, 'project_tasks', fromTaskId), { end_date: newEndDate, updated_at: serverTimestamp() })`, local `tasks[]` patch, and `recomputeParentDates(fromTask.parent_task_id)` — unchanged from Plan 03.
9. **Generalized error surface** — `console.error('[ProjectPlan] bar resize failed:', err)` and `showToast('Could not adjust task duration.', 'error')` (was "bar extend failed" / "Could not extend task duration").

The bar-to-bar branch (toWrapper resolved → predecessor link + cycle check + `applyFsAutoSchedule`) is untouched, so Phase 86.1 link behavior is preserved.
