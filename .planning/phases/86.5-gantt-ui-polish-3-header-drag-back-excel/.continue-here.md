<decisions_made>

**Plan 01 requires TWO CSS changes, not one:**
- `renderCustomGanttHeader()` is STILL ACTIVE in project-plan.js (called at lines ~1122 and ~1138)
- It creates a `position: absolute; top: 8px` div (`#ganttHeaderOverlay`) appended inside `#ganttPane`
- When `#ganttPane` padding-top is removed (8px → 0), the overlay's `top: 8px` must also become `top: 0`
- Without both changes: overlay floats 8px below SVG origin, making the header misalignment WORSE
- Plan 01 originally said "do NOT change `top: 8px`" — this was wrong and has been corrected

**Bar extend formula:**
- `xPerDay2 = (gantt.config.column_width || 45) / (gantt.config.step || 1)`
- Matches Frappe's own pixel-per-day mapping; correct for all zoom modes (Day/Week/Month)
