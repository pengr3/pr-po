# Phase 86.5 Plan 03: Unified Bar Drag — Empty-Space Extends Duration Summary

One-liner: `if (!toWrapper) return` in the Gantt drag-link mouseup handler is replaced with an SVG-coord-based rightward-drop branch that calls `updateDoc(project_tasks.end_date)` plus parent-rollup, while leftward drops keep the silent-cancel UX.

## What Changed

In `app/views/project-plan.js`, inside `initGanttDragLink()` at the `svg.addEventListener('mouseup', ...)` handler, the single line that previously read

```js
if (!toWrapper) return; // dropped on empty space — cancel silently
```

now expands into a guarded `if (!toWrapper) { ... return; }` block that:

1. Resolves the dragged task and its bar (`fromTaskId` → `tasks` → `.bar-wrapper[data-id]` → `.bar`).
2. Computes the bar's right edge in SVG coords via `getBBox()`.
3. Transforms the mouseup client-coords (`e.clientX/Y`) into the same SVG space.
4. If the drop point is at-or-left-of the right edge → silent cancel (preserves prior behavior).
5. Otherwise computes pixels-per-day as `(gantt.config.column_width || 45) / (gantt.config.step || 1)` and rounds the rightward overhang into `daysToAdd = Math.max(1, Math.round((svgPt2.x - barRightX) / xPerDay2))`.
6. Calls `addDays(fromTask.end_date, daysToAdd)` and writes `{ end_date, updated_at: serverTimestamp() }` to `project_tasks/{fromTaskId}`.
7. Patches the local `tasks[]` cache so derived UI doesn't lag the snapshot, and triggers `recomputeParentDates(fromTask.parent_task_id)` when the dragged task is nested.
8. Catches Firestore errors with `console.error('[ProjectPlan] bar extend failed:', err)` and a `showToast('Could not extend task duration.', 'error')`.

The fall-through remainder of the handler (toWrapper-bound branch — FS predecessor link via `dependencies` + cycle check + `applyFsAutoSchedule`) is untouched, so existing Phase 86.1 link behavior is preserved.
