## Critical Anti-Pattern (CR-01) — Closed

The prior reverted 87.1 attempt failed because `renderProposalPillsFor` emitted suffixed window-function names (`proposalSelectPersonnelIn`, `proposalRemovePersonnelFrom`) while `engagements.js` registered the unsuffixed variants. The revert deleted that suffixed helper entirely. The current implementation in `app/engagement-create.js` (Plan 87.1-03) uses unsuffixed names consistently across emitter, registrar (`initEngagementForm`), and deleter (`destroyEngagementForm`). UAT steps 7–12 exercised the picker end-to-end with the DevTools console open and observed no errors of the CR-01 class.

This anti-pattern is now considered structurally closed: there is no suffixed pill helper remaining in the codebase that could reintroduce the mismatch.
