## Gaps Summary

One gap blocking full goal achievement:

**Personnel pill interaction broken in Engagements tab (CR-01, originally identified in 87.1-REVIEW.md)**

The `engagements.js` init() registers personnel pill window functions under the old unsuffixed names (from the original private helpers in proposals.js), but the imported `renderProposalPillsFor` wrapper from proposals.js generates HTML that calls the suffixed names. The four suffixed names (`window.proposalSelectPersonnelIn`, `window.proposalRemovePersonnelFrom`, `window.proposalShowPersonnelDropdownFor`, `window.proposalFilterPersonnelDropdownFor`) are never registered anywhere. Every personnel picker interaction — selecting a user, removing a pill, typing to filter — silently fails.

The fix is mechanical: rename the four window function registrations in engagements.js init() from the unsuffixed to the suffixed forms, and update the four delete calls in destroy() to match.

The REVIEW (87.1-REVIEW.md, commit 498bc2a) identified this as CR-01 critical. No fix commit was applied after the review.

All other phase deliverables are fully implemented and structurally correct.
