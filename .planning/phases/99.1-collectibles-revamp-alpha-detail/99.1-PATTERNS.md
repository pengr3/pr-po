## Stage-Derivation Logic to Reuse (D-07 ↔ D-13 — the single source of truth)

**The canonical cash/status formula is `deriveCollectibleStatus` — finance.js:110, byte-for-byte duplicated at expense-modal.js:24:**
```javascript
function deriveCollectibleStatus(coll) {
    const totalPaid = (coll.payment_records || [])
        .filter(r => r.status !== 'voided')                                  // D-03 voided exclusion
        .reduce((s, r) => s + (parseFloat(r.amount) || 0), 0);
    const isOverdue = coll.due_date && new Date(coll.due_date) < new Date();
    if (totalPaid >= coll.amount_requested && coll.amount_requested > 0) return 'Fully Paid';   // 100%
    if (isOverdue) return 'Overdue';
    if (totalPaid > 0) return 'Partially Paid';                             // 0 < pct < 100
    return 'Pending';                                                        // pct = 0
}
```
*(finance.js:106-108 even notes the expense-modal.js duplication is intentional, flagged for a future hygiene phase. DO NOT add a third divergent copy.)*

**The D-07 lifecycle stage is `deriveCollectibleStatus` cross-referenced with the billing_request status.** Build the new `computeTrancheLifecycle(...)` so it computes `totalPaid` and `pct` with the EXACT formula above (voided-excluded, `amount_requested` as denominator), then layers the billing dimension:

| Inputs | D-07 stage | Badge |
|--------|-----------|-------|
| no billing_request for tranche | **Not Filed** | gray, 45% opacity, dashed "— Not Filed" (D-08) |
| billing status `pending` | **Pending** | amber `#f59e0b` "Pending Review" |
| billing status `rejected` | **Rejected** | red `#ef4444` |
| `approved` + no matching collectible | **Approved — Not Yet Invoiced** | indigo (the currently-invisible state; also D-19 ghost row in modal) |
| `approved` + collectible + pct = 0 | **Invoiced — Awaiting Payment** | teal |
| `approved` + collectible + 0 < pct < 100 | **Billed / Collecting** | progress bar + "₱X of ₱Y · Z%" note (D-09) |
| `approved` + collectible + pct = 100 | **Fully Collected ✓** | green `#059669` |

**Do NOT** re-derive cash % with a different denominator or a different voided rule (D-13). `pct = round(totalPaid / amount_requested * 100)` using the collectible doc's frozen `amount_requested` (D-02).
