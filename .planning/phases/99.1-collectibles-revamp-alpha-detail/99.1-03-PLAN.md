<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Firestore (billing_requests write via service submit) | operations user submits an advisory amount + doc-link URLs + free-text notes for a service |
| client → Firestore (billing_requests/collectibles read) | listener renders tranche_label, rejection_reason, notes, doc labels into the card/modal |
| client → notification fan-out | service submit notifies Finance role |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99.1-06 | Tampering | service submitBillingRequest amount_requested | accept | Advisory; Finance re-derives the authoritative amount at approval (D-01). department:'services' is a non-privileged discriminator. |
| T-99.1-07 | Information disclosure / XSS | billing modal + lifecycle footer (tranche_label, rejection_reason, notes, doc labels) | mitigate | escapeHTML() on every interpolated user string (ported verbatim from project side); doc inputs `type="url"`; rendered doc anchors `target="_blank" rel="noopener noreferrer"`. |
| T-99.1-08 | Elevation of privilege | unauthorized status transition on service billing_requests | mitigate | firestore.rules billing_requests update = finance roles only (dept-agnostic, firestore.rules:549 — unchanged, D-27). |
| T-99.1-09 | DoS / availability | a notification failure aborting the submit | mitigate | Finance fan-out is fire-and-forget in its OWN try/catch after the addDoc commits (D-24). |
</threat_model>
