# Phase 99.1 Plan 02: Expense Modal Money-In Revamp Summary

**5-chip Money-In header strip + 3-chip Collectibles summary with a Billed-vs-Invoiced formula note + "awaiting COLL-xxx" ghost rows, backed by the modal's first-ever approved-billing_requests fetch (mode-aware).**

## Performance
- **Duration:** ~18 min
- **Tasks:** 2 (committed together — shared fetch dependency)
- **Files modified:** 1 (`app/expense-modal.js`)

## Accomplishments
- Added a mode-aware approved-`billing_requests` fetch into `billingReqsForModal` (service mode: `where('service_code','==',identifier)`; project mode reuses the already-resolved `projectCodeForColl` — no extra projects fetch).
- Computed `billedToClient` (approved-only sum) + `ghosts` (approved billing_requests with no collectible at the same `tranche_index`) once, right after the existing coll totals.
- Added the 5-chip **Money-In** header strip after the Row-3 block: Expense / Rem. Payable / Billed to Client / Invoiced (COLLs) / Cash Received.
- Added the Collectibles-tab 3-chip summary (Total Billed / Cash Collected / Outstanding) + a formula note spelling out Billed vs Invoiced vs Outstanding; rebuilt `collectiblesHTML = collMoneyInSummary + collectiblesTableHTML`.
- Added muted `awaiting COLL-xxx` ghost rows inside the existing `modal-items-table`; the empty-tab guard now considers ghosts (`collectiblesForTab.length > 0 || ghosts.length > 0`).
- All cash sums reuse the canonical non-voided `payment_records[].amount` formula (D-13). All ghost-row interpolations escaped (`escapeHTML(br.tranche_label …)`).

## Chip markup reused by Plan 03 (service mode)
The modal is shared — Plan 03's `window.openServiceFullBreakdown` calls `showExpenseBreakdownModal(service_code, { mode:'service', displayName, budget })`, which now renders the same 5-chip strip + 3-chip summary + ghost rows for services automatically. No service-specific modal code is needed; service `billing_requests` (Plan 03) flow straight into `billedToClient`/`ghosts` via the `service_code` branch.
