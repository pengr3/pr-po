# Phase 99.1 Plan 04: Finance Generalization Summary

**Finance pending banner, approve bridge, and decision notification are now department-aware — service billing requests are reviewable/approvable end-to-end while missing-department docs still behave as projects; notifications.js confirmed needing no change.**

## Performance
- **Duration:** ~12 min (code) + browser UAT pending
- **Tasks:** 2 automated done + 1 blocking browser-UAT (pending-in-browser)
- **Files modified:** 1 (`app/views/finance.js`); `app/notifications.js` confirmed unchanged

## Accomplishments
- **Task 1 (finance.js):**
  - `renderPendingBillingBanner()` now shows `req.project_name || req.service_name` + a `Service`/`Project` dept chip (escaping preserved).
  - `approveBillingRequest()` builds the `preselectKey` from `req.department` — `'services:' + service_code + ':' + tranche_index` for services, else `'projects:' + project_code + ':' + tranche_index`. The hardcoded `'projects:'`-only line + its comment are gone. `openCreateCollectibleModal`'s parser is untouched (already supports the `services:` dept + 3rd tranche segment).
  - `_notifyBillingDecision()` is dept-aware: services get `link '#/services/detail/' + service_code`, `name = service_name`, `source_id = service_code`; projects keep the existing `#/projects/detail/` behavior. `rejectBillingRequest` needed no change (link logic lives in the helper).
- **Task 2 (notifications.js — CONFIRMATION ONLY, no code change):**
  - Verified `handleNotificationClick` honors the per-notification link first: `const link = cached?.link || TYPE_META[cached?.type]?.target_route || '';` (notifications.js:448). Because finance.js now passes `'#/services/detail/'+service_code` for service decisions, the submitter lands on the service detail. **D-26 satisfied with NO notifications.js change.**
  - `TYPE_META.BILLING_REQUEST_DECIDED.target_route` left as `#/projects` — it is only the fallback for legacy/linkless docs; changing it would mis-route project decisions.
- **D-22 back-compat:** missing `department` treated as `'projects'` in all three sites.
- **D-27:** `firestore.rules` untouched (verified: 0 rules changes across the phase).
