<decisions>
## Implementation Decisions

### D-01: Icon set — Heroicons outline, inline SVG (Spike 003a WINNER)
- All 16 types in `TYPE_META` get a unique Heroicons outline SVG (24×24 viewBox, stroke-width 1.75, round linecap+linejoin)
- Icon stored as the `icon` field value: a self-contained SVG string via `svg(path)` helper
- `svg()` helper: `` `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.75" stroke-linecap="round" stroke-linejoin="round">${path}</svg>` ``
- The badge wrapper switches from `<span class="badge-text">` to `<span class="badge-svg">` (or the existing class must be updated to remove `font-size/font-weight` and add `svg` sizing rules)
- Zero changes to `index.html` (003a wins over 003b for exactly this reason)
- Exact SVG paths for all 16 types are in `.planning/spikes/003-notification-polish/spike.html` (`TYPES_SVG_INLINE` const) — use those paths verbatim

### D-02: Dropdown animation — translateY slide (Spike 004a WINNER)
- Dropdown starts hidden: `opacity:0; transform:translateY(-12px); visibility:hidden; pointer-events:none`
- Open state (`.open` class): `opacity:1; transform:translateY(0); visibility:visible; pointer-events:auto`
- Transition: `transform 200ms ease-out, opacity 200ms ease-out, visibility 0ms 0ms` (open) / `visibility 0ms 200ms` (close — deferred so element hides after fade completes)
- Bell click toggles `.open` class on the dropdown element — no inline style manipulation
- Close on outside-click: document click listener (same as current pattern if one exists; add one if not)
- Duration: 200ms (confirmed via spike 004 controls)
- Easing: ease-out (confirmed via spike 004 controls)
- translateY offset: -12px (confirmed via spike 004 controls)

### D-03: Unread row indicator — left-border + pale tint (Spike 005 Variant B WINNER)
- Unread row: `background: #f8fbff; border-left: 3px solid #1a73e8; padding-left: calc(1rem - 3px)`
- All rows (read + unread): `border-left: 3px solid transparent` — alignment anchor so content never shifts
- Unread hover: `background: #eff6ff`
- Read hover: `background: #f8fafc` (unchanged)
- Remove the old `.nd-row.unread { background: #eff6ff }` rule entirely
- Apply to both dropdown rows AND history page rows (same CSS class structure)

### D-04: Badge CSS — svg sizing rules
- Existing `.notification-type-badge` (or equivalent class) must size the SVG child: `width:16px; height:16px; stroke:currentColor; fill:none; stroke-width:1.75; stroke-linecap:round; stroke-linejoin:round`
- Check existing CSS class names in `app/notifications.js` renderDropdownRows template literal before choosing selector
- Badge background/color styling (`background: ${meta.color}18; color: ${meta.color}`) is already in the JS template and does not change

### D-05: History page parity
- `app/views/notifications.js` history page rows use the same unread indicator CSS (D-03)
- History page does NOT get the dropdown animation (it's a full page, not a dropdown)
- History page row structure may differ from dropdown rows — read the file before assuming shared CSS applies

### Claude's Discretion
- Whether to add a `svg()` helper at module scope or inline the full SVG string per entry (module-scope helper preferred — less repetition)
- Whether to combine dropdown animation CSS with existing `.notif-dropdown` rules or add new rules (combine where possible)
- Whether history page uses `.nd-row` class already or has its own class name (read first)

</decisions>
