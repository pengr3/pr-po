    The history page renderRows() already applies class="notif-row notif-row--unread" on unread rows. However the row div has a competing inline style: `style="display:flex;align-items:flex-start;gap:0.75rem;padding:0.875rem 1rem;border-bottom:1px solid #e5e7eb;cursor:pointer;"`.

    Fix: Remove the inline `padding` property from the row div's style attribute and let CSS own padding. The .notif-row CSS rule in components.css already provides: padding: 0.75rem 1rem and border-bottom: 1px solid #f1f5f9. The history page rows currently use a slightly different padding (0.875rem top/bottom vs 0.75rem in dropdown). Accept the slight visual difference — the CSS rule value is correct for parity. Do not add a history-page-specific padding override.

    Specifically: in the renderRows() template literal, change the row div opening tag from:
      `<div class="notif-row ${isUnread ? 'notif-row--unread' : ''}"
           style="display:flex;align-items:flex-start;gap:0.75rem;padding:0.875rem 1rem;border-bottom:1px solid #e5e7eb;cursor:pointer;"
           onclick="window.handleNotificationClick('${docId}')">`

    To (remove the padding and border-bottom inline properties, keep display/align/gap/cursor):
      `<div class="notif-row ${isUnread ? 'notif-row--unread' : ''}"
           style="display:flex;align-items:flex-start;gap:0.75rem;cursor:pointer;"
           onclick="window.handleNotificationClick('${docId}')">`

    Rationale: padding and border-bottom are now owned by .notif-row CSS; removing inline overrides lets CSS padding-left: calc(1rem - 3px) work on unread rows. The .notif-row CSS rule has border-bottom: 1px solid #f1f5f9 — slightly lighter than the history page's old #e5e7eb; this is an acceptable minor visual delta.
