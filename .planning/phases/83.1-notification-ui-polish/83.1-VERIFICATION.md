### Anti-Patterns and Known Gaps (Post-Fix)

#### WR-01: Missing CSS rules for dropdown structural classes (Warning, not Blocker)

The code review flagged WR-01: `.notif-row-body`, `.notif-row-content`, and `.notif-row-read-btn` are used as class names in `app/notifications.js` `renderDropdownRows()` (lines 197, 203, 208) but have no corresponding CSS rules in `styles/components.css`.

**Assessment:** These classes carry no visual styling in the current design — all their layout is handled by inline styles on parent elements and by `.notif-row` flex layout. The current behavior is correct because:
- `.notif-row-body` wraps the clickable content area; its `onclick` and `title` are in the HTML; layout inherits from flex parent `.notif-row`
- `.notif-row-content` is the message+time column; it gets no explicit width because flexbox allocates remaining space
- `.notif-row-read-btn` has its own CSS at lines 1988–2001 under `.notif-row-mark-read` — however the dropdown uses `notif-row-read-btn` (not `notif-row-mark-read`); this means the mark-as-read button in the dropdown does NOT get the CSS hover style defined for `.notif-row-mark-read`

**Impact on must-have truths:** None of the 11 must-have truths cover dropdown button hover styling. The button is rendered and functional (uses inline styles in the history page, fallback browser defaults in the dropdown). This is a cosmetic gap only — the mark-read button in the dropdown has no hover effect from CSS.

**Classification:** Warning — does not block any plan must-have truth; no blocker anti-patterns found.

No `TBD`, `FIXME`, or `XXX` markers in any file modified by this phase.

### 2. Dropdown Animation Visual Check

**Test:** Click bell to open; click bell again to close; click bell then click outside.
**Expected:** Open: dropdown slides in from above over 200ms ease-out (translateY -12px to 0, opacity 0 to 1). Close (bell or outside click): dropdown fades out over 200ms, then becomes non-interactive/hidden. No abrupt pop-in or pop-out.

### 3. History Page Icon Container Shape Check

**Test:** Navigate to `#/notifications`. Observe the icon area on each notification row.
**Expected:** Icons appear inside square containers (border-radius 6px colored background), matching the dropdown style — NOT circles. Each icon is a small SVG shape (not a text char like ✓ or !).
**Why human:** The b9bf2d2 fix switched from `.notif-row-icon` circle to `.notif-type-badge` square; visual confirmation required to ensure the CSS border-radius:6px applies correctly (not overridden by inline style).

### 4. Unread Left-Border Alignment Check

**Test:** Navigate to `#/notifications`. With both read and unread notifications visible, compare the horizontal start position of content (icon left edge) between a read row and an unread row.
**Expected:** Content starts at the same horizontal position — the 3px blue left border on unread rows does not shift content rightward because `.notif-row` has `border-left: 3px solid transparent` as an alignment anchor, and `.notif-row--unread` compensates with `padding-left: calc(1rem - 3px)`.
**Why human:** Pixel-level alignment verification requires visual inspection in a browser.

### 5. Unread Hover Color Check

**Test:** Hover over an unread row, then hover over a read row on either the dropdown or history page.
**Expected:** Unread hover: background deepens to `#eff6ff`. Read row hover: `#f8fafc` (very light grey, barely visible). The colors are visually distinct from each other.
**Why human:** The two hover background colors (#eff6ff vs #f8fafc) are perceptually similar and require direct visual comparison.
