### Post-Mortem Flag — At-Risk Status (future phase)

During UAT, identified a gap in the 4-state status model (not-started / in-progress / overdue / complete): no warning state before a task becomes overdue.

**Proposed: computed `at-risk` state (Option A)**
- Trigger: `time_elapsed % > (progress / 100) + 0.2` — task is 20%+ behind pace
- Formula: `time_elapsed = (today - start_date) / (end_date - start_date)`
- Color: orange `#f97316` (distinct from milestone amber `#f59e0b`)
- Applies to: grid row tint, Gantt bar fill — same infrastructure as existing status classes
- No new Firestore fields required — computed entirely from `start_date`, `end_date`, `progress`, `today`
- Implementation: extend `computeStatus()` with one new branch between `in-progress` and `overdue` checks
