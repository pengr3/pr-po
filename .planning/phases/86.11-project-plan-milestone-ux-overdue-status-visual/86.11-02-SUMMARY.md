# Phase 86.11 Plan 02: Milestone Row Tinting + Context Menu Toggle Summary

**Amber milestone toggle in right-click context menu writes is_milestone to Firestore; milestone rows render with #fffbeb tint, 3px amber left border, and ◆ CSS prefix that overrides Plan 01 status tints.**

## Accomplishments
- `gridToggleMilestone(taskId)` async function added at module scope — toggles `is_milestone` via Firestore `updateDoc` with optimistic local patch
- Amber "Mark as Milestone" / "Remove Milestone" menu item added to single-task context menu branch (with divider, #f59e0b color, #fffbeb hover)
- `window.gridToggleMilestone` registered in `init()` and deleted in `destroy()` — clean lifecycle
- `milestoneClass` appended after `statusClass` in `visibleSorted.map()` so both classes coexist on overdue milestones
- CSS: `.tg-row.tg-row-milestone` amber tint + box-shadow; hover rule; `.tg-name::before` ◆ symbol in amber

## Decisions Made
- Cascade order (milestone rules appear after status rules in CSS) is sufficient to override status tints when both classes are present — no `!important` or higher specificity needed
- `::before` on `<td class="tg-name">` rather than on the input itself — pseudo-elements on replaced elements are not reliable; td is a block container
- Group menu branch left unchanged — milestone toggle is single-task operation only
