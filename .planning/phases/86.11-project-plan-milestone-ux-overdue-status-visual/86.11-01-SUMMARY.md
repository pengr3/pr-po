# Phase 86.11 Plan 01: Row Status Tinting — computeStatus() + CSS Summary

Introduced the `computeStatus()` pure helper and wired it into the task-grid row renderer so leaf task rows are visually distinguished by status (overdue, complete, not-started) via three new CSS classes.

## What Was Built

**computeStatus(task, today)** — module-level pure function in `app/views/project-plan.js`:
- Returns `'complete'` if `task.progress >= 100`
- Returns `'overdue'` if `task.end_date < today` (ISO string comparison, lexicographically correct)
- Returns `'not-started'` if `task.start_date > today && task.progress === 0`
- Returns `'in-progress'` otherwise

**Wiring in renderTaskGrid()**:
- `today` computed once via `new Date().toISOString().slice(0, 10)` before `visibleSorted.map()`
- `statusClass` computed per row: empty string for parent rows, `' tg-row-' + computeStatus(t, today)` for leaf rows
- `statusClass` appended to `<tr class="tg-row${contextClass}${statusClass}">` template

**CSS in styles/views.css** (inserted after `.tg-row-context:hover`, before `.gantt-pane`):
- `.tg-row.tg-row-overdue`: background `#fff5f5`, inset 3px `#ef4444` left accent
- `.tg-row.tg-row-overdue:hover`: background `#fee2e2`
- `.tg-row.tg-row-complete`: background `#f0fdf4`, inset 3px `#059669` left accent
- `.tg-row.tg-row-complete:hover`: background `#dcfce7`
- `.tg-row.tg-row-not-started`: background `#f8fafc` (no border per spec)
- `.tg-row.tg-row-not-started:hover`: background `#f1f5f9`
