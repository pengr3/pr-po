## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application code → createNotification* | Call sites supply object_name and actor_name strings sourced from Firestore docs or getCurrentUser() — these are data values that will be HTML-escaped at render time (Plan 95-01 Task 2) |
| getCurrentUser() → actor_name | User's full_name from the auth system — treated as display data only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-02-01 | Information Disclosure | actor_name stores getCurrentUser().full_name in Firestore notification docs | accept | Notification docs are already user-scoped (where user_id == uid); full_name is non-sensitive display data already visible in the app; existing Firestore rules govern read access |
| T-95-02-02 | Tampering | object_name sourced from Firestore doc fields (e.g. project_name, supplier_name) | accept | These fields are written by other operations under their own Firestore rules; no additional injection surface created here — escapeHTML at render time (Plan 95-01) is the mitigation |
| T-95-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — pure JS edits to existing view files |
</threat_model>
