|----------|-------------|
| Firestore → renderDropdownRows | n.object_name, n.actor_name, n.source_id are user-supplied strings from Firestore docs — must be escaped before HTML injection |
| createNotification* callers → Firestore | object_name and actor_name supplied by callers — treated as data, not evaluated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-01 | Tampering | renderDropdownRows — n.object_name / n.actor_name HTML injection | mitigate | Wrap n.object_name, n.actor_name, n.source_id in escapeHTML() per line 159 rule; escapeHTML already imported and used for existing fields |
| T-95-02 | Information Disclosure | TYPE_META target_route | accept | Routes are static compile-time strings pointing to app-internal hash routes — no user data, no secrets |
| T-95-03 | Tampering | createNotification Firestore write — object_name / actor_name fields | accept | Firestore security rules already govern who can write to notifications collection; these are display-only fields, no privilege escalation path |
| T-95-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — pure JS edits to existing files |
</threat_model>
