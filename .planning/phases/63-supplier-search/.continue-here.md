<decisions_made>

- RFP IDs: `RFP-[PROJECT CODE]-###` format, scoped per project code (e.g. `RFP-CLMC-001`) — sequence resets per project, not year-based; custom inline generator needed (not generateSequentialId())
- Supplier search: pure client-side, filter in-memory `suppliersData` array, maintain `filteredSuppliersData` for pagination — no Firestore changes
- Proof URL: optional `proof_url` string field on `pos` documents; any `https://` URL accepted; Firebase Storage rejected (cost); Google Drive Picker deferred to v4.0+
- RFP/payment data: dedicated `rfps` Firestore collection (not embedded on POs); payment status auto-derived from arithmetic, Finance never sets it manually
- payment_records: array on RFP document (recommended for simplicity at 1-5 records/RFP, vs subcollection)

</decisions_made>
