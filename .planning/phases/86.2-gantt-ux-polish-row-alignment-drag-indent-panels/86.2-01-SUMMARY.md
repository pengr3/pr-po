# Phase 86.2 Plan 01: Row Alignment and Gantt Full-Width Fix Summary

CSS-only fix: `.tg-row height: 42px` syncs left-rail rows with Frappe Gantt SVG bars; `.gantt-pane min-width: 0` stops Gantt from overflowing its CSS grid column.

## What Was Built

Four targeted CSS edits to `styles/views.css` address Defects 1 and 9 from the Phase 86.2 UAT gap list:

**Defect 1 — Row misalignment (left-rail vs Gantt SVG bars):**
- `.tg-row { height: 42px; box-sizing: border-box; }` — locks each grid row to exactly the Frappe row pitch (bar_height 24 + padding 18 = 42px)
- `.task-grid thead tr { height: 85px; }` — aligns the grid header with Frappe's 85px header region (upper_header 45 + lower_header 30 + 10)
- `.task-grid thead th { vertical-align: bottom; padding-bottom: 8px; }` — pins column labels to the bottom of the taller header, matching Frappe's label position
- `.task-grid tbody td { padding-top: 0; padding-bottom: 0; box-sizing: border-box; vertical-align: middle; }` — prevents td padding from inflating rows past 42px

**Defect 9 — Gantt right panel not full-width:**
- `.gantt-pane { min-width: 0; }` — CSS grid child was defaulting to `min-width: auto` (= content width), allowing it to overflow the `1fr` column boundary
- `.gantt-pane .gantt-container { min-width: 100%; }` — ensures the Frappe SVG container fills the available column width
