### Human Verification Required

The automated checks confirm all implementation is present and wired. The following behaviors require a live browser session to confirm:

#### 1. Visual Row Alignment

**Test:** Open a project plan with at least 5 tasks. Compare left-rail rows to Frappe Gantt SVG bars side by side.
**Expected:** Each grid row sits flush (no vertical gap or offset) with its corresponding Gantt bar. Grid column header labels sit at the bottom of the 85px header region, aligning with Frappe's day/week label row.
**Why human:** CSS height values (42px/85px) are derived from Frappe constants, but browser rendering of inline SVG coordinate offsets cannot be confirmed by grep.

#### 2. Drag Handle Isolation — Inputs Do Not Drag

**Test:** Click and slowly drag from inside a task Name input cell. Repeat for Duration and Resources cells.
**Expected:** No drag cursor appears. Row order does not change. Then drag from the # (row number) column — drag initiates normally.
**Why human:** HTML5 dragstart event interaction with browser drag threshold requires live testing.

#### 3. Drag-Over Highlight Isolation

**Test:** Begin dragging row B (from the # cell). Hover over row B itself, then over row C without releasing.
**Expected:** Row C shows a blue top border only when the cursor is directly over row C. Moving back to row B removes row C's highlight.
**Why human:** CSS class toggling during mousemove is a real-time visual effect.

#### 4. Enter Key Commits Duration/Resources

**Test:** Click a Duration cell, type "5d", press Enter (do not Tab or click away).
**Expected:** The Gantt bar's end date updates, confirming the Firestore write completed via the blur path.
**Why human:** Requires live Firestore connection to confirm the round-trip; End date update is visual.

#### 5. Smart Indent Inheritance

**Test:** Create a task at indent level 1 (with a parent). Then type a name in the empty trailing row and press Enter.
**Expected:** The new task appears at indent level 1 (same as the row above it), not at root level.
**Why human:** Row indentation is a visual tree rendering based on parent_task_id; requires Firestore write and re-render.

#### 6. Gantt Popup Suppression

**Test:** Click a Gantt bar.
**Expected:** No popup or tooltip appears. The bar may highlight or select, but no floating DOM element is injected.
**Why human:** `popup: false` suppresses Frappe's internal popup logic but DOM injection must be confirmed absent in browser DevTools.

#### 7. Resizable Panel Divider

**Test:** Locate the thin 4px divider between the task grid and Gantt pane. Drag it left and right.
**Expected:** Left rail grows/shrinks in real time. Gantt pane adjusts. Divider highlights blue on hover and while dragging. Panel cannot drag past 15% or 75% of total pane width.
**Why human:** CSS grid resize via `style.gridTemplateColumns` and the clamp limits require drag-to-edge testing in a browser.

#### 8. Delete Confirm Modal

**Test:** Right-click a task that has subtasks (indented children), choose Delete Row. Click Cancel. Re-confirm task exists. Then delete via Delete button.
**Expected:** Centered modal appears over a darkened backdrop. Shows child count. Cancel dismisses modal, task remains. Delete removes task and all subtasks. Backdrop click also dismisses.
**Why human:** Fixed overlay positioning, z-index stacking, and modal interaction are visual/behavioral.
