<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client JS → Firestore | Paste writes new documents; source data is from the authenticated onSnapshot, not user-typed strings |
| DOM → JS (onclick strings) | Task IDs passed via JSON.stringify in onclick attributes; parsed back with JSON.parse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86.10-03-01 | Tampering | gridPasteRows — new task ID generation | accept | generateTaskId uses the existing sequential ID pattern scoped to the project; Firestore security rules enforce project membership for writes |
| T-86.10-03-02 | Tampering | JSON.stringify of task IDs passed through onclick | accept | IDs are escaped with escapeHTML before embedding in onclick strings; they are then parsed with JSON.parse inside the handler — round-trip is safe for UUID/slug task IDs |
| T-86.10-03-03 | Denial of Service | Pasting a large clipboard to Firestore | accept | _clipboardTasks is bounded by the number of selected rows, which is bounded by the project's task count; no unbounded write loop |
| T-86.10-03-04 | Information Disclosure | _clipboardTasks persisting after destroy | mitigate | destroy() clears `_clipboardTasks = []`; clipboard does not persist across route navigation |
</threat_model>
