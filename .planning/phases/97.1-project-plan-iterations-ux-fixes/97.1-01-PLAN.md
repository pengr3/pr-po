<objective>
Fix two behavioral bugs introduced in Phase 97: (1) the baseline "cleared" state does not persist across page refresh — loadBaselines() unconditionally fetches and auto-selects the most-recent baseline, undoing the user's clearBaseline() action; (2) restoreIteration() unconditionally creates an auto-snapshot before restoring, so switching between named auto-snapshot iterations (auto: true) creates a new auto-snapshot each time, causing exponential history growth.

Purpose: These bugs degrade the core iteration UX. Bug 1 makes "clear baseline" feel broken. Bug 2 fills the history rail with redundant auto-snapshots every time the user browses their iteration history.

Output: Modified project-plan.js with localStorage-based baseline persistence (Bug 1) and an iter.auto guard in restoreIteration() (Bug 2).
</objective>
