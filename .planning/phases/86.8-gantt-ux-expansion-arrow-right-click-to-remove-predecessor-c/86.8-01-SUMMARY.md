# Phase 86.8 Plan 01: Gantt UX Expansion (Features 1, 2, 3, 7) Summary

**Four interactive Gantt features land together in `app/views/project-plan.js` and `styles/views.css`: arrow right-click → remove predecessor, collapsible parent toggles, drag-parent-moves-children cascade, and keyboard shortcuts (Delete / ArrowUp/Down / Enter / Escape) with single-target row/arrow selection.**

- **Tasks landed:** 3 of 4 (Task 4 is a `checkpoint:human-verify` deferred to the orchestrator's user prompt)
- **Files modified:** 2 (`app/views/project-plan.js`, `styles/views.css`)
- **Net diff:** +507 / −27 across 2 files (project-plan.js +465 net, views.css +42)

## Accomplishments

- **Feature 1 — Arrow right-click → remove predecessor.** Added `tagArrowsWithFromTo(svg)` reverse-mapper, `mountGanttArrowContextMenu()` listener, `showArrowContextMenu()` UI, and `removeArrowDependency(fromId, toId)` Firestore writer. Re-attached on every `renderGantt()` AND on every `setGanttZoom()` because Frappe v1.2.2 rebuilds the SVG node on each refresh / zoom.
- **Feature 2 — Collapsible parent tasks.** Added `_collapsedParents: Set<task_id>`, `getDescendantIds(parentId)`, and `isHiddenByCollapse(t)` predicate shared by `renderTaskGrid` (filters `visibleSorted`) and `renderGantt` (post-walk filter on `frappeTasks`). Toggle UI: `<span class="tg-collapse-toggle">▼</span>` prepended to parent name cells, rotates to ▶ via CSS when collapsed. Delegated click handler in `bindGridEvents` (`_gridCollapseHandler`) flips state and re-renders both panes simultaneously to preserve row-pitch parity.
- **Feature 3 — Drag parent → moves children.** Replaced the early-return reject in `handleGanttDateChange`'s parent branch with a cascade path: compute `daysDelta` from descendants' true minimum start (envelope-derived, not `t.start_date` which can be stale during in-flight snapshots), build `childUpdates[]`, stash as `_pendingDragWrite = { taskId, parentDrag: true, childUpdates, parentId }`. Extended `mountGanttBarDragGuard()` mouseup flush to switch on `_pendingDragWrite.parentDrag` and commit one `writeBatch` for all child updates, then `recomputeParentDates(taskId)` and (if applicable) the grandparent.
- **Feature 7 — Keyboard shortcuts.** Added `_planKeydownHandler` window-level listener with INPUT/TEXTAREA/SELECT short-circuit and `planViewSurface` mount guard. Switch on `event.key`:
  - `Delete` / `Backspace` → arrow selected: `removeArrowDependency` (no confirm); row selected: `gridDeleteRow` (existing modal).
  - `ArrowDown` / `ArrowUp` → step row selection through visible rows; auto-select first visible row if none selected; `scrollIntoView({ block: 'nearest' })`.
  - `Enter` → focus first `input.tg-input:not([disabled])` on selected row.
  - `Escape` → clear selection.
- **Selection state.** Added `selectRow`, `selectArrow`, `clearSelection` helpers (row XOR arrow, mutually exclusive). Wired row click via delegated `_gridRowClickHandler` (skips toggle / INPUT / SELECT / BUTTON). Wired arrow click via `mountGanttArrowClickSelect()` (re-attached on every `renderGantt` and `setGanttZoom`, like the contextmenu handler). CSS `.tg-row.tg-selected` paints `#eff6ff` background + 2px `#1a73e8` left-edge stripe; `.gantt-container .arrow.tg-arrow-selected` paints `#1a73e8` 2.5px stroke.
- **destroy() cleanup.** Every new `addEventListener` has a matching `removeEventListener` in `destroy()`: `_ganttArrowContextHandler`, `_ganttArrowMenuClickaway`, `_ganttArrowClickHandler`, `_planKeydownHandler`, `_gridCollapseHandler`, `_gridRowClickHandler`. State variables nulled. `window.removeArrowDependency` deleted. Open `.gantt-arrow-menu` removed defensively.

## Decisions Made

- **Frappe arrow tagging via coordinate reverse-mapping** — chose `path.getBBox()` + `bar.getBBox()` matching within 6px tolerance over forking Frappe to add `data-from` / `data-to` natively. Cheaper to maintain, survives `gantt.refresh()` because we re-tag on every renderGantt step 5b. Ambiguous matches stay untagged and surface a "Could not identify" toast (T-86.8.1-01).
- **Cascade write reuses the Phase 86.7 phantom-drag slot** — `_pendingDragWrite` extended with `{ parentDrag: true, childUpdates[] }` shape. Mouseup flush switches on `parentDrag` to pick updateDoc vs writeBatch. Single Firestore round-trip per drag preserved.
- **Collapse state lives in-memory only** — per CONTEXT D-2 deferral. Cleared on view destroy. No Firestore round-trip required.
- **Backspace mirrors Delete** — Mac compatibility (Delete-without-Fn = Backspace).
- **Row XOR Arrow selection** — `selectRow` clears `_selectedArrow`, `selectArrow` clears `_selectedTaskId`. Single-target keeps the keyboard handler unambiguous.
- **CLAUDE.md compliance** — no new dependencies, ES6 modules, hash-based routing untouched, listener cleanup in destroy() per project pattern. Status string casing unchanged. Window functions follow the established `window.fnName = fnName; ... delete window.fnName;` pattern.
