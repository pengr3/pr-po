**(b) Add `computeCriticalPath(taskList)` helper** (place after `applyFsViolationStyles` at ~line 1444):
Forward pass: compute earliest_finish for each task. earliest_start(t) = max(earliest_finish(p) + 1day) over all p ∈ t.dependencies; if t has no deps, earliest_start = t.start_date. Earliest_finish = earliest_start + duration_days - 1 (inclusive end). Walk in topological order — derive via simple DFS or repeated-pass-until-stable since real plans are tiny (<1000 tasks).

Backward pass: project_end = max(earliest_finish) over all tasks. latest_finish(t) = min(latest_start(s) - 1day) over all s where t ∈ s.dependencies; if t has no successors, latest_finish = project_end. latest_start = latest_finish - duration_days + 1.

Slack = latest_start - earliest_start (in days). Critical = slack === 0.

Implementation hints:
- Use ms-since-epoch for date math: `new Date(iso + 'T00:00:00').getTime() / 86400000` to get day index. Avoid timezone surprises.
- Use a `Map<task_id, { es, ef, ls, lf, dur }>`.
- Skip parent tasks — only leaf tasks participate in CPM; parents inherit critical state from their children (any critical descendant → parent is critical).
- Skip tasks missing start_date or end_date (incomplete data).
- Cycle protection: if forward pass doesn't converge in `tasks.length + 1` iterations, log a console warning and return an empty Set (the cycle is already toasted by `detectDependencyCycle` elsewhere).
- Return `Set<task_id>` of tasks where slack === 0 AND duration ≥ 1 day.

WHY comment: "CPM ignores parent tasks because parent dates are envelope-computed (D-11). Including them would duplicate critical-path edges through every parent. Parents inherit critical visual state from any critical descendant during the DOM tag pass."

function computeWeightedProgress(parentId) {
  // Children-of-parent only (one level); but children that are themselves parents recurse.
  const kids = tasks.filter(t => t.parent_task_id === parentId);
  if (kids.length === 0) return 0;
  let totalDays = 0, weightedSum = 0;
  kids.forEach(k => {
    const isKidParent = tasks.some(x => x.parent_task_id === k.task_id);
    const kidProgress = isKidParent ? computeWeightedProgress(k.task_id) : (k.progress ?? 0);
    const sd = k.start_date, ed = k.end_date;
    if (!sd || !ed) return;
    const dur = Math.max(1, Math.round((new Date(ed + 'T00:00:00') - new Date(sd + 'T00:00:00')) / 86400000) + 1);
    totalDays += dur;
    weightedSum += dur * kidProgress;
  });
  return totalDays > 0 ? Math.round(weightedSum / totalDays) : 0;
}
```

function matchesSearch(t) {
  if (!_searchQuery) return true;
  return (t.name || '').toLowerCase().includes(_searchQuery);
}
function getVisibleByFilter(t) {
  // A task is visible if:
  //  (1) it matches directly, OR
  //  (2) any descendant matches (parent stays visible, greyed), OR
  //  (3) any ancestor matches (child stays visible — context aid)
  if (matchesSearch(t)) return { visible: true, isContextOnly: false };
  // Descendant match → keep parent visible, greyed
  const descendants = getDescendantIds(t.task_id);
  if (descendants.some(id => {
    const d = tasks.find(x => x.task_id === id);
    return d && matchesSearch(d);
  })) return { visible: true, isContextOnly: true };
  // Ancestor match → keep child visible (so user sees context)
  let cur = tasks.find(x => x.task_id === t.parent_task_id);
  while (cur) {
    if (matchesSearch(cur)) return { visible: true, isContextOnly: false };
    cur = tasks.find(x => x.task_id === cur.parent_task_id);
  }
  return { visible: false, isContextOnly: false };
}
```
