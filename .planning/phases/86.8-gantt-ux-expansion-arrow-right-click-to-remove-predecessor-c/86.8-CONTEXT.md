# Phase 86.8: Gantt UX Expansion — Context

**Gathered:** 2026-05-09
**Status:** Ready for planning
**Source:** Feature list locked at /gsd-phase --insert (no discuss-phase)

<domain>
## Phase Boundary

Expand interactive UX of the Gantt chart in the project-plan view. Seven features bundled, all touching the same surfaces (`app/views/project-plan.js`, `styles/views.css`). No backend collection changes except a single new `progress` field on `project_tasks`.

**In scope (locked):**

1. **Right-click arrow → remove predecessor.** Right-clicking a Frappe `.arrow` element opens a small context menu (or directly confirms) to delete the dependency edge. Updates Firestore `dependencies` array on the dependent task; arrow disappears on next snapshot.
2. **Collapsible parent tasks.** Add a (-)/(+) toggle on parent rows in both panes (left grid + right Gantt). When collapsed, all descendant rows are hidden in the left grid AND their bars hidden in the right pane. State persisted in-memory per session (deferred: Firestore persistence).
3. **Drag parent → moves all children.** When a parent bar is dragged horizontally, all descendant bars shift by the same `daysDelta`. Single mouseup commits all date updates atomically.
4. **Critical-path highlight.** Compute the longest dependency chain ending at the latest task end-date. Color those bars and arrows red (or `--danger` token). Recompute on any date change.
5. **Progress % per bar.** Add `progress` field (0-100, default 0) to `project_tasks` schema. Render as shaded fill within each bar (left portion = done). Inline grid editor gets a Progress column with numeric input.
6. **Task search/filter bar.** Search box above the grid. Filters visible rows in left grid AND hides corresponding bars in the right pane. Clear-X button. No persistence.
7. **Keyboard shortcuts (Delete to remove).** Delete key removes the selected task (with confirm) OR selected arrow (no confirm — already destructive but reversible by re-linking). Arrow keys navigate row selection up/down.

**Out of scope (deferred):**

- Dependency types (currently FS only — no SS/FF/SF or lag/lead times).
- Milestones (zero-duration diamonds).
- Working calendar (skip weekends).
- Multi-select bulk operations.
- Export/print.
- Persisted collapse state in Firestore.

</domain>

<decisions>
## Implementation Decisions

### Plan organization (recommended split)

The planner SHOULD split this into 2 plans for execution discipline:

- **Plan 01 — Interactive UX cluster:** features (1) arrow right-click, (2) collapsible parents, (3) drag-parent-moves-children, (7) keyboard shortcuts. All share the same DOM-event-routing layer (mouse/keyboard handlers on `.gantt-pane`/`.task-grid`, hit-testing, selection state). High coupling — best implemented together.
- **Plan 02 — Analytical/schema cluster:** features (4) critical path, (5) progress %, (6) search/filter. Pure compute + minor schema. Can land independently of Plan 01.

The planner MAY override this split if a different organization yields lower coupling — but justify it.

### Feature 1 — Arrow right-click

- Frappe `.arrow` SVG paths exist in DOM after `gantt.refresh()`. They have no native id attrs (per existing comment at `applyFsViolationStyles`).
- Identification approach: parse the arrow's `d` attribute (start/end points) and reverse-map to `(from_task, to_task)` by matching to known task bar positions. OR add a Frappe patch that tags arrows with `data-from`/`data-to`. **Choose the lower-risk path during planning.**
- UI: right-click → confirm dialog ("Remove dependency `<from_id>` → `<to_id>`?") → mutate Firestore `project_tasks/<to_id>.dependencies` (filter out from_id) → `gantt.refresh()` on snapshot.
- Edge case: parent-summary arrows are already styled `pointer-events: none` — right-click should also be no-op there.

### Feature 2 — Collapsible parents

- Parent detection: `tasks.some(t => t.parent_task_id === parent.task_id)`.
- Toggle UI: small `▼`/`▶` icon in the `Name` cell of the left grid. Click toggles state.
- State: in-memory `Set<task_id>` of collapsed parents. NOT persisted yet (deferred).
- Render: when collapsing, hide left-grid `<tr>` of all descendants (recursive) AND set bar wrappers `display: none` (or filter out from `gantt.tasks` and refresh — pick whichever produces less flicker).
- Vertical alignment: critical to maintain row pitch parity (lessons from gantt-row-vertical-misalign session). Hidden rows must be hidden in BOTH panes simultaneously, no half-states.

### Feature 3 — Drag parent → moves all children

- Hook into `gantt.on('date_change')` for parent bars. If task is a parent, compute `daysDelta` from old start to new start, then iterate descendants and shift each by the same delta.
- Phase 86.7 phantom-drag pattern: `_ganttBarDragging` flag suppresses mid-drag writes; commit only on mouseup. Reuse this pattern — single Firestore batch write at mouseup containing parent + all children.
- Conflict: child task constraints (e.g., child can't move before its earliest predecessor outside the family). For Phase 86.8, accept "parent-drag bypasses dependency checks" — reschedule violations are flagged by existing `applyFsViolationStyles` post-snapshot.

### Feature 4 — Critical path

- Algorithm: forward pass (compute earliest start/finish for each task following `dependencies` arrays) → backward pass (latest start/finish from project end) → tasks where slack=0 are on critical path.
- Trigger: recompute on each `tasks` snapshot. Cache the result; invalidate on any date or dependency change.
- Render: add CSS class `.bar-wrapper.critical` and `.arrow.critical`; style red. Toggle visibility via a "Show critical path" checkbox in the Gantt header (default ON).

### Feature 5 — Progress %

- Schema: add `progress: number` (0-100, default 0) to `project_tasks` documents. Schemaless Firestore — no migration; reads default to 0 when absent.
- Render: shaded fill at left portion of each bar via Frappe's built-in `progress` field on the task object (Frappe v1.2.2 supports this natively — pass it through during `tasks.map()` in `renderGantt`).
- Edit: add a Progress column to the inline grid editor. Numeric input with min=0, max=100. Update Firestore on blur/Enter (consistent with existing duration/resources fields).
- Parent rollup: parent progress = weighted average of child progress (by duration). Compute client-side; never persist parent progress.

### Feature 6 — Task search/filter bar

- Location: above the left-pane grid header, full pane width.
- Behavior: substring match on task name (case-insensitive). When active, filter left-grid rows AND hide non-matching bar wrappers.
- Clear: X icon resets to show all.
- Edge case: If a parent matches but its children don't, still show the parent. If a child matches but its parent doesn't, show the parent (greyed) so the child is in context.

### Feature 7 — Keyboard shortcuts

- **Delete (selected task):** confirm dialog ("Delete task `<name>` and N children?") → cascade delete via existing `deleteTask()` flow.
- **Delete (selected arrow):** no confirm — quick delete, route through Feature 1's removal path.
- **Up/Down arrows:** move row selection. `selectedRowIndex` state in module scope.
- **Enter on a row:** focus first editable cell in inline grid editor (consistent with existing Excel-style behavior from Phase 86.5).
- **Escape:** clear selection.
- Listener: window-level keydown, scoped to `document.querySelector('#projectPlanView')` being visible. Tear down in `destroy()`.

### Claude's Discretion

- Context-menu UI style (custom div vs `<menu>` element vs immediate confirm dialog).
- Critical-path color exact value (`--danger` `#ef4444` is the brand red — likely fine).
- Whether to use `gantt.refresh()` or direct DOM mutation for collapse — pick whichever has lower visual flicker.
- Critical-path checkbox placement in the Gantt header.

</decisions>

<specifics>
## Specific Ideas

- **Hit-testing arrows:** Frappe arrows are SVG `<path>` elements with `d="M…L…"`. Mouse coordinates inside the SVG can be matched to the path's bounding box; on right-click, traverse `event.target` ancestors looking for `.arrow` class.
- **Collapse persistence (deferred):** Future phase could persist `collapsed_parents` array per-user-per-project in a new `project_plan_view_state` collection.
- **Progress % visual consistency:** Frappe paints the progress overlay using the bar's primary color at reduced alpha — match this so we don't fight the renderer.

</specifics>
