## What Was Built

### applyFsAutoSchedule(successorId, predecessorId)

New async helper function added after `recomputeParentDates` in `project-plan.js`.

Behavior:
- Looks up successor and predecessor from module-scope `tasks[]`
- Guard: returns immediately if either task missing or lacks date fields
- No-op guard: returns if `successor.start_date > predecessor.end_date` (already FS-compliant)
- Computes `newStart = addDays(predecessor.end_date, 1)`
- Preserves duration: calculates `durationDays` from current start-to-end span, recomputes `newEnd = addDays(newStart, durationDays - 1)`
- Writes `{start_date: newStart, end_date: newEnd, updated_at: serverTimestamp()}` to Firestore
- Patches `tasks[idx]` immediately (synchronous local update before onSnapshot fires)
- Calls `recomputeParentDates(successor.parent_task_id)` if successor is a child task
- Internal try/catch: non-fatal — logs warning, does not throw to callers

### Wire-up: Gantt Drag-to-Link

In the `svg.addEventListener('mouseup', ...)` handler, inside the existing try block after the dependency `updateDoc`:


### Wire-up: handleGridCellBlur predecessors branch

In the `handleGridCellBlur` try block, after the `recomputeParentDates` call:
```javascript
if (col === 'predecessors' && updateData.dependencies) {
    const prevDeps = Array.isArray(t.dependencies) ? t.dependencies : [];
    const newlyAdded = updateData.dependencies.filter(id => !prevDeps.includes(id));
    for (const predId of newlyAdded) {
        await applyFsAutoSchedule(taskId, predId);
    }
}
```
Only newly-added predecessors trigger the shift (prevDeps diff).
