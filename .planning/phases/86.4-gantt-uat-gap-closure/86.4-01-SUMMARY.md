# Phase 86.4 Plan 01: D-CLAMP Start>End Guard Summary

**Silent end_date auto-clamp in handleGridCellBlur col='start' — Frappe never receives start_date > end_date from grid cell edits**

## Accomplishments

- Replaced the bare `return` guard in the `col === 'start'` branch of `handleGridCellBlur` with a D-CLAMP auto-clamp
- `clampedEnd` is computed as: if `t.end_date` exists and is earlier than `rawValue`, clamp to `rawValue`; otherwise keep `t.end_date` (or `rawValue` as fallback for tasks without an end_date)
- `end_date` is now always included in the Firestore updateData payload for start edits — no conditional payload structure, no drift risk
- UAT test 11 partial gap ("editing Start past End should clamp End") closed; zero Frappe "start of task can't be after end" errors possible from this path

## Files Created/Modified

## Decisions Made

- D-CLAMP is always-include: writing `end_date: clampedEnd` even when `clampedEnd === t.end_date` (no real change to end_date) is harmless at the Firestore level and avoids a conditional payload structure that could drift in future edits
- Silent clamp per spec — no toast fired when clamping occurs
- `t.end_date || rawValue` fallback in clampedEnd computation is defensive for tasks created without an end_date (rare schema state)
