## Notable Deviation: Week View Label Injection

**Plan 04 truth:** "In Week view, no extra label row is injected — Frappe's native '11 May' / '18 May' labels are sufficient."

**Actual implementation:** The debug session (commit bb89717) added Week view HTML injection into Frappe's `.lower-header` DOM element. Each week column receives a `'D MMM YY'` date label and an M T W T F S S sub-row wrapped in a `div.gantt-week-lower-label`. Cleanup is handled idempotently (`querySelectorAll('.gantt-week-lower-label').forEach(el => el.remove())`) in both `renderGanttHeaderSvg()` and `destroy()`.

**Assessment:** This is an intentional and documented improvement (SUMMARY key-decisions explicitly records this). The phase goal success criteria do not specify that Week view must be skipped — only that Day and Month views have scrolling SVG labels. The Week injection adds value. The `gantt-week-lower-label` class is properly cleaned up. **Not a blocker.**

## Human Verification Required

### 1. Day View — M T W Th F S S Labels

**Test:** Open a project plan with tasks spanning at least 2 weeks. Switch to Day zoom. Scroll the Gantt timeline horizontally left and right.
**Expected:** Single-letter weekday labels appear in the lower Gantt header band. Labels align to their respective date columns. Labels scroll with the timeline (do not stay pinned to the left edge). Saturday and Sunday labels are lighter in color and italic.
**Why human:** SVG x-coordinate calculations depend on `gantt.config.step`, `gantt.config.column_width`, and `labelY = headerHeight - 8`. These are correct in code, but pixel-level alignment and scrolling behavior require a running browser.

### 2. Month View — Week-Range Sub-Labels

**Test:** Switch to Month zoom. Scroll horizontally.
**Expected:** Labels like "03 May - 09", "10 - 16", "31 - 06 Jun" appear in the SVG lower header band. First cell of each month's run includes the MMM abbreviation; subsequent same-month cells omit it; cross-month end carries the new MMM. Labels scroll with the timeline content.
**Why human:** Month xPerDay uses `configColWidth / 30` approximation; correct label centering within Frappe's variable-width month columns requires visual inspection.

### 3. Week View — HTML Label Row

**Test:** Switch to Week zoom.
**Expected:** A date label row ("4 May 26") and M T W T F S S sub-row appear at the bottom of each week column header. Labels scroll with the week columns.
**Why human:** The DOM injection into `.lower-header` uses `position: absolute` inside a newly-set `position: relative` parent. Whether the labels render within bounds and scroll correctly requires browser observation.

### 4. FS Auto-Scheduling — Link Creation

**Test:** In a project with two tasks (Task A ends 2026-05-10, Task B starts 2026-05-05). Drag-link from Task A to Task B (or type A's row# into B's Predecessors cell).
**Expected:** After save, Task B's start_date shifts to 2026-05-11. Task B's duration is preserved (end_date recalculated). Gantt bar for Task B moves right.
**Why human:** Firestore write + onSnapshot re-render + Gantt bar movement require a live browser session.

### 5. Synchronized Scroll — Long Task List

**Test:** Open a project plan with 30+ tasks so the task list overflows both panels. Scroll the left rail downward. Then scroll the Gantt pane downward.
**Expected:** Scrolling either panel causes the other to follow to the same position. Rows and bars stay aligned. No runaway scroll loop on rapid drag-scrolling.
**Why human:** rAF-based `_syncingScroll` reset behavior with rapid successive events requires a live browser to observe.
