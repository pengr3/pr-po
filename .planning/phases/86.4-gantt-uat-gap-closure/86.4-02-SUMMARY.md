# Phase 86.4 Plan 02: D-SCROLL Synchronized Vertical Scroll Summary

**One-liner:** Synchronized vertical scroll between left task-grid rail and right Gantt pane via idempotent `bindScrollSync()` with `_syncingScroll` guard.

## What Was Built

UAT gap D-SCROLL (test 14): when the task list is long, the left rail and the right Gantt pane now share vertical scroll position. Scrolling either panel drives the other to the same `scrollTop`. An infinite-loop guard (`_syncingScroll`) prevents the A‚ÜíB‚ÜíA feedback cycle.

## Implementation Details

### Task 1 ‚Äî project-plan.js changes

**Module-scope vars added (lines 44-47):**
- `_syncScrollRailHandler` ‚Äî stores the rail scroll listener for idempotent cleanup
- `_syncScrollPaneHandler` ‚Äî stores the pane scroll listener for idempotent cleanup
- `_syncingScroll` ‚Äî boolean guard flag that breaks the A‚ÜíB‚ÜíA infinite loop

**`bindScrollSync()` function (added after `installGanttScrollClamp()`):**
- Selects `#taskGridRail` and `.gantt-pane`; returns early if either is missing
- Removes previously attached listeners before attaching new ones (idempotent)
- Attaches rail‚Üípane and pane‚Üírail synchronization handlers, each guarded by `_syncingScroll`

**Call site:** Wired in the `onSnapshot` callback, immediately after `renderGantt()`, so both DOM elements are guaranteed to exist.

**`destroy()` cleanup:** Both listeners removed; all three vars nulled/reset to `false`.
