## Implementation Decisions

### D-03: Calendar Header — SVG Text Injection (replaces broken DOM overlay)

The Phase 86.3 DOM overlay approach used `position: absolute; width: 100%` on `.gantt-pane`. Since `width: 100%` equals viewport width (~700px), cells scrolled off-screen immediately. The overlay was removed (renderGanttHeaderOverlay calls disabled).

**New approach:** After `new Gantt()` / `gantt.refresh()` renders the SVG, inject `<text>` elements directly into Frappe's `<svg>` for Day view and Month view.

**Day view:** Below Frappe's lower header row, add M/T/W/Th/F/S/S single-letter labels. One `<text>` per day column, x-position matches the column center (same `dayDiff * xPerDay` formula that renderTodayLine uses). Letters for Saturday and Sunday use a lighter fill (weekend styling).

**Week view:** Frappe's native labels already show "11 May", "18 May" format correctly. No SVG injection needed for Week view.

**Month view:** Frappe's native upper row shows month names. Inject week-range sub-labels (e.g., "03 May - 09", "10 - 16", "17 - 23") as `<text>` elements in the lower header band. First cell of each month's run carries the MMM abbreviation; subsequent same-month cells omit it; cross-month cells include end MMM. This is the same H-01 labelling contract from Phase 86.3 Plan 05.

**Integration:** `renderGanttHeaderSvg()` (new function name) called from `renderGantt()` step 9 and `setGanttZoom()`. Must be idempotent — query and remove any prior `.gantt-header-svg-label` elements before re-injecting.

**Frappe SVG structure (verified from existing code patterns):**
- SVG is at `#ganttPane svg`
- Column x positions: `dayDiff * (24/stepHours) * colWidth` from `gantt.gantt_start` (same as renderTodayLine)
- Frappe header band height: ~45px from top of SVG (upper row ~0-23px, lower row ~24-45px)
- Day view: inject at y ≈ 38px (below Frappe's lower header text at y ≈ 28px)
- Month view: inject at y ≈ 38px (same band, below Frappe's existing month-name upper row)

### D-FS: FS Auto-Scheduling (successor auto-shift on predecessor link)

When a predecessor link is created (via drag-to-link OR Predecessors cell edit), the successor task's `start_date` should automatically shift to `predecessorEnd + 1 day` if the successor currently starts BEFORE the predecessor ends (i.e., the link would be violated).

**Trigger points:**
1. `handleGanttBarDrop()` — after the FS dependency is written to Firestore, check if new successor.start_date ≤ predecessor.end_date; if so, update successor.start_date = predecessor.end_date + 1d and recompute successor.end_date via parseDuration
2. `handleGridCellBlur(col = 'predecessors')` — after dependency write, same shift check for each newly added dep

**Cascading:** Only shift the DIRECT successor. Do not cascade (A→B→C: linking A→B shifts B, does NOT auto-shift C). Cascade is a future feature.

**No-op conditions:** If successor.start_date is already after predecessor.end_date, do nothing.

### D-CLAMP: Start > End Guard

When the user edits the Start cell of a task via Tab/blur and the new start_date would be AFTER the task's current end_date, auto-set end_date = start_date (same-day, 1-day inclusive bar). Show no toast — silently clamp.

**Where:** In `handleGridCellBlur(col = 'start')`, after validating the new start date: if `rawValue > t.end_date`, include `end_date: rawValue` in the Firestore updateData alongside `start_date: rawValue`.

**Why:** Frappe throws "start of task can't be after end" error on render; this is the simplest guard.

### D-SCROLL: Synchronized Vertical Scroll

Left rail (`.task-grid-rail`) and Gantt pane (`.gantt-pane`) must scroll vertically together. When the user scrolls either panel vertically, the other panel scrolls to match.

**Implementation:** In `bindGridEvents()`, after the grid is rendered, attach `scroll` event listeners on both `.task-grid-rail` and `.gantt-pane`. Use a `_syncingScroll` flag (boolean) to prevent infinite loop (A scrolls B, B fires event scrolling A, etc.).

```javascript
let _syncingScroll = false;
rail.addEventListener('scroll', () => {
    if (_syncingScroll) return;
    _syncingScroll = true;
    pane.scrollTop = rail.scrollTop;
    _syncingScroll = false;
});
pane.addEventListener('scroll', () => {
    if (_syncingScroll) return;
    _syncingScroll = true;
    rail.scrollTop = pane.scrollTop;
    _syncingScroll = false;
});
```

Store listener refs for cleanup in `destroy()`. Note: `.gantt-pane` already has `overflow: auto`; ensure `.task-grid-rail` also allows vertical scroll (`overflow-y: auto`) — verify CSS.
