### 3. Week view — HTML label row renders correctly
expected: |
  Switch to Week zoom. A date label row ("4 May 26") and M T W T F S S sub-row
  appear at the bottom of each week column header. Labels scroll with the week
  columns and stay within header bounds.
result: issue
reported: "date label and row border appear on Tuesday — should be Monday"
severity: minor
resolved_by: "renderCustomGanttHeader Week branch now snaps cur back to preceding Monday
  (daysBack = gantt_start.getDay()-1) and stores weekOffsetPx on overlay dataset.
  _bindOverlayScrollSync applies -(weekOffset+scrollLeft) so Monday cells align with
  Frappe's day columns. Commit pending re-test."

### 5. Synchronized scroll — rail and Gantt pane stay aligned
expected: |
  Open a project plan with 30+ tasks so both panels overflow. Scroll the left
  rail down — Gantt pane follows. Scroll the Gantt pane down — rail follows.
  Rows and bars stay aligned. No runaway scroll loop on rapid scrolling.
result: [pending]
note: "Root cause identified and fixed: .gantt-container was auto-sized to SVG
  height (no overflow) so scroll events went to .gantt-pane instead, bypassing
  bindScrollSync. Fix: fixGanttContainerScroll() constrains container height =
  pane - headerH - 16px and shifts SVG up by headerH so task bars start at
  container y=0. .gantt-pane changed to overflow-y: hidden. Awaiting re-test."
