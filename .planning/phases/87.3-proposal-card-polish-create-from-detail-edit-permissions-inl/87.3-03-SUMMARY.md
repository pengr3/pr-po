# Phase 87.3 Plan 03: Proposal Card Polish — Gate Removal, Start Proposal, Card Redesign Summary

**isInProposalRange gate removed from both detail views; proposalInlineCard always rendered; loadProposalCard has three-way logic with canDrive-gated Start Proposal CTA; renderInlineProposalCard shows dynamic stage label + colored dot + version number**

## Performance

- **Duration:** ~15 min
- **Completed:** 2026-05-22
- **Tasks:** 2
- **Files modified:** 3

## Accomplishments

### Task 1: project-detail.js
- Removed `isInProposalRange` gate; `proposalInlineCard` div is now unconditionally in the HTML template (D-07)
- Removed `PROPOSAL_RANGE_STATUSES` and `getProposalStatusBadge` from the proposals.js import (no longer needed)
- Added `openCreateProposalModal` to proposal-modal.js import
- Added `_proposalStageLabel(status)` helper mapping 6 status strings to human-readable labels (D-12)
- Added `_proposalStatusDotColor(status)` helper returning hex color per status (D-13)
- `renderInlineProposalCard(proposal, canDrive)`: header now shows colored dot + stage label (no badge); proposal ID line shows `PROP-2026-001 · v1`; Submit button gated by `canDrive` (D-12/D-13/D-14/D-15)
- `loadProposalCard` three-way snap.empty logic: Start Proposal card (canDrive + For Proposal), placeholder (non-canDrive + For Proposal), hide container (all other statuses with no proposal)
- `window._startProposalCallback` registered as closure over parentDocId for the Start Proposal button onclick
- `attachWindowFunctions` registers `window.openCreateProposalModal`
- `destroy()` cleans up `window.openCreateProposalModal` and `window._startProposalCallback`

### Task 2: service-detail.js + components.css
- Full parity with project-detail.js applied to service-detail.js (D-06): same import changes, same gate removal, same three-way loadProposalCard logic using `currentService`/`currentServiceDocId`, same renderInlineProposalCard redesign, same attachWindowFunctions and destroy() additions
- `styles/components.css`: added `.proposal-inline-card--start` (dashed border + light gray background for Start Proposal empty-state variant); updated `.proposal-inline-card__header` with `gap: 6px`, `margin-bottom: 0.75rem`, `font-size: 0.875rem`, `font-weight: 600`, `color: #1e293b`
