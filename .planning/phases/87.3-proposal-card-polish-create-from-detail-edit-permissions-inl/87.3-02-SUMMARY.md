## Accomplishments

- Added `let _createModalOnClose = null` module-level variable alongside `createModalMode` / `createModalEditingId`
- Extended `openCreateProposalModal` to accept `(preselectedProjectId = null, onClose = null)` — stores callback, then after `showCreateModal(null)` returns, finds `#proposalCreateProject`, sets value, disables it, and dispatches `change` event so existing client auto-fill listener fires
- Extended `closeCreateProposalModal` to use the local-cb-before-null pattern: `const cb = _createModalOnClose; _createModalOnClose = null; if (typeof cb === 'function') cb()` — prevents double-fire re-entry (T-87.3-02-02 mitigation)

## Files Created/Modified

## Decisions Made

- Preselect step fires after `showCreateModal()` returns (synchronously) so the `change` event listener is already bound on the `#proposalCreateProject` element — no race condition possible
- Callback stored locally before nulling module var to handle the edge case where a callback re-invokes `closeCreateProposalModal` (guard from threat register T-87.3-02-02)
- `disabled=true` is UI-only; the real security enforcement remains in Firestore rules (Plan 01)
