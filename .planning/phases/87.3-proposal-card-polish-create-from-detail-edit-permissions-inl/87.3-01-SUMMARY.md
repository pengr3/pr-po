## Files Created/Modified

## Decisions Made
- `request.resource.data.get('parent_collection', 'projects')` with default used in create rule (vs `resource.data.parent_collection`) because the document doesn't exist yet at create time — this is a correctness requirement, not a style choice
- Ops_user and services_user written as separate `|| (...)` branches rather than a single nested `isRole('operations_user') || isRole('services_user')` block for readability and to keep the `get()` path unambiguous (`projects/` vs `services/`)
- `created_by == request.auth.uid` pinning present in all three create branches (T-87.3-01-03)
