<objective>
Stop TR generation from silently changing MRF status. Add a dedicated "Rejected MRFs" section in MRF Processing that shows all rejected MRFs (PR Rejected, TR Rejected, Finance Rejected) separate from the Pending MRFs sections.

Purpose: When a TR is submitted, the MRF status changes to 'TR Submitted' — this hides the MRF from the Pending list and misrepresents MRF lifecycle. Per Phase 60 decision, TR actions must not cascade to MRF status. Additionally, rejected MRFs are buried inside the Pending Material/Transport sections with red highlights — they deserve their own dedicated section mirroring the Rejected TRs panel.
Output: TR submission no longer modifies MRF status. Rejected MRFs appear in a dedicated panel below the Pending sections.
</objective>

// Rejected MRFs section (PR Rejected, TR Rejected, Finance Rejected)
if (allRejectedMRFs.length > 0) {

            <div class="mrf-item" data-mrf-id="${mrf.id}" onclick="window.selectMRF('${mrf.id}', this)"
                 style="border: 2px solid #dc2626; background: #fee2e2; border-left: 4px solid ${urgencyLevelColor};">
                <div style="display: flex; justify-content: space-between; align-items: center; margin-bottom: 0.25rem;">
                    <span style="font-weight: 600;">${escapeHTML(mrf.mrf_id)}</span>
                    <span style="background: #fef2f2; color: #dc2626; padding: 0.125rem 0.5rem; border-radius: 4px; font-size: 0.7rem; font-weight: 600;">${statusLabel}</span>
                </div>
                <div style="font-size: 0.875rem; color: #5f6368;">${escapeHTML(getMRFLabel(mrf))}</div>
                <div style="margin-top: 0.5rem; padding: 0.5rem; background: white; border-radius: 4px; font-size: 0.75rem; color: #dc2626;">
