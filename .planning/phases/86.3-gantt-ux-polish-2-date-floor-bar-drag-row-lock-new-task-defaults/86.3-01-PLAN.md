<objective>
Fix the shared root cause of two defects (D-06 row position preservation, D-07 bar drag row-lock) by changing the `walk()` function inside `renderGantt` so children are ordered by `row_order` instead of `start_date`. This is the SAME sort key the left-rail grid already uses (`renderTaskGrid` line 248), so the Gantt and the grid will agree on row order. With the sort key no longer being the field changed by drag (`start_date`), `gantt.refresh(frappeTasks)` no longer reorders bars on horizontal drag, which fixes D-07. With the sort key no longer being the field auto-changed on insert by Firestore latency, the new row stays at its inserted `row_order`, which fixes D-06.

Purpose: PATTERNS.md §D-06 / §D-07 identifies the `walk()` sort as the probable shared root cause for both row-shift on drag AND insert-position resort. CONTEXT.md D-07 explicitly directs investigators to "fix at the source" rather than layering compensating logic. This is the smallest, highest-leverage fix in the phase — one line of code closes two defects.

Output: One-line sort-key change in `walk()`. Verification step that confirms `initGanttDragLink` (Phase 86.1 D-08) is untouched.
</objective>
