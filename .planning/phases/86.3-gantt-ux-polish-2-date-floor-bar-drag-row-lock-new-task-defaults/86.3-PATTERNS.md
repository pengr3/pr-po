### D-01 / D-02 — Soft Date Floor (`min(today, earliest task start_date)`, scroll-clamp only)

**Defect:** Leftmost reachable date = `min(today, min(tasks.start_date))`. User cannot scroll left of that floor. No data write side effect.

**Surface:** `app/views/project-plan.js` — integrate next to today-line / auto-scroll infrastructure.

**Analog 1 — Today-line SVG overlay** (`project-plan.js:1161–1192`):

```javascript
function renderTodayLine() {
    const ganttSvg = document.querySelector('#ganttPane svg');
    if (!ganttSvg) return;
    // Remove old line
    ganttSvg.querySelectorAll('.gantt-today-line').forEach(el => el.remove());
    if (!gantt) return;
    try {
        const startDate = gantt.gantt_start instanceof Date ? gantt.gantt_start : new Date(gantt.gantt_start);
        const today = new Date();
        today.setHours(0, 0, 0, 0);
        startDate.setHours(0, 0, 0, 0);
        const dayDiff = Math.round((today - startDate) / (1000 * 60 * 60 * 24));
        const stepHours = gantt.options.step || 24;
        const colWidth = gantt.options.column_width || 30;
        const xPerDay = (24 / stepHours) * colWidth;
        const x = dayDiff * xPerDay;
        const ns = 'http://www.w3.org/2000/svg';
        const line = document.createElementNS(ns, 'line');
        line.setAttribute('class', 'gantt-today-line');
        line.setAttribute('x1', x);
        line.setAttribute('y1', 0);
        line.setAttribute('x2', x);
        line.setAttribute('y2', '100%');
        ganttSvg.appendChild(line);
    } catch (e) { /* swallow */ }
}
```

**Analog 2 — `scrollGanttToToday()`** (`project-plan.js:1194–1210`) — same x-coordinate math the floor must reuse:

```javascript
function scrollGanttToToday() {
    const pane = document.querySelector('.gantt-pane');
    if (!pane || !gantt) return;
    try {
        const startDate = gantt.gantt_start instanceof Date ? gantt.gantt_start : new Date(gantt.gantt_start);
        const today = new Date();
        today.setHours(0, 0, 0, 0);
        startDate.setHours(0, 0, 0, 0);
        const dayDiff = Math.round((today - startDate) / (1000 * 60 * 60 * 24));
        const stepHours = gantt.options.step || 24;
        const colWidth = gantt.options.column_width || 30;
        const xPerDay = (24 / stepHours) * colWidth;
        const x = dayDiff * xPerDay;
        pane.scrollLeft = Math.max(0, x - pane.clientWidth / 2);
    } catch (e) { /* swallow */ }
}
```

**Pattern to copy:** Convert an ISO date to a Gantt x-coordinate using `gantt.gantt_start`, `gantt.options.step`, `gantt.options.column_width`. Compute `dayDiff` between target date and `gantt_start`, then `x = dayDiff * (24/step) * column_width`.

1. Compute `floorDate = min(today, min(tasks.filter(t => t.start_date).map(t => t.start_date)))` — empty / future projects → today.
2. Convert `floorDate` → `floorX` using the same math as `scrollGanttToToday()`.
3. Attach a scroll-clamp listener to `.gantt-pane` (the element that owns `overflow: auto` per `views.css:2587`):
   ```javascript

       if (pane.scrollLeft < floorX) pane.scrollLeft = floorX;
   };
   pane.addEventListener('scroll', _ganttScrollClampHandler);
   ```
4. **destroy() registration mandatory** — see "Listener cleanup in destroy()" shared pattern below. Add module-scope `let _ganttScrollClampHandler = null;` and clear in `destroy()` mirroring the Phase 86.2 `_resizeMouseMoveHandler` precedent (`project-plan.js:193–197`).

**No write-back to Firestore.** Per D-02 the floor is purely a scroll-bound constraint; tasks keep their original `start_date` / `end_date`.

### D-07 — Bar Row-Lock During Drag

**Defect:** Bars never change rows when dragged; horizontal drag only repositions in time on the same row.

**Surface:** `app/views/project-plan.js` — investigate `handleGanttDateChange` and `gantt.refresh` flow.

**Analog — `handleGanttDateChange`** (`project-plan.js:1067–1104`):

```javascript
function handleGanttDateChange(task, start, end) {
    // task.id === task_id; start/end are Date objects from Frappe
    const t = tasks.find(x => x.task_id === task.id);
    if (!t) return;

    // Defense-in-depth: parent bars MUST NOT be draggable (D-11).
    const isParent = tasks.some(x => x.parent_task_id === t.task_id);
    if (isParent) {
        showToast('Parent task dates are computed from subtasks. Edit subtask dates instead.', 'warning');
        renderGantt();
        return;
    }

    const newStart = formatDateISO(start);
    const newEnd = formatDateISO(end);
    if (newStart === t.start_date && newEnd === t.end_date) return;

    updateDoc(doc(db, 'project_tasks', t.task_id), {
        start_date: newStart,
        end_date: newEnd,
        updated_at: serverTimestamp()
    }).then(async () => {
        if (t.parent_task_id) {
            const idx = tasks.findIndex(x => x.task_id === t.task_id);
            if (idx >= 0) tasks[idx] = { ...tasks[idx], start_date: newStart, end_date: newEnd };
            await recomputeParentDates(t.parent_task_id);
        }
    }).catch(err => {
        console.error('[ProjectPlan] Drag write failed:', err);
        showToast(err?.code === 'permission-denied'
            ? `You don't have permission to edit tasks on this project.`
            : 'Could not save task. Please try again.', 'error');
        renderGantt(); // revert
    });
}
```

**Analog — `gantt.refresh(frappeTasks)` post-snapshot** (`project-plan.js:1020–1024`):

```javascript
// 4. Initialize or refresh Frappe Gantt
if (gantt) {
    // Refresh rather than rebuild — preserves zoom + scroll
    try { gantt.refresh(frappeTasks); } catch (e) { /* fall through to rebuild */ gantt = null; }

```

**Investigation pattern:** `handleGanttDateChange` writes ONLY `start_date` / `end_date` / `updated_at` — no `parent_task_id`, no `row_order`, no other field. So the application code never relocates rows on drag. The row-shift, if observed, originates from one of:

1. **Frappe core's own drag handler** — Frappe v1.2.2 may shuffle bars based on dependency arrows or visual collision. Researcher should test with a no-dep task.
2. **DOM reorder during `gantt.refresh(frappeTasks)`** — the `frappeTasks` array order is determined by the depth-first `walk()` (line 1004), which currently sorts kids by `start_date` (see D-06). When start_date changes via drag, the next refresh re-orders the array, and Frappe redraws bars in the new order. **This is the probable root cause and shares the fix with D-06.**
3. **No custom drag handler in our code** — confirmed by grep: only `handleGanttDateChange` is wired to `on_date_change`.

**Fix candidate (shared with D-06):** Change `walk()`'s sort key from `start_date` to `row_order`. Then `gantt.refresh(frappeTasks)` receives a stable array and bars stay on their rows regardless of date drag.

**No code change pre-locked here** — D-10 is explicitly "diagnose first" per CONTEXT.md.

## Shared Patterns

### Listener Cleanup in `destroy()`

**Source:** `app/views/project-plan.js:145–198` (full `destroy()` block)
**Apply to:** All new event listeners added in 86.3 (especially D-01 scroll-clamp handler).

Every new module-scope handler MUST follow this exact lifecycle:

1. **Declare at module scope** as `let _newHandler = null;` near other handler vars (e.g., `project-plan.js:33–38` block).
2. **Wire in `init()` or in a sub-init function called from `init()`** (e.g., `initPanelResize()` is called from `init()` line 139).
3. **Unwire in `destroy()`** with the existing pattern:
   ```javascript
   if (_newHandler) targetEl.removeEventListener('event', _newHandler);
   _newHandler = null;
   ```

**Reference precedent — Phase 86.2 resize handlers** (`project-plan.js:35–38`, `193–197`):

let _resizeMouseUpHandler = null;
