### M-01: Plan 04 scroll-clamp `passive: true`
**Issue:** Scroll-clamp listener registered with `{ passive: true }`. Passive listeners run post-composite, so the user sees one frame of over-scrolled position before snap-back.
**Fix:** Remove `passive: true` from the listener registration in `installGanttScrollClamp`. Handler is O(1), no benefit to passive.
**Impact:** Visual hitch on the floor edge — minor. Noticeable only if you're looking for it.
result: [pending — fix vs defer]

**Impact:** Tasks inserted via context menu get no Gantt bar until user enters Duration — inconsistent with D-04 goal.
