## Date floor — projects with tasks scheduled before today (defect #1)

| Option | Description | Selected |
|--------|-------------|----------|
| Hard floor at today — historical bars off-screen-left, inaccessible | Simplest. But existing tasks scheduled before today become invisible until you scroll to them — and you said no left-scroll past today, so they'd be permanently hidden. Risky for in-flight projects. | |
| Floor at min(today, earliest task start_date) — today is the leftmost UNLESS tasks exist before today, in which case the earliest task is the floor | Recommended. New/future projects: floor = today. In-progress projects with past tasks: you can still see them. Scrollable left only as far as the earliest existing task. | ✓ |
| Floor at project's earliest task always — today is just where auto-scroll lands | Most permissive. The 'no scroll past today' rule effectively becomes 'auto-scroll lands on today'. Past dates remain reachable always. | |

**User's choice:** Floor at min(today, earliest task start_date)
**Notes:** Soft floor preserves access to historical task bars on in-flight projects while still preventing pointless left-scroll on new/empty projects. Floor enforced at the scroll-bound level — tasks keep their actual dates in Firestore.

## New-task default duration anchor (defect #7)

| Option | Description | Selected |
|--------|-------------|----------|
| start = today, end = today (1d, inclusive) | Simplest. Every new task lands on today regardless of context. | |
| start = today, BUT if the row above has a start_date, use that as the anchor | Tasks naturally cluster near where you're working in the grid. New row at the bottom of an in-progress project doesn't drop to today — it lands near the prior task. | ✓ |
| start = end_date of last task above + 1 day (auto-chain) | Sequential project model — new tasks queue up after the previous one. Aggressive default; could surprise users who don't want chaining. | |

**User's choice:** start = today, BUT if the row above has a start_date, use that as the anchor
**Notes:** Reuses Phase 86.2 DEFECT-6 `sortedForInherit[last]` to find the row above. End_date = start_date (1-day inclusive, matching 86.1 D-03 `addDays(start, days - 1)` convention). Falls back to today when row-above has no start_date OR new row is the first row in the project.
