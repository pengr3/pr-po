## D-03 Implementation Notes

- **Overlay is DOM, not Frappe option override** — Phase 86 D-05 honored. `renderGanttHeaderOverlay()` creates `div.gantt-header-overlay` as a child of `.gantt-pane` (sibling to the Frappe SVG), not a Frappe constructor option. The `new Gantt(...)` option literal at line 1050 is untouched.
- **Overlay re-applied on view-mode change** — `setGanttZoom()` calls `renderGanttHeaderOverlay()` immediately after `installGanttScrollClamp()`. Each call removes any prior `.gantt-header-overlay` before creating a new one (idempotent cleanup).
- **pointer-events: none** — `.gantt-header-overlay { pointer-events: none }` ensures the overlay div never intercepts mouse events destined for Frappe SVG bars. Bar clicks, drag handles, and FS-link gestures are unaffected.
- **destroy() defensive cleanup** — `document.querySelectorAll('.gantt-header-overlay').forEach(el => el.remove())` added to `destroy()` immediately after Phase 86.3 D-01 scroll-clamp cleanup. Prevents overlay leaks on rapid view navigation.
- **Silent-fail** — `renderGanttHeaderOverlay()` wraps all math in try/catch and logs `console.warn` only; never throws to interrupt Gantt rendering.

## Decisions Made

- Cross-month week-range labels use `DD - DD MMM` format (e.g., `31 - 06 Jun`) per CONTEXT.md D-03 literal examples — the leading day comes from the prior month whose name the parent month-name row already displays.
- Day-of-week letters use Monday-first index `(jsDow + 6) % 7` mapping to `['M', 'T', 'W', 'Th', 'F', 'S', 'S']`.
- Week-view first Monday computed as `((1 - firstMonday.getDay()) + 7) % 7` days after `gantt_start` — handles gantt_start falling mid-week.
