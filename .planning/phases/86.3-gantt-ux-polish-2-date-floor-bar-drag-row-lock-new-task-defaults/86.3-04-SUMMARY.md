## D-11 Diagnosis Note

D-11 (scroll jank) called for live diagnosis via Chrome DevTools Performance recording, which is not available during overnight batch execution.

Applied Root Cause A from PATTERNS.md §D-11 — the recommended default. Hypothesis: this plan's NEW scroll-clamp listener (Task 1) recomputed `floorX` on every scroll event, iterating over `tasks[]` per fire. For projects with 100+ tasks, that O(n) work in the hot path causes visible jank.

Patch: cache `floorX` once at install time inside the closure. Re-installation on snapshot (renderGantt step 8) and view-mode change (setGanttZoom) keeps the cached value current — so the on-scroll handler is now O(1).

Two-commit approach preserves auditability: `11c7804` is the "naive" Task 1 implementation; `c9eceeb` is the isolated Root Cause A fix. A future debugger can revert `c9eceeb` without losing floor functionality.

**If the morning UAT (Test 7 Performance trace) still shows long tasks > 16ms during pure scroll, the root cause is NOT A.** Re-investigate via DevTools and consider PATTERNS.md candidates:
- B: snapshot loop (check for write-triggered re-snapshot during scroll)
- C: ad-hoc `renderGantt` call site outside snapshot listener
- D: today-line teardown leaking SVG nodes after `gantt.refresh()`

## D-02 Confirmation

No `setDoc` / `updateDoc` / `addDoc` / `updateField` call was added in this plan. The floor constraint mutates only `pane.scrollLeft` (DOM, not Firestore). `computeGanttFloorX()` reads the in-memory `tasks[]` array already populated by the existing `onSnapshot` listener — zero additional Firestore RPCs during scroll.

## Decisions Made

- **D-11 Root Cause A (cached closure)** — chosen as the batch-mode default per plan guidance. The new scroll-clamp listener (Task 1) was the only new O(n) work introduced in this phase, making it the most likely jank source.
- **Two-commit approach** — Tasks 1 and 2 are separate atomic commits so the D-11 fix is independently revertable in `git log`.
- **Fail-open on errors** — `computeGanttFloorX()` returns `0` on any exception, ensuring no clamp is applied rather than blocking all scroll.
