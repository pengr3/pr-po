# Plan 92.2-01 — Execution Summary

## Goal
Mirror the Phase 92 scorecard pattern from `app/views/projects.js` into
`app/views/services.js`, replacing the `#serviceProjectStatusFilter` dropdown
with an interactive status scorecard strip.

## Changes (all in `app/views/services.js`)

### Module state and constants
- Added module-scope `let activeServiceStatusFilter = null;` (drives status filter).
- Added `const SCORECARD_STATUS_OPTIONS = UNIFIED_STATUS_OPTIONS.filter(s => s !== 'Draft');`.
  `UNIFIED_STATUS_OPTIONS` was left untouched (still used by Add/Edit form dropdowns).

### `attachWindowFunctions()`
- Wired `window.handleServiceScorecardClick = handleServiceScorecardClick;`.

### `render()` HTML
- Inserted `<div id="serviceScorecards" class="project-scorecards">` strip
  immediately after `#addServiceForm`. Strip uses the existing Phase 92 CSS
  classes (`project-scorecard-card`, `project-scorecard-card--total`,
  `project-scorecard-card--active`, `scorecard-label`, `scorecard-count`).
- Removed the Status `form-group` containing `#serviceProjectStatusFilter`
  from the filter bar. Client and Search filters remain.

### New functions
- `renderServiceScorecards(baseServices)` — updates per-status counts, total
  count, and the active highlight. Counts are computed from the
  sub-tab + assignment-scoped pool so toggling a status card never changes
  its own count.
- `handleServiceScorecardClick(status)` — toggles `activeServiceStatusFilter`
  (clicking the active card or the Total card clears it) and calls
  `applyServiceFilters()`.

### `applyServiceFilters()` refactor
- Dropped the `projectStatusFilter` DOM read.
- Built `scopedServicesForTab` first (filtered by `service_type` and
  `assignedCodes`).
- Filtered `filteredServices` from `scopedServicesForTab` using only
  search + scorecard status state + client filters.
- After `renderServicesTable()`, called
  `renderServiceScorecards(scopedServicesForTab)`.

### `init()` and `destroy()`
- `init()`: reset `activeServiceStatusFilter = null`; attached
  `window._serviceScorecardClickOutside` document click handler that clears
  the active scorecard when clicking outside the strip.
- `destroy()`: tore down the click-outside listener, reset
  `activeServiceStatusFilter`, and added
  `delete window.handleServiceScorecardClick;` to the existing teardown
  block.

### Dead code removal
- Removed the now-orphaned `rebuildServiceStatusFilterOptions()` helper and
  its call in `renderServicesTable()` (the dropdown it rebuilt no longer
  exists). Replaced with a short explanatory comment.

## Behavior delta for the user
- Old: Status `<select>` dropdown in the filter bar (text-based).
- New: Horizontal scorecard strip above filters showing live counts per
  status plus a Total card. Clicking a card filters the table to that
  status; clicking it again (or the Total card, or anywhere outside the
  strip) clears the filter. Counts always reflect the current sub-tab
  (one-time vs recurring) and `services_user` assignment scope.
