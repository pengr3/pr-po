## Decisions Made

- Chart.js v4.4.7 pinned (latest 4.x patch as of plan creation); chart.umd.min.js UMD build required for window.Chart global — no bundler available in zero-build SPA
- chartInstances Map (not object) for O(1) get/set and clean forEach+clear semantics
- animation: false for home page performance — avoids animation overhead on every onSnapshot update
- One Chart instance per section (6 total: 2 in Projects, 4 in Services) — each section has independent data and sizing
- Zero-count bars shown as zero-length bars (not hidden) — consistent axis alignment and no "disappearing bars" confusion
- getChartSizeClass() uses containerId suffix (-internal → 110px, -project → 160px) — auto-selects correct height without explicit parameter
