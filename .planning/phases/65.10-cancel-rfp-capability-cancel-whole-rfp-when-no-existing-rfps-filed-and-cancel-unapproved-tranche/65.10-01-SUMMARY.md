# Phase 65.10 Plan 01: Cancel RFP Capability Summary

**One-liner:** RFP cancel-and-refile flow: zero-payment guard + deleteDoc + re-open modal with pre-filled invoice, due date, payment mode, and bank details for correction.

## What Was Built

**Firestore rules (Task 1):**
- Changed `rfps` collection delete rule from `super_admin` only to `['super_admin', 'procurement']`
- Enables procurement users to delete zero-payment RFP documents for cancel-and-refile

**procurement.js (Task 2):**

Three new module-level functions inserted before `showRFPContextMenu`:

1. `isRFPCancellable(rfp)` ŌĆö filters `payment_records` for non-voided entries and sums amounts; returns `true` only when total is 0. Used as a guard in `cancelRFPDocument` and to filter context menu items.

2. `prefillRFPForm(savedData)` ŌĆö sets values on: `rfpInvoiceNumber`, `rfpDueDate`, `rfpPaymentMode`, `rfpBankName`, `rfpBankAccountName`, `rfpBankDetails`, `rfpAltBankName`, `rfpAltBankAccountName`, `rfpAltBankDetails`, `rfpPaymentModeOther`, `rfpTrancheSelect`. Calls `toggleRFPBankFields()` after setting payment mode, and `showAltBank()` when alt bank data is present.

3. `cancelRFPDocument(rfpDocId)` ŌĆö async handler that: removes context menu, finds rfp in rfpsData, guards with `isRFPCancellable`, confirms with user, captures all relevant fields into `savedData`, calls `deleteDoc`, shows success toast, then re-opens the appropriate modal (TR RFP / Delivery Fee RFP / Regular PO RFP) and calls `prefillRFPForm(savedData)`.

**Context menu extensions:**
- `showRFPContextMenu`: computes `cancellableRFPs = existingRFPs.filter(isRFPCancellable)` and appends red Cancel items with divider separator when list is non-empty
- `showTRRFPContextMenu`: finds TR via `poData.flatMap`, gets `cancellableTRRFP = existingTRRFPs.find(isRFPCancellable)`, appends single red Cancel item with divider when found

**Window registration and cleanup:**
- `window.cancelRFPDocument = cancelRFPDocument` added in `init()`
- `delete window.cancelRFPDocument` added in `destroy()`
