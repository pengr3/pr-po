# Quick Task 260430-a4b: codeless projects/services personnel_user_ids MRF filter

## Background

`rebuildPSOptions()` in `mrf-form.js` filters projects and services for scoped roles. When `assignedCodes !== null`
(i.e., `operations_user` with specific codes, or `services_user` with specific codes), the filter
`assignedCodes.includes(p.project_code)` silently drops codeless entries because `includes(null)` is always false.

`syncPersonnelToAssignments()` deliberately skips codeless projects (line 643: `if (!projectCode) return`).
So a user added to a codeless project's `personnel_user_ids` never gets it reflected in `assigned_project_codes`.
The MRF dropdown must compensate by checking `personnel_user_ids` directly.

Prior debug sessions: blanket `!p.project_code ||` was reverted for being too broad. The correct fix scopes by uid.

## Task 1 — Extend projects filter in rebuildPSOptions()

**Files:** `app/views/mrf-form.js`

**Action:** In `rebuildPSOptions()`, update the projects filter (currently ~line 1208–1213).

Current:
```javascript
if (assignedCodes !== null) {
    // Phase 78 D-02: clientless projects (no project_code) are not in assigned_project_codes.
    // Users with all_projects:true get assignedCodes===null (above), so they see everything.
    // Users with specific codes only see their coded projects — codeless projects are excluded.
    projects = cachedProjects.filter(p => assignedCodes.includes(p.project_code));
}
```

New:
```javascript
if (assignedCodes !== null) {
    const uid = window.getCurrentUser?.()?.uid;
    // Coded projects: include if in assignedCodes.
    // Codeless projects: include if user is in personnel_user_ids
    // (syncPersonnelToAssignments is skipped for codeless projects — no code to sync).
    projects = cachedProjects.filter(p =>
        assignedCodes.includes(p.project_code) ||
        (!p.project_code && uid && (p.personnel_user_ids || []).includes(uid))
    );
}
```

**Verify:** `cachedProjects` contains codeless project with personnel_user_ids=['user123']; `getAssignedProjectCodes()` returns ['CODE-A'] (operations_user); getCurrentUser().uid = 'user123'. After filter: codeless project IS included; other unassigned coded projects NOT included.

**Done:** `rebuildPSOptions()` projects section updated

```javascript
    );

```

**Verify:** Symmetric with projects — codeless services with user in personnel_user_ids pass through; coded unassigned services do not.
