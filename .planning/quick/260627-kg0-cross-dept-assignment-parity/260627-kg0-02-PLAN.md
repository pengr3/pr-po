Helper (union semantics, no-leak-preserving):
  function isMrfInAssignedScope(mrf) {
    const projScope = window.getAssignedProjectCodes?.();   // null = no project filter
    const svcScope  = window.getAssignedServiceCodes?.();   // null = no service filter
    // Exempt roles: BOTH null -> no filtering at all.
    if (projScope === null && svcScope === null) return true;
    // Union: visible if it matches the user's assigned projects OR their assigned services.
    // NO code-equals-null escape term — an uncoded/cross-dept item matches neither scope and stays hidden.
    const matchesProject = projScope !== null && projScope.includes(mrf.project_code);
    const matchesService = svcScope  !== null && svcScope.includes(mrf.service_code);
    return matchesProject || matchesService;
  }
KEY CORRECTNESS POINT (why this is OR, not AND): today a non-exempt user has exactly one non-null
scope, so projScope.includes(project_code) OR svcScope.includes(service_code) reduces to that one
scope — behavior for single-dept users is IDENTICAL to the old code. When a cross-dept user has
BOTH scopes non-null, OR is the correct union (their own dept's items survive). The old sequential
two-`if` filtering was an implicit AND that would have dropped own-dept items — that is the bug
this task fixes.
