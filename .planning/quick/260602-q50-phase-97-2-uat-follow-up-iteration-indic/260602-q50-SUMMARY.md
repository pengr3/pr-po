## Functions Added / Modified

### Added — `applyLastLoadedFromProject()` (~line 3376)
- Synchronous helper called in `init()` between `loadIterations()` and `renderIterRail()`
- Reads `currentProject.last_loaded_iteration_id`; finds matching entry in `_iterations`
- Sets `_loadedIterationId` and `_loadedIterationLabel` so `renderIterRail()` paints the strip + badge on first render
- If the stored id is stale (iteration deleted or is auto-only), fires a fire-and-forget `updateDoc(..., { last_loaded_iteration_id: null })` to clean up

### Added — `async function overwriteLoadedIteration()` (~line 3398)
- Builds same 17-field snapshot as `saveIteration(label)` (verbatim copy)
- Calls `updateDoc(doc(db, 'project_iterations', _loadedIterationId), { tasks, saved_at })` — does NOT mutate `label`, `project_id`, or `auto`
- After success: `loadIterations()` + `renderIterRail()` + `showToast('Iteration "X" updated.', 'success')`
- Handles `_pendingLoadAfterSave` chain identically to `saveIteration` success path
- Error path: resets `_pendingLoadAfterSave = null`, logs, toasts error

### Modified — `saveIteration(label = null)` (null-label branch)
- Added short-circuit at top of null-label branch: `if (_loadedIterationId) return overwriteLoadedIteration();`
- Modal (reached only when no named iteration loaded) heading changed to `Save Iteration As`
- Confirm button text changed to `Save As`

### Modified — `restoreIteration(iterationId)` (after batch-commit, before loadIterations)
- Added `try { await updateDoc(doc(db, 'projects', currentProject.id), { last_loaded_iteration_id: iter.auto ? null : iterationId }) } catch (persistErr) { console.error(...); }` block
- Non-fatal — in-memory `_loadedIterationId` / `_loadedIterationLabel` already correct at this point

### Modified — `init()` (~line 372-374)
- Inserted `applyLastLoadedFromProject();` between `await loadIterations();` and `renderIterRail();`

## What Is Written to the Project Doc on First Overwrite

On the first `restoreIteration(iterationId)` success after this fix, Firestore receives:
```
updateDoc(doc(db, 'projects', <currentProject.id>), {
    last_loaded_iteration_id: <iterationId>   // or null for auto-snapshot loads
})
```
This write is **non-fatal** — page renders correctly even if it fails (in-memory state wins).
