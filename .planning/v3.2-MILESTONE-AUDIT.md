## Spec-Code Mismatches (Require Decision)

These were intentional design/scope changes made after verification — the spec was never updated to match.

### 1. RFPBANK-01/02 — Saved bank selector removed (`app/views/procurement.js`)
**Spec:** Button shows dropdown of unique bank accounts from past RFPs; selecting auto-fills fields.  
**Current code:** `showAltBank`/`removeAltBank` — manual entry of alternative bank. No saved-account dropdown.  
**Commit:** `b613ca4` "Remove saved bank account selector feature (not needed)"  
**Decision:** Amend RFPBANK-01/02 spec to describe the alt-bank entry UX (preferred), or restore the dropdown.

### 2. POBAR-01/02/03 — Badge gradient replaces flush bar (`app/views/procurement.js:4992-4996`)
**Spec:** Separate 3px flush progress bar rendered below each PO badge.  
**Current code:** `background: linear-gradient(to right, ${color} ${pct}%, ...)` applied to the badge chip itself.  
**Commit:** `cd47790` "replace status badge with pure payment progress bar"  
**Decision:** Amend POBAR-01/02/03 to describe gradient-fill-inside-badge design.

### 3. EXPMOD-02 — Delivery Fees column reverted (`app/expense-modal.js:586,589`)
**Spec:** "Delivery Fees table shows Supplier instead of PO ID"  
**Current code:** `<th>PO ID</th>` / `<td>${item.po_id}</td>`  
**Commit:** `32a18f9` "revert Delivery Fees to PO ID column"  
**Decision:** Amend EXPMOD-02 to read "Delivery Fees table shows PO ID | Amount". The supplier data is available at `item.supplier` if the column is ever needed.
