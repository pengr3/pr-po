# Spike 024: Billing Request Flow — VALIDATED ✓

Design is fully locked. This spike is complete. The next step is planning the real implementation.

<current_state>
Spike 024 VALIDATED and pushed to origin/v3.3.

The spike explored how project users (operations_user) can initiate billing from the project-detail page, when they currently can't access Finance > Collectibles. Two design surfaces were prototyped and user-selected:

1. Project Detail surface (Part 1): **Option C — footer link** — "↑ Initiate Billing →" text link at the bottom of the Collectibles section in the Financial Summary card.
2. Finance queue surface (Part 2): **Option A — collapsible banner** — blue "Pending Billing Requests" section that auto-appears above the Collectibles table and collapses when empty.
</current_state>

<decisions_made>
- **Workflow**: project user submits billing_request (not direct collectible creation) → Finance reviews + approves → Finance creates the collectible via existing modal
- **Authority**: operations_user does NOT get collectible write authority; the billing_requests collection is their surface
- **Billing type determination**: user-selected via pill UI (Progress / Completion / Other), with auto-hint from tranche label (label contains "completion" → pre-selects Completion pill). NOT inferred purely from label — user can override.
- **Project Detail surface**: footer link in Financial Summary card's Collectibles section (Option C) — zero card expansion
- **Finance queue surface**: collapsible blue banner above Collectibles table (Option A) — inline, auto-hides when empty
- **Document attachment**: URL/link per document (Google Drive, SharePoint etc.) — no Firebase Storage (avoids Blaze plan requirement)
- **Approve bridge**: Approve → calls `openCreateCollectibleModal('projects:CODE:TRANCHE_INDEX')` — extend preselectKey to include tranche_index so approve truly pre-fills all fields
</decisions_made>

## Proposed Firestore Schema

```
billing_requests/{id}:
  project_code: string          // e.g. 'CLMC-ACME-001'
  project_name: string
  tranche_index: number         // position in collection_tranches[]
  tranche_label: string         // frozen at submission
  tranche_percentage: number    // frozen at submission
  amount_requested: number      // contract_cost * percentage / 100
  billing_type: 'progress' | 'completion' | 'other'
  documents: [{ key, label, url }]
  notes: string
  status: 'pending' | 'approved' | 'rejected'
  requested_by_uid: string
  requested_by_name: string
  requested_at: Timestamp
  reviewed_by: string           // on approve/reject
  reviewed_at: Timestamp        // on approve/reject
  rejection_reason: string      // on reject only
```
