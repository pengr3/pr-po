CSS IMPLEMENTATION
/* ── A: full block tint — current production CSS ── */

.nd-row.unread  { background: #eff6ff; }
.nd-row.unread:hover { background: #dbeafe; }

/* Change to JS: class toggle
   row.classList.toggle('unread', !isRead);

 Pros:
   - High contrast, easy to spot unread zone
   - Common pattern (Gmail, Slack, Linear)
   - Hover deepens tint — responsive feel

 Cons:
   - Blue tint competes with badge colors (especially
     blue badges like PROPOSAL_SUBMITTED look subdued)
   - High density of unread rows = very blue panel
   - Tint disappears on read — abrupt, no fade
*/
/* ── B: left-border accent + very pale tint ── */

.nd-row.unread {
  background: #f8fbff;
  border-left: 3px solid #1a73e8;
  padding-left: calc(1rem - 3px); /* compensate */
}
.nd-row.unread:hover { background: #eff6ff; }
.nd-row { border-left: 3px solid transparent; } /* layout anchor */

/* Pros:
   - Indicator occupies its own lane — doesn't tint content
   - Very pale #f8fbff barely changes badge color appearance
   - Border snaps away on read = clean indicator pattern
   - Familiar (GitHub, Notion notification sidebar)

 Cons:
   - 3px leftward nudge shifts content — need transparent border
     on read rows to keep alignment stable
   - border-left doesn't round with border-radius on the row
*/
/* ── C: dot indicator — no background change ── */

.nd-row { padding-left: 1.25rem; position: relative; }

.nd-row.unread::before {
  content: '';
  position: absolute;
  left: 0.375rem;
  top: 50%;
  transform: translateY(-50%);
  width: 6px; height: 6px;
  border-radius: 50%;
  background: #1a73e8;
}

/* Pros:
   - Minimal — no background color change at all
   - Dot disappears on read = clean
   - Badge colors fully unaffected
   - iMessage / macOS Mail pattern

 Cons:
   - Easy to miss — small dot in a dense notification list
   - Padding shift on all rows needed for alignment
   - No hover affordance difference for unread vs read
   - When there are 0 unread rows the extra left padding
     looks slightly wasteful
*/

Observations

  • Variant A (full block): Highest contrast — immediately clear which rows are new. But heavy blue fill can wash out blue-badged types (PROPOSAL_SUBMITTED, PROJECT_STATUS). With 5+ unread items the panel feels very blue and fatiguing.
  • Variant B (left border): Best balance — the border occupies its own visual lane without tinting the badge zone. Pale #f8fbff is barely perceptible but adds warmth. Hover deepening to #eff6ff gives tactile feedback. The transparent-border anchor on read rows keeps content alignment rock-solid.
  • Variant C (dot): Cleanest look — badge colors are 100% unaffected. However the dot is easy to miss at a glance, especially when the bell already has a red badge indicating unread count. The global padding shift adds visual waste when inbox is empty.
  • Recommendation: Variant B (left border + pale tint). It signals "new" clearly without overwhelming the content, plays well with all 16 badge color types, and the padding-compensation pattern is a 2-line CSS fix with zero JS change.
