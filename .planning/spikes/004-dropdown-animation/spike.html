Spike 004a/b — Dropdown Animation

Given the notification dropdown opens via an .open class, which CSS transition feels better — slide (translateY) or scale (scaleY)?

CSS IMPLEMENTATION
/* ── 004a: slide — translate down from -12px ── */

.notif-dropdown {
  opacity: 0;
  transform: translateY(-12px);
  visibility: hidden;
  pointer-events: none;
  transition: transform 200ms ease-out,
              opacity 200ms ease-out,
              visibility 0ms 200ms;
}
.notif-dropdown.open {
  opacity: 1;
  transform: translateY(0);
  visibility: visible;
  pointer-events: auto;
  transition: transform 200ms ease-out,
              opacity 200ms ease-out,
              visibility 0ms 0ms;
}

/* Pros:
 - Content reads top-to-bottom as it enters — feels natural
 - Works regardless of dropdown height
 - No content squash during animation
 - -12px offset is subtle — feels "snappy" not "floaty"

 Cons:
 - Slightly less "originating from the bell" visual metaphor
 - Same opacity fade on all axes (no focal point) */
/* ── 004b: scale — grow from top-right origin ── */

.notif-dropdown {
  opacity: 0;
  transform: scaleY(0);
  transform-origin: top right;
  visibility: hidden;
  pointer-events: none;
  transition: transform 200ms ease-out,
              opacity 200ms ease-out,
              visibility 0ms 200ms;
}
.notif-dropdown.open {
  opacity: 1;
  transform: scaleY(1);
  transform-origin: top right;
  visibility: visible;
  pointer-events: auto;
  transition: transform 200ms ease-out,
              opacity 200ms ease-out,
              visibility 0ms 0ms;
}

/* Pros:
 - Visually "opens from" the bell button — strong affordance
 - Familiar pattern (macOS Notification Center, GitHub bell)

 Cons:
 - scaleY squashes text/rows mid-animation — rows look crushed
 - scaleY(0) collapses height, then rows "snap" into place
 - transform-origin: top right works, but top left may look
   wrong when panel is on the far right edge of a wider page
 - Content animates distorted (not clean entry) */

Observations

  • Slide (004a): Clean, content-respecting entry. Rows appear readable throughout. The -12px offset keeps it subtle — not a long travel distance. Works well with the opacity fade.
  • Scale (004b): The squash effect on rows during scaleY is visually jarring — rows appear distorted before settling. The "opens from bell" metaphor is nice in theory but the content crush undermines it.
  • scaleY fix exists — adding transition: transform 150ms, opacity 150ms and a faster duration reduces the squash window, but doesn't eliminate it without also scaling the children inversely (overly complex).
  • Timing: 150ms feels snappy but abrupt. 250ms feels laggy on repeat opens. 200ms is the sweet spot for this panel size.
  • Easing: ease-out on open (decelerates to final position) + ease-in on close (accelerates away) = most natural feel. Use duration controls to validate.
