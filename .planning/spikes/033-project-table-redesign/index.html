Why this pairing works

  • Priority Feed is the daily briefing — open the page, see what needs your eyes, act. No cognitive work required to find problems.
  • Browse All satisfies the "I want to see everything by stage" mental model — useful for weekly reviews, reporting, or answering "how many proposals do we have out right now?"
  • The toggle is in the toolbar, not the nav — both modes are the same view, same URL, same data. No extra route or tab to maintain.
  • Collapse state on Browse All groups persists to localStorage — "Completed" stays folded across sessions.
  • Urgency signals computed from updated_at + stage entry timestamp — same fields already tracked, no new schema changes needed for the feed.

Implementation notes

  • Default view is Priority Feed — the last-used mode can optionally be persisted to localStorage so users who prefer Browse All don't need to re-toggle each session.
  • Urgency thresholds are configurable constants (e.g. 14d = watch for Client Review, 7d for Revision, 5d no-activity for On-going) — tune after a few weeks of real usage.
  • A project can appear in the feed at "watch" level even while its Browse All group looks fine — this is expected behavior, not a bug.
