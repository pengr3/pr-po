# Requirements: CLMC Procurement System v1.0

**Defined:** 2026-01-25
**Core Value:** Projects tab must work - it's the foundation where project name and code originate, and everything in the procurement system connects to it.

## v1.0 Requirements

Requirements for Core Projects Foundation. Each maps to roadmap phases.

### Clients Management

- [x] **CLIENT-01**: User can create client with client_code, company_name, contact_person, contact_details — ✓ VALIDATED v1.0
- [x] **CLIENT-02**: User can edit existing client information — ✓ VALIDATED v1.0
- [x] **CLIENT-03**: Super Admin can delete clients (v1.0: anyone can delete, role enforcement in v2.0) — ✓ VALIDATED v1.0
- [x] **CLIENT-04**: User can view list of all clients — ✓ VALIDATED v1.0
- [x] **CLIENT-05**: Client code is manually entered by admin (validated for uniqueness) — ✓ VALIDATED v1.0

### Projects Management

- [x] **PROJ-01**: User can create project with auto-generated project code (CLMC_CLIENT_YYYY###) — ✓ VALIDATED v1.0
- [x] **PROJ-02**: Project creation requires project_name and client (dropdown) — ✓ VALIDATED v1.0
- [x] **PROJ-03**: Project creation allows optional: budget, contract_cost, personnel (freetext) — ✓ VALIDATED v1.0
- [x] **PROJ-04**: User can select Internal Status from dropdown (For Inspection, For Proposal, For Internal Approval, Ready to Submit) — ✓ VALIDATED v1.0
- [x] **PROJ-05**: User can select Project Status from dropdown (Pending Client Review, Under Client Review, Approved by Client, For Mobilization, On-going, Completed, Loss) — ✓ VALIDATED v1.0
- [x] **PROJ-06**: User can toggle project active/inactive status — ✓ VALIDATED v1.0
- [x] **PROJ-07**: User can edit existing projects — ✓ VALIDATED v1.0
- [x] **PROJ-08**: Super Admin can delete projects (v1.0: anyone can delete, role enforcement in v2.0) — ✓ VALIDATED v1.0
- [x] **PROJ-09**: User can view full project details by clicking project in list — ✓ VALIDATED v1.0
- [x] **PROJ-10**: User can view project list with columns: Code, Name, Client, Internal Status, Project Status — ✓ VALIDATED v1.0
- [x] **PROJ-11**: User can filter projects by Internal Status — ✓ VALIDATED v1.0
- [x] **PROJ-12**: User can filter projects by Project Status — ✓ VALIDATED v1.0
- [x] **PROJ-13**: User can filter projects by Client — ✓ VALIDATED v1.0
- [x] **PROJ-14**: User can search projects by project code or project name — ✓ VALIDATED v1.0
- [x] **PROJ-15**: Project list displays most recent projects first — ✓ VALIDATED v1.0
- [x] **PROJ-16**: Project create/edit uses new page UI (not modal) with back navigation — ✓ VALIDATED v1.0
- [x] **PROJ-17**: Budget and contract_cost must be positive numbers if provided — ✓ VALIDATED v1.0
- [x] **PROJ-18**: Project code uniqueness is enforced per client per year — ✓ VALIDATED v1.0

### MRF-Project Integration

- [x] **MRF-01**: User can select project from dropdown in MRF form — ✓ VALIDATED v1.0
- [x] **MRF-02**: Project dropdown displays format: "CLMC_CODE_YYYY### - Project Name" — ✓ VALIDATED v1.0
- [x] **MRF-03**: Project dropdown shows only active projects (inactive excluded) — ✓ VALIDATED v1.0
- [x] **MRF-04**: Project dropdown is sorted by most recent first — ✓ VALIDATED v1.0
- [x] **MRF-05**: Project code is stored in MRF document — ✓ VALIDATED v1.0
- [x] **MRF-06**: MRF list displays project code and project name — ✓ VALIDATED v1.0
- [x] **MRF-07**: MRF details view displays project information — ✓ VALIDATED v1.0
- [x] **MRF-08**: User can create MRFs for completed projects (warranty work) — ✓ VALIDATED v1.0
- [x] **MRF-09**: User cannot create MRFs for inactive projects — ✓ VALIDATED v1.0
