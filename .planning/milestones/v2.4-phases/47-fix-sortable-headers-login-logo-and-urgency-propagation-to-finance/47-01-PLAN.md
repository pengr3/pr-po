```javascript
function sortMaterialPRs(column) {
    if (prSortColumn === column) {
        prSortDirection = prSortDirection === 'asc' ? 'desc' : 'asc';
    } else {
        prSortColumn = column;
        prSortDirection = 'asc';

    const filtered = activeDeptFilter
        ? materialPRs.filter(pr => (pr.department || 'projects') === activeDeptFilter)
        : materialPRs;
    filtered.sort((a, b) => {
        let aVal = a[column];
        let bVal = b[column];
        if (aVal == null) return prSortDirection === 'asc' ? 1 : -1;
        if (bVal == null) return prSortDirection === 'asc' ? -1 : 1;
        if (typeof aVal === 'string') {
            return prSortDirection === 'asc'
                ? aVal.localeCompare(bVal) : bVal.localeCompare(aVal);

        return prSortDirection === 'asc' ? aVal - bVal : bVal - aVal;
    });

}
```

IMPORTANT: The sort function sorts the `filtered` array (post dept-filter), but `renderMaterialPRs()` re-applies the dept filter from `materialPRs`. So actually, sort must sort `materialPRs` directly (the source array), not the filtered subset. This matches how `sortPOs()` sorts `poData` directly. Corrected approach:

```javascript
function sortMaterialPRs(column) {

        let aVal = a[column];

                ? aVal.localeCompare(bVal) : bVal.localeCompare(aVal);

```
