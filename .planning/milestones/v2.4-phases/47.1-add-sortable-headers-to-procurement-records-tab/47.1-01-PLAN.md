**Step 3: Add sortPRPORecords() function.**

Place it AFTER `filterPRPORecords()` (after line 2373) and BEFORE the `exportPRPORecordsCSV()` function (line 2375). The sort function:

1. Toggles direction if same column clicked, or sets new column to ascending
2. Sorts `allPRPORecords` (the source array, NOT `filteredPRPORecords`)
3. Calls `filterPRPORecords()` which rebuilds `filteredPRPORecords` from the now-sorted source and calls `renderPRPORecords()`

```javascript
function sortPRPORecords(column) {
    if (prpoSortColumn === column) {
        prpoSortDirection = prpoSortDirection === 'asc' ? 'desc' : 'asc';
    } else {
        prpoSortColumn = column;
        prpoSortDirection = 'asc';
    }
    allPRPORecords.sort((a, b) => {
        let aVal = a[column];
        let bVal = b[column];
        if (aVal == null) return prpoSortDirection === 'asc' ? 1 : -1;
        if (bVal == null) return prpoSortDirection === 'asc' ? -1 : 1;
        if (typeof aVal === 'string') {
            return prpoSortDirection === 'asc'
                ? aVal.localeCompare(bVal) : bVal.localeCompare(aVal);
        }
        return prpoSortDirection === 'asc' ? aVal - bVal : bVal - aVal;
    });
    filterPRPORecords();
}
```

IMPORTANT: Sort `allPRPORecords` (source), NOT `filteredPRPORecords`. `filterPRPORecords()` regenerates `filteredPRPORecords` from `allPRPORecords`, so the sort order on the source array flows through. This matches the pattern documented in STATE.md decision [Phase 47-01].
