  <name>Task 1: Add text-decoration: none and font: inherit to components.css</name>
  <files>styles/components.css</files>
  <action>
Make two targeted edits to `styles/components.css`:

**Edit 1 — `.tab-btn` / `.tab-button` rule (around lines 963-974):**
Add `text-decoration: none;` as the last property in the rule block. This prevents `<a class="tab-btn">` elements (used in Finance and Procurement tabs) from showing browser-default link underlines. `<button class="tab-btn">` elements are unaffected by this property.

After the edit the rule should look like:
```css
.tab-btn,
.tab-button {
    padding: 0.75rem 1.5rem;
    background: transparent;
    border: none;
    border-bottom: 3px solid transparent;
    cursor: pointer;
    font-size: 1rem;
    color: var(--gray-700);
    transition: all 0.2s;
    font-weight: 500;
    text-decoration: none;
}
```

**Edit 2 — `.nav-dropdown-trigger` rule (around lines 87-92):**
Add `font: inherit;` and `line-height: inherit;` and remove the stale CSS comment `/* Inherits .nav-link styles */` (it is misleading — CSS class inheritance does not work that way). The `font: inherit` shorthand overrides the browser user-agent stylesheet which sets `font-family: initial; font-size: initial` on `<button>` elements, causing the Admin button to render with a different font than `<a class="nav-link">` items.

After the edit the rule should look like:
```css
.nav-dropdown-trigger {
    background: none;
    border: none;
    cursor: pointer;
    font: inherit;
    line-height: inherit;
}
```
