<objective>
Fix the service edit history path defect identified in the v2.3 milestone audit.

Purpose: Service edits currently record audit entries under projects/{docId}/edit_history due to hardcoded collection names in edit-history.js. This silently corrupts the audit trail — history is written to the wrong document path and is unrecoverable from the service detail view. SERV-09 requires the path services/{docId}/edit_history; SERV-04 requires the Edit History button/modal to work on service detail.

Output: edit-history.js gains an optional collectionName parameter (backward-compatible). service-detail.js gains the Edit History button and corrected call sites. services.js gains corrected call sites. firestore.rules gains the services/edit_history subcollection rule. Rules are deployed to production.
</objective>
