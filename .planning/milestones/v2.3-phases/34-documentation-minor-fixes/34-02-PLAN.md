  <action>
Read finance.js and locate the Tab 2 card-header block at approximately lines 717-720. The current content is:

```javascript
<div class="card-header">
    <h2>Recently Generated Purchase Orders</h2>
    <button class="btn btn-secondary" onclick="window.refreshPOs()">🔄 Refresh</button>
</div>
```

Replace this block with:

```javascript
<div class="card-header">
    <h2>Recently Generated Purchase Orders</h2>
    <div style="display:flex;gap:0.5rem;align-items:center;">
        <select id="deptFilterPOs"
                onchange="window.applyFinanceDeptFilter(this.value)"
                style="padding:0.35rem 0.6rem;border:1.5px solid #e2e8f0;border-radius:6px;font-size:0.875rem;color:#475569;">
            <option value="">All Departments</option>
            <option value="projects">Projects</option>
            <option value="services">Services</option>
        </select>
        <button class="btn btn-secondary" onclick="window.refreshPOs()">🔄 Refresh</button>
    </div>
</div>
```

This is a template literal string change inside the render() function of finance.js. The card-header is inside the `<!-- Tab 2: Purchase Orders -->` section block (the `<section id="pos-section" ...>` element).

**Critical implementation notes:**
- Use id="deptFilterPOs" — NOT "deptFilterApprovals" (Tab 1 uses that id; using the same id would create duplicate DOM ids)
- The onchange handler calls `window.applyFinanceDeptFilter(this.value)` — this function is already registered on window inside attachWindowFunctions() at line ~133. Do NOT add another window registration.
- The <select> style is copied exactly from the Tab 1 dropdown (lines 651-657 in finance.js) for visual consistency
- The Refresh button must remain and keep its existing onclick="window.refreshPOs()" — only wrap it in the flex div alongside the new select
- No JavaScript changes are needed — renderPOs() already reads activeDeptFilter at lines 2021-2023; applyFinanceDeptFilter() already calls renderPOs() at line ~90

**Side effect awareness (by design):**
applyFinanceDeptFilter() sets activeDeptFilter and calls renderMaterialPRs(), renderTransportRequests(), AND renderPOs(). The Tab 2 dropdown will therefore also re-filter Tab 1 tables. This is intentional — both tabs share the same filter state per Phase 30 design. The two dropdowns are independent DOM elements and do not visually sync their selected value to each other; this is an acceptable known limitation.
