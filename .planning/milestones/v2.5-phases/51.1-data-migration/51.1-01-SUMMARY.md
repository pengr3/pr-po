## Files Created/Modified
- `scripts/import.js` - CSV-to-Firestore import script for clients, projects, and services

## Decisions Made
- CSV/TSV parsed with built-in Node.js — no external library needed; single character-by-character pass handles all edge cases including multiline quoted fields
- `normalizeHeader()` updated to strip all non-alphanumeric characters (e.g. parentheses in Excel column headers like "Budget (PHP)")
- `parseNumber()` strips Philippine peso symbol (₱) and other currency symbols before `parseFloat()`
- Services CSV reuses the same column structure as projects (PROJECT CODE, PROJECT NAME) because both come from the same Excel format — mapped to service_code/service_name in Firestore
- `client_name` CSV column stored as `company_name` to match existing app Firestore schema
- `proposal_status_internal` CSV column stored as `internal_status` to match app field name
- Services `internal_status` hardcoded to 'Ready to Submit' (single consistent default, not read from CSV)
- `service_type` stored lowercased (e.g. `RECURRING` → `recurring`) to match app enum values

## Deviations from Plan

### Auto-fixed Issues during Human Verification

**1. [Rule 1 - Bug] CSV parser couldn't handle real Excel TSV export format**
- **Found during:** Task 2 (human verification with real CSVs)
- **Issue:** Original line-split parser failed on multiline quoted fields from Excel; Excel also exports as TSV not CSV
- **Fix:** Rewrote to character-by-character single-pass parser with auto-delimiter detection; collapses newlines inside quotes to spaces
- **Files modified:** `scripts/import.js`
- **Verification:** All three CSVs parsed correctly; row counts matched Excel sheet counts
- **Committed in:** `9d0f31b`

**2. [Rule 1 - Bug] normalizeHeader() didn't strip parentheses from Excel column names**
- **Found during:** Task 2 (human verification)
- **Issue:** Column "Budget (PHP)" normalized to "budget__php_" instead of "budget_php"; regex only targeted spaces/hyphens
- **Fix:** Updated regex to replace all non-alphanumeric chars with underscore, then strip leading/trailing underscores
- **Files modified:** `scripts/import.js`
- **Verification:** Headers normalize cleanly; "Budget (PHP)" → "budget_php"
- **Committed in:** `9d0f31b`

**3. [Rule 1 - Bug] parseNumber() failed on PHP peso amounts formatted with ₱ symbol**
- **Found during:** Task 2 (human verification)
- **Issue:** `parseFloat("₱1,234,567")` returns NaN — currency symbol not stripped
- **Fix:** Extended cleanup regex to include `₱$€£¥` in addition to commas and spaces
- **Files modified:** `scripts/import.js`
- **Verification:** `parseNumber("₱1,234,567.50")` correctly returns `1234567.5`
- **Committed in:** `9d0f31b`

**4. [Rule 1 - Bug] Client validator expected company_name column, real CSV has client_name**
- **Found during:** Task 2 (human verification)
- **Issue:** validateClients() checked for `row.company_name` but real Excel sheet uses column "CLIENT NAME" → normalizes to `client_name`
- **Fix:** Validator now reads `row.client_name`, stores result as `company_name` in Firestore document
- **Files modified:** `scripts/import.js`
- **Verification:** Clients parsed correctly with company_name stored in Firestore
- **Committed in:** `9d0f31b`

**5. [Rule 1 - Bug] Services column mapping didn't match real CSV structure**
- **Found during:** Task 2 (human verification)
- **Issue:** Real services CSV uses same PROJECT CODE/PROJECT NAME column headers as projects sheet (from same Excel workbook); script expected service_code/service_name columns; also needed project_location, total_expense, date_of_proposal_sent, and internal_status default
- **Fix:** validateServices() now reads project_code → service_code, project_name → service_name, project_location → location, total_expense → expense; adds date_of_proposal_sent; sets internal_status: 'Ready to Submit'; lowercases service_type
- **Files modified:** `scripts/import.js`
- **Verification:** Services imported correctly with all fields populated
- **Committed in:** `9d0f31b`
