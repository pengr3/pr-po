JSON format contract (shared between backup and restore):
- One JSON file per collection: `{collection_name}.json`
- Each file contains an array of objects: `[{ "_id": "docId", ...fields }, ...]`
- `_id` field preserves the Firestore document ID
- Subcollections stored as `{parent}_{child}.json` (e.g., `projects_edit_history.json`)
- Each subcollection entry includes `_parentId` to identify the parent document
- Timestamp fields serialized as ISO 8601 strings with a `__type: "timestamp"` marker for restore

Create `scripts/backup.js` — a Node.js script using Firebase Admin SDK that exports all Firestore collections to JSON files.

**Firebase Admin SDK setup:**
- Use `firebase-admin` package (will need `npm install firebase-admin` — add to existing package.json or script instructions)
- Initialize with service account key: `const admin = require('firebase-admin'); const serviceAccount = require(process.env.GOOGLE_APPLICATION_CREDENTIALS || './serviceAccountKey.json'); admin.initializeApp({ credential: admin.credential.cert(serviceAccount) });`
- Alternative: Use `GOOGLE_APPLICATION_CREDENTIALS` env var pointing to the key file

**Script behavior:**
1. Create timestamped output directory: `backups/YYYY-MM-DDTHHMMSS/`
2. For each of the 13 top-level collections (users, role_templates, invitation_codes, projects, clients, mrfs, prs, pos, transport_requests, suppliers, deleted_mrfs, deleted_users, services):
   - Fetch all documents via `admin.firestore().collection(name).get()`
   - Serialize each document as `{ _id: doc.id, ...doc.data() }`
   - Handle Firestore Timestamps: convert to `{ __type: "timestamp", value: timestamp.toDate().toISOString() }`
   - Write array of documents to `{collection_name}.json`
3. For subcollections (projects/edit_history, services/edit_history):
   - For each parent document, fetch `doc.ref.collection('edit_history').get()`
   - Serialize as `{ _id: doc.id, _parentId: parentDoc.id, ...doc.data() }`
   - Write to `projects_edit_history.json` and `services_edit_history.json`
4. Print summary: collection name, document count, file path for each exported collection
5. Print total documents exported and backup directory path

**Error handling:**
- Wrap in try/catch, log errors per collection but continue with remaining collections
- Exit with non-zero code if any collection fails
- Validate service account key exists before starting

**Console output format:**

  role_templates: 7 documents → backups/2026-03-01T120000/role_templates.json

Backup complete: 13 collections + 2 subcollections, 342 total documents
Output: backups/2026-03-01T120000/

<task type="auto">
  <name>Task 2: Create Firestore restore script</name>
  <files>scripts/restore.js</files>
  <action>
Create `scripts/restore.js` — a Node.js script that re-imports JSON backup files into Firestore.

**Script invocation:**
```
node scripts/restore.js <backup-directory>
# Example: node scripts/restore.js backups/2026-03-01T120000
```

**Script behavior:**
1. Accept backup directory path as CLI argument (required)
2. Validate directory exists and contains JSON files
3. Support `--dry-run` flag: lists what would be imported without writing
4. Support `--collections` flag: comma-separated list to restore specific collections only (e.g., `--collections projects,services`)
5. **Without `--dry-run`:** Require typed confirmation before writing:
   ```
   WARNING: This will overwrite existing documents in Firestore.
   Backup: backups/2026-03-01T120000 (342 documents across 15 files)
   Type "RESTORE" to confirm:
   ```
6. For each JSON file in the backup directory:
   - Read and parse the JSON array
   - For regular collections: use `admin.firestore().collection(name).doc(item._id).set(data)` (strip `_id` from the data payload)
   - For subcollection files (detected by `_` in filename matching known patterns like `projects_edit_history`): use `admin.firestore().collection(parent).doc(item._parentId).collection(child).doc(item._id).set(data)` (strip `_id` and `_parentId`)
   - Deserialize Timestamp markers: convert `{ __type: "timestamp", value: "..." }` back to `admin.firestore.Timestamp.fromDate(new Date(isoString))`
7. Use batched writes (500 docs per batch — Firestore limit) for performance
8. Print progress per collection and final summary

**Dry-run output:**
```
DRY RUN — no data will be written

  users.json: 12 documents would be restored
  role_templates.json: 7 documents would be restored
  ...

Total: 342 documents across 15 files
```

**Error handling:**
- Skip files that fail to parse with warning (don't abort entire restore)
- Log any documents that fail to write with doc ID and error
- Exit with non-zero code if any writes fail
