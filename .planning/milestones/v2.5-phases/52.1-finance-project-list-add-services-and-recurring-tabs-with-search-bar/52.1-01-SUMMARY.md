## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Replaced getAggregateFromServer TR query with client-side calculation**
- **Found during:** Task 2 (Service/Recurring expense data loading)
- **Issue:** `getAggregateFromServer` with `where('service_code', '==', code)` + `where('finance_status', '==', 'Approved')` on `transport_requests` requires a Firestore composite index that does not exist, causing runtime errors
- **Fix:** Replaced with `getDocs()` fetching all TRs by service_code, then filtering `finance_status === 'Approved'` in JavaScript, summing `total_amount` client-side
- **Files modified:** `app/views/finance.js`
- **Verification:** Services and Recurring sub-tabs load without Firestore index errors
- **Committed in:** `a995d69` (post-task fix)
