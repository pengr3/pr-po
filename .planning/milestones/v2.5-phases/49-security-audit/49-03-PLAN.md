## Goal
CSP headers are hardened for production with Firebase origins whitelisted and standard security headers added. A comprehensive security audit report documents all findings from Plans A, B, C, and D with severity ratings.

## Context
- Current CSP in both files: `Content-Security-Policy: frame-ancestors 'self'` — minimal, only prevents framing
- No `script-src`, `style-src`, `connect-src`, or other directives currently set
- Application loads Firebase SDK from CDN (gstatic.com), connects to Firestore (firebaseio.com, googleapis.com)
- Hundreds of inline onclick handlers require `'unsafe-inline'` in script-src
- Two header config files must stay in sync: `netlify.toml` and `_headers`
- Current headers only have: X-Content-Type-Options (nosniff) and Cache-Control directives
- No Referrer-Policy, X-Frame-Options, or Permissions-Policy headers currently

## Decisions (from CONTEXT.md — non-negotiable)
- Moderate CSP level: add `script-src`, `style-src`, `connect-src` directives whitelisting Firebase origins
- Allow `'unsafe-inline'` in script-src — required for hundreds of inline onclick handlers
- Add standard security headers: Referrer-Policy (no-referrer-when-downgrade), X-Frame-Options (DENY), Permissions-Policy (restrict camera, mic, geolocation)
- Update BOTH `_headers` and `netlify.toml` to stay in sync
- Create `SECURITY-AUDIT.md` in `.planning/phases/49-security-audit/`
- Include severity ratings: Critical / High / Medium / Low per finding
- Each finding: location, description, severity, fix applied

<name>Harden CSP and add security headers to Netlify config</name>
<files>

<action>
1. **Update the Content-Security-Policy directive** in both `netlify.toml` and `_headers` for the `/*` and `/*.html` routes. The new CSP should include:

   ```
   Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' https://*.gstatic.com https://*.googleapis.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; connect-src 'self' https://*.firebaseio.com https://*.googleapis.com wss://*.firebaseio.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' data: https:; frame-ancestors 'self'
   ```

   Breakdown of directives:
   - `default-src 'self'`: Block everything not explicitly allowed
   - `script-src 'self' 'unsafe-inline' https://*.gstatic.com https://*.googleapis.com`: Own scripts + inline handlers + Firebase SDK CDN
   - `style-src 'self' 'unsafe-inline' https://fonts.googleapis.com`: Own styles + inline styles + Google Fonts (if used)
   - `connect-src 'self' https://*.firebaseio.com https://*.googleapis.com wss://*.firebaseio.com`: Firestore HTTP + WebSocket connections
   - `font-src 'self' https://fonts.gstatic.com`: Google Fonts font files (if used)
   - `img-src 'self' data: https:`: Own images + data URIs + any HTTPS images
   - `frame-ancestors 'self'`: Prevent framing (existing directive, preserved)

2. **Add new security headers** to BOTH `netlify.toml` and `_headers` for the `/*` route:

   ```
   X-Frame-Options: DENY
   Referrer-Policy: no-referrer-when-downgrade
   Permissions-Policy: camera=(), microphone=(), geolocation=()
   ```

   - `X-Frame-Options: DENY`: Redundant with frame-ancestors but provides legacy browser support
   - `Referrer-Policy: no-referrer-when-downgrade`: Send referrer for same-security navigations only
   - `Permissions-Policy: camera=(), microphone=(), geolocation=()`: Deny access to device APIs the app doesn't use

3. **netlify.toml format** for the `/*` block:
   ```toml
   [[headers]]
     for = "/*"
     [headers.values]
       X-Content-Type-Options = "nosniff"
       X-Frame-Options = "DENY"
       Referrer-Policy = "no-referrer-when-downgrade"
       Permissions-Policy = "camera=(), microphone=(), geolocation=()"
       Content-Security-Policy = "default-src 'self'; script-src 'self' 'unsafe-inline' https://*.gstatic.com https://*.googleapis.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; connect-src 'self' https://*.firebaseio.com https://*.googleapis.com wss://*.firebaseio.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' data: https:; frame-ancestors 'self'"
       Cache-Control = "public, max-age=0, must-revalidate"
   ```

4. **_headers format** for the `/*` block:
   ```
   /*
     X-Content-Type-Options: nosniff
     X-Frame-Options: DENY
     Referrer-Policy: no-referrer-when-downgrade
     Permissions-Policy: camera=(), microphone=(), geolocation=()
     Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' https://*.gstatic.com https://*.googleapis.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; connect-src 'self' https://*.firebaseio.com https://*.googleapis.com wss://*.firebaseio.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' data: https:; frame-ancestors 'self'
     Cache-Control: public, max-age=0, must-revalidate
   ```

5. **Update the `/*.html` block** in both files to use the same full CSP (not just frame-ancestors) and include the new security headers.

6. **Do NOT add CSP to CSS/JS/image/font routes** — those routes only need X-Content-Type-Options and Cache-Control.

7. **Verify sync**: Both files must have identical header values for every route. Compare line by line after editing.
</action>
