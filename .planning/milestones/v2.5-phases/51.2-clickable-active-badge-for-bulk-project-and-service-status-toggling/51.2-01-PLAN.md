active status directly, replacing the separate Activate/Deactivate button in the Actions column.

user can toggle multiple projects/services in quick succession ("bulk toggling") by clicking badges
directly in the table rows.

removed redundant Activate/Deactivate buttons from Actions column.

<task type="auto">
  <name>Task 1: Make list-view Active badges clickable in projects.js and services.js</name>
  <files>app/views/projects.js, app/views/services.js</files>
  <action>
In **app/views/projects.js**, modify the `renderProjectsTable()` function (around line 902):

1. Replace the static badge with a clickable one when `showEditControls` is true.
   The badge needs `onclick="event.stopPropagation(); toggleProjectActive('${escapeHTML(project.id)}', ${project.active})"`.
   The `event.stopPropagation()` is critical because the `<tr>` has its own `onclick` that navigates to detail.
   Add `cursor: pointer` and `title` attribute for UX clarity.
   For view-only users (`!showEditControls`), keep the badge static (no onclick, no cursor:pointer).

   Change from:
   ```html
   <td>
       <span class="status-badge ${project.active ? 'approved' : 'rejected'}">
           ${project.active ? 'Active' : 'Inactive'}
       </span>
   </td>
   ```
   To:
   ```html
   <td>
       <span class="status-badge clickable-badge ${project.active ? 'approved' : 'rejected'}"
             ${showEditControls ? `onclick="event.stopPropagation(); toggleProjectActive('${escapeHTML(project.id)}', ${project.active})" title="Click to ${project.active ? 'deactivate' : 'activate'}" style="cursor: pointer;"` : ''}>
           ${project.active ? 'Active' : 'Inactive'}
       </span>
   </td>
   ```

2. Remove the separate Activate/Deactivate button from the Actions column (line 909).
   Delete just the `<button>` that calls `toggleProjectActive()`. Keep the Edit and Delete buttons.

   Change from:
   ```html
   <button class="btn btn-sm btn-primary" onclick="editProject(...)">Edit</button>
   <button class="btn btn-sm btn-secondary" onclick="toggleProjectActive(...)">...</button>
   <button class="btn btn-sm btn-danger" onclick="deleteProject(...)">Delete</button>
   ```
   To:
   ```html
   <button class="btn btn-sm btn-primary" onclick="editProject(...)">Edit</button>
   <button class="btn btn-sm btn-danger" onclick="deleteProject(...)">Delete</button>
   ```

In **app/views/services.js**, apply the identical pattern in `renderServicesTable()` (around line 972):

1. Replace the static badge with a clickable one when `showEditControls` is true, using
   `onclick="event.stopPropagation(); window.toggleServiceActive('${escapeHTML(service.id)}', ${service.active})"`.

2. Remove the separate Activate/Deactivate button from the Actions column (line 979).
   Keep Edit and Delete buttons.

IMPORTANT: Both existing `toggleProjectActive()` and `toggleServiceActive()` functions remain
unchanged — they already handle permission checks, confirmation, Firestore update, edit history,
and toast feedback. The badge simply becomes the new trigger.
