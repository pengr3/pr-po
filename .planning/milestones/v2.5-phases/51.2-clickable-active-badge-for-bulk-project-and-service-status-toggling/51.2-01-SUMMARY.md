# Phase 51.2 Plan 01: Clickable Active Badge Summary

**Active/Inactive status badges in projects and services list views made clickable for bulk toggling, replacing the separate Activate/Deactivate button in each Actions column.**

## Performance

- **Duration:** ~15 min
- **Started:** 2026-03-01
- **Completed:** 2026-03-01
- **Tasks:** 3 (2 auto + 1 human-verify)
- **Files modified:** 3

## Accomplishments

- Projects list view: Active/Inactive badge is now clickable (with edit permissions), calls existing toggleProjectActive() with confirmation prompt and real-time Firestore update
- Services list view: Same clickable badge pattern applied, calls toggleServiceActive()
- event.stopPropagation() prevents row-level navigation from firing when clicking the badge
- View-only users see static badges (no onclick, no cursor:pointer)
- Removed separate Activate/Deactivate button from Actions column in both views — badge is the sole toggle trigger
- Added .clickable-badge CSS modifier with hover effect (opacity + scale + box-shadow) for visual feedback

## Task Commits

Each task was committed atomically:

1. **Task 1: Make list-view Active badges clickable in projects.js and services.js** - `54c7cb9` (feat)
2. **Task 2: Add CSS hover styles for clickable badges** - `1f2d704` (feat)
3. **Task 3: Visual verification of clickable badges** - human-verify checkpoint, approved by user

## Files Created/Modified

- `app/views/projects.js` - Added clickable-badge class and onclick to status badges in renderProjectsTable(); removed Activate/Deactivate button from Actions column
- `app/views/services.js` - Same pattern applied in renderServicesTable()
- `styles/components.css` - Added .status-badge.clickable-badge hover rules (opacity, scale, box-shadow, transition)

## Decisions Made

- event.stopPropagation() on badge onclick is required because the parent `<tr>` has its own onclick handler for row-level detail navigation — without stopPropagation, clicking the badge would also navigate away
- Used a CSS modifier class (.clickable-badge) rather than inline styles so hover states can be applied in stylesheet; cursor:pointer kept inline because it's conditionally rendered based on permissions
- Removed Activate/Deactivate button entirely rather than hiding it — the badge fully replaces its function, no redundancy needed
- Toggle functions (toggleProjectActive, toggleServiceActive) left untouched — they already handle confirmation, Firestore update, edit history, and toast feedback
