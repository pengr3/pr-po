Create scripts/seed-dev-database.js as a browser-console IIFE. The script must:

1. SAFETY GUARD: At the top of the async IIFE, check `window.location.hostname`. If not 'localhost' or '127.0.0.1', log an error and return immediately. This prevents accidental writes to production.

2. IMPORTS: Use the established project pattern — import `db` from `/app/firebase.js` (gets the already-authenticated dev db instance) and import Firestore methods from the CDN URL `https://www.gstatic.com/firebasejs/10.7.1/firebase-firestore.js`. Import: `doc, setDoc, addDoc, collection, serverTimestamp`.

3. VERIFY DB: Check `if (!db)` and abort with console.error if db is unavailable.

4. SEED ROLE_TEMPLATES — 7 documents using setDoc (full overwrite, no merge):
   Document IDs MUST match ROLE_ORDER from role-config.js exactly. Each document has:
   - `role_id`: same as document ID
   - `role_name`: human-readable label
   - `permissions.tabs`: object with ALL 8 tab IDs as keys (dashboard, clients, projects, services, mrf_form, procurement, finance, role_config), each having `{ access: boolean, edit: boolean }`

   Use these exact permission defaults (derived from business logic in RESEARCH.md):
   ```
   super_admin:       all tabs access:true edit:true
   operations_admin:  dashboard(T,F) clients(T,T) projects(T,T) services(F,F) mrf_form(T,T) procurement(T,T) finance(T,F) role_config(F,F)
   operations_user:   dashboard(T,F) clients(T,F) projects(T,F) services(F,F) mrf_form(T,T) procurement(T,T) finance(T,F) role_config(F,F)
   services_admin:    dashboard(T,F) clients(T,T) projects(F,F) services(T,T) mrf_form(T,T) procurement(T,T) finance(T,F) role_config(F,F)
   services_user:     dashboard(T,F) clients(T,F) projects(F,F) services(T,F) mrf_form(T,T) procurement(T,T) finance(T,F) role_config(F,F)
   finance:           dashboard(T,F) clients(F,F) projects(F,F) services(T,F) mrf_form(F,F) procurement(F,F) finance(T,T) role_config(F,F)
   procurement:       dashboard(T,F) clients(F,F) projects(F,F) services(F,F) mrf_form(T,T) procurement(T,T) finance(F,F) role_config(F,F)
   ```

   Log each setDoc result: `console.log('[SeedDev] role_templates/${roleId}: OK')` on success, `console.error('[SeedDev] role_templates/${roleId}: FAILED', err)` on error.

5. SEED INVITATION_CODE — 1 document using addDoc (auto-ID):
   ```javascript
   await addDoc(collection(db, 'invitation_codes'), {
       code: 'DEV-SEED-2026',
       status: 'active',
       created_at: serverTimestamp(),
       created_by: 'seed-script',
       used_at: null,
       used_by: null
   });
   ```
   Log: `console.log('[SeedDev] invitation_codes: DEV-SEED-2026 created OK')`

   NOTE: Do NOT deduplicate — addDoc is fine here since the invitation_code status is checked by the app (used codes become 'used'). If this runs twice, there will be two active codes — harmless.
