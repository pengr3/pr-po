## Decisions Made

- Used `addDoc` (not `setDoc`) for invitation_codes — if the script is run twice, two active invitation codes exist, which the app handles correctly (both remain usable until consumed). No deduplication logic needed.
- Script committed to git so dev database can be re-seeded any time after a wipe without reconstructing the data.
- Localhost guard checks `window.location.hostname` and aborts if not `localhost` or `127.0.0.1` — prevents accidental production writes.
