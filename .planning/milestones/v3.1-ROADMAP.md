## Milestone Goal

Fix the edge case where supplier-delivered goods generate a TR instead of being included in the PO, by introducing a dedicated "DELIVERY BY SUPPLIER" category that routes exclusively through the PR/PO path. Extended to include TR rejection independence, MRF soft-reject, TR details modal, and multiple bug fixes.

## Key Accomplishments

1. "DELIVERY BY SUPPLIER" category routing items to PR/PO path instead of TR
2. TR rejection independence — TR lifecycle fully decoupled from MRF status with dedicated rejected panel, editable items, line item add/delete, and resubmit flow
3. TR code visibility — TR badges in PRs column on both MRF Records and My Requests, clickable to TR details modal
4. MRF soft-reject replacing hard delete — preserves audit trail with rejection reason, actor, and timestamp shown on timeline
5. Project/service dropdown alphabetical sorting and project code dash format (CLMC-CLIENT-YYYYnnn)
6. Responsive workspace optimizations for 1366px laptops, sortable My Requests headers, and real-time MRF Records rendering
