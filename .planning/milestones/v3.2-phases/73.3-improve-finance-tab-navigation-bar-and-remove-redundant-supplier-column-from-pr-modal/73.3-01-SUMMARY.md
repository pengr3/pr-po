# Phase 73.3 Plan 01: Finance Tab Nav + PR Modal Supplier Column Summary

**Unified sticky pill/tab nav bar for Finance (replaces desktop tabs + mobile dropdown) with scroll-hide/show UX, and PR Details modal items table trimmed from 6 to 5 columns by removing redundant Supplier column**

## Performance

- **Duration:** ~20 min
- **Started:** 2026-04-15T10:00:00Z
- **Completed:** 2026-04-15T10:20:00Z
- **Tasks:** 2
- **Files modified:** 2

## Accomplishments

- Replaced split desktop-tab-bar + mobile-`<select>` Finance navigation with a single `.finance-sub-nav` `<nav>` element containing `.finance-sub-nav-tab` pill buttons that work on all viewport widths via `overflow-x: auto`
- Added `position: sticky; top: 0` with scroll-hide/show behavior: scrolling down adds `.finance-sub-nav--hidden` (`translateY(-100%)`); scrolling up removes it
- Added module-level `_financeNavScrollHandler` / `_financeNavLastScrollY` with init() guard (`if (!_financeNavScrollHandler)`) and destroy() cleanup to prevent duplicate listeners across sub-tab switches
- Removed the Supplier `<th>` header, `<td>` body cell, and corrected `colspan` from 5 to 4 in the PR Details modal items table — supplier is already shown in the modal header details grid

## Task Commits

1. **Task 1: Remove Supplier column from PR Details modal items table** - `fec5a51` (fix)
2. **Task 2: Rebuild Finance sub-tab navigation as unified sticky pill/tab bar + add CSS** - `ba0f9fd` (feat)

## Files Created/Modified

- `app/views/finance.js` - New `.finance-sub-nav` HTML in render(); scroll handler attach in init(); scroll handler remove in destroy(); Supplier column removed from PR modal items table; `_financeNavScrollHandler`/`_financeNavLastScrollY` module-level state added
- `styles/views.css` - Replaced `FINANCE TAB — MOBILE RESPONSIVE` block with new `FINANCE TAB — SUB-NAV (Phase 73.3)` block; `.finance-expense-subtab-bar` rules preserved verbatim inside new `@media` block

## Decisions Made

- Used `.finance-sub-nav-tab` class prefix (not `.tab-btn`) to keep Finance pill styling scoped — Procurement view uses `.tab-btn` globally and must remain untouched
- Single `overflow-x: auto` on `.finance-sub-nav-tabs` handles both desktop and mobile without any JS viewport detection (CSS-only approach)
- Init guard pattern (`if (!_financeNavScrollHandler)`) is the correct solution because the router does NOT call `destroy()` on intra-view sub-tab switches — `init()` fires repeatedly but listener must attach only once per Finance mount
- Supplier column removed from the PR items table only; the modal header `.modal-details-grid` continues to show supplier as the canonical display location
