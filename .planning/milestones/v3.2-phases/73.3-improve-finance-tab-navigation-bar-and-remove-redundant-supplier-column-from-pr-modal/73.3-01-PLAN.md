**6. Finance view listener/destroy pattern (finance.js lines 2750–2810, destroy()):**
- `listeners` array holds Firestore unsubscribe functions
- destroy() unsubs, resets state, deletes `window.*` handlers
- For our new scroll listener: store as module-level `let _financeNavScrollHandler = null` and `let _financeNavLastScrollY = 0`; remove in destroy() via `window.removeEventListener('scroll', _financeNavScrollHandler)` and reset both to `null`/`0`.

**7. Router context (CLAUDE.md):**
- Router does NOT call destroy() when switching tabs within the same view — only when navigating to a different view
- render() and init() are called with a new tab parameter on sub-tab switches
- Therefore: the scroll listener must be attached ONCE in init() and removed in destroy(). Attaching it on every sub-tab switch would stack duplicates — use a guard check `if (_financeNavScrollHandler) return;` before attaching.

            if (!nav) return;
            const currentY = window.scrollY || 0;
            const delta = currentY - _financeNavLastScrollY;
            // Ignore tiny jitters and ignore when near top (always show near top).
            if (Math.abs(delta) < 5) return;
