## Decisions Made

- **180px / 220px wrapper heights:** Computed from 28 (bar) + 78 (legend ~3 rows × 22px line-height + padding) + 24 (top/bottom breathing room) ≈ 130px, rounded to 180px to absorb 4-row legend wrapping in narrow cards. Mobile gets +40px because wider per-item padding pushes legend to 4-5 rows at 360-414px viewports.
- **Bumping barThickness in lockstep with shrinking the card:** The card shrink alone would have left the bar still tiny relative to the (smaller) card; growing the bar 22→28 ensures the bar visually dominates the chart area.
- **+2px on all legend dimensions (not just font):** User's request was "bump legend 2px and adjust square accordingly" — applying the +2px uniformly to font.size, boxWidth, and boxHeight keeps the legend visually balanced (text and color square scale together).
- **No per-card divergence:** All three chart wrappers (Projects, Services One-time, Services Recurring) share the single `.hs-chart-status` CSS class via `getChartSizeClass()` in home.js (Phase 81 architecture), so a single CSS rule and a single Chart.js options block update propagated to all three with zero additional code.
