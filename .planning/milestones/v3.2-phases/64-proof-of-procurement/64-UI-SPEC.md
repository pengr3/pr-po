## Component Inventory

All new visual elements introduced in this phase:

### 1. Proof Indicator (Checkmark Column)

Used in: MRF Records table (Procurement), My Requests table, Finance PO Tracking table.

**Empty state (no proof):**
- 20px × 20px circle
- Border: 1.5px solid `#bdc1c6`
- Background: transparent
- Cursor: pointer
- Title attribute: "Click to attach proof"
- Click: opens Proof URL Modal

**Filled state (proof attached):**
- 20px × 20px filled circle
- Background: `#34a853`
- Foreground: white checkmark (✓ or SVG path)
- Cursor: pointer
- Title attribute: "Proof attached — left-click to open, right-click to replace"
- Left-click: opens `proof_url` in new tab (`window.open(url, '_blank')`)
- Right-click (`contextmenu` event): opens Proof URL Modal pre-filled with current URL (for replacement)
- Long-press on mobile (touchstart + 600ms timer): opens Proof URL Modal pre-filled

**Column placement:**
- In MRF Records table: one indicator per PO sub-row, rendered in a new narrow column to the right of the POs column
- In PO Tracking table: new column to the left of the Actions column
- In Finance PO table: new column to the left of the Actions column
- Column header label: "Proof" (plain text, 12px, weight 600, centered)
- Column width: 40px (narrow, centered content)

**Sub-row alignment (MRF Records):**
- Each PO sub-row (`height: 30px; display: flex; align-items: center`) gets its own indicator
- Indicator centered vertically within the 30px row
- The new Proof column uses `border-top: 1px dashed #e5e7eb` between sub-rows (matches existing sub-row dividers)

### 2. Proof URL Modal

Triggered by: status change to "Procured" (material) or "Processed" (SUBCON), OR clicking an empty indicator at any time, OR right-clicking/long-pressing a filled indicator.

Structure: uses existing `createModal()` from `app/components.js` — window-style, blur backdrop, `border-radius: 12px`.

Modal id: `proofUrlModal`
Modal title: "Attach Proof of Procurement" (status-change trigger) or "Update Proof URL" (edit trigger)
Modal size: `normal` (max-width: 900px × 90%, but this content is compact — modal-content width will appear ~480px via inner constraint)

**Body:**
```
[Label] Proof URL
[Input] type="url" placeholder="https://..." style="width:100%; padding: 0.5rem 0.75rem; border: 1px solid #e5e7eb; border-radius: 6px; font-size: 0.875rem"
[Helper text] "Paste a Google Drive, OneDrive, or SharePoint link (https:// required)" — 12px, color #64748b
```

When triggered by status change:
```
[Info text] "Status will be updated to [Procured/Processed] regardless of whether you attach proof." — 12px, color #64748b, margin-top 8px
```

**Footer buttons:**
- Status-change trigger: "Skip" (secondary, closes modal, proceeds with status update) + "Attach & Save" (primary, validates and saves)
- Edit trigger: "Cancel" (secondary) + "Save Proof URL" (primary)

**Validation:**
- On submit: check `url.startsWith('https://')` — if not, show inline error below input: "URL must start with https://" in `#ea4335`, font-size 12px
- Do not close modal on invalid input
- On valid: save to Firestore, close modal, show success toast

### 3. Timeline "Proof Attached" Event

Location: inside `viewPOTimeline()` alert content (text-only, no HTML — uses `alert()`).

Format (material timeline — inserted between Items Procured and Delivered):
```
✓ Proof Attached: [date] — [truncated url, max 60 chars]
```
or
```
○ Proof Attached: (none)
```

Format (SUBCON timeline — inserted between Processing Started and Processed):
Same pattern.

Note: `viewPOTimeline()` currently uses `alert()` for output. This phase adds proof event lines to the text content string. No modal conversion — keep as alert per existing pattern. URL displayed in full up to 60 chars, truncated with `...` if longer.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Column header | Proof |
| Primary CTA (attach new) | Attach Proof |
| Primary CTA (save replacement) | Save Proof URL |
| Skip button (status-change flow) | Skip |

| Input placeholder | https://drive.google.com/... |
| Input helper | Paste a Google Drive, OneDrive, or SharePoint link (https:// required) |
| Status-change info | Status will be updated to [X] regardless of whether you attach proof. |

| Indicator tooltip (filled) | Left-click to open proof · Right-click to replace |
| Inline validation error | URL must start with https:// |
| Toast — proof attached | Proof URL attached |
| Toast — proof updated | Proof URL updated |
| Toast — save error | Failed to save proof URL. Please try again. |

- "Proof of Procurement" is the full feature name; column header is shortened to "Proof" to preserve table width
- No destructive actions in this phase — no confirmation dialogs beyond the standard status-change `confirm()` already in `updatePOStatus()`
- proof_attached_at is set only on first attach; if user replaces URL later, only proof_url is updated (proof_attached_at preserved). Add `proof_updated_at` on replacement — no copy needed for this, it's a silent Firestore write.
