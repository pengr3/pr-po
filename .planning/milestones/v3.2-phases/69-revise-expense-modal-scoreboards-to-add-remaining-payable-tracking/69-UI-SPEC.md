## Scoreboard Row Layout Contract

### Existing rows (do not change)

**Row 1 — Budget row** (2-column grid, `grid-template-columns: 1fr 1fr`, `gap: 1rem`, `margin-bottom: 1.5rem`):
- Budget card: green left-border accent, `#f0fdf4` background
- Remaining Budget card: green/red conditional per sign of `remaining`

**Row 2 — Category row** (3-column grid, `grid-template-columns: repeat(3, 1fr)`, `gap: 1rem`, `margin-bottom: 1.5rem`):
- Material Purchases | Transport Fees | Subcon Cost
- Each: border `1px solid #e2e8f0`, `border-radius: 8px`, `padding: 1rem`

**Total Cost block** (full-width, `background: #eff6ff`, `border: 2px solid #3b82f6`, `margin-bottom: 2rem`):
- Stands alone between Row 2 and the tab navigation

### New Row 3 — Payable Tracking row (Phase 69 addition)

**Position:** Inserted between Row 2 (Category scoreboards) and the Total Cost block.

**Layout:** 3-column grid, `grid-template-columns: repeat(3, 1fr)`, `gap: 1rem`, `margin-bottom: 1.5rem`.

**Visibility rule:** Render only when `totalRequested > 0`. When no RFPs exist, this entire row is omitted — no ₱0.00 placeholders shown.

**Cards (left to right):**

1. **Total Requested (RFPs)** — neutral card
   - Label: "Total Requested (RFPs)"
   - Value: `&#8369;${formatCurrency(totalRequested)}`
   - Value color: `#1e293b`
   - Background: `#ffffff`, Border: `1px solid #e2e8f0`

2. **Total Paid** — success card
   - Label: "Total Paid"
   - Value: `&#8369;${formatCurrency(totalPaid)}`
   - Value color: `#059669`
   - Background: `#ffffff`, Border: `1px solid #e2e8f0`

3. **Remaining Payable** — conditional danger/success card
   - Label: "Remaining Payable"
   - Value: `&#8369;${formatCurrency(remainingPayable)}`
   - When `remainingPayable > 0`: background `#fef2f2`, border `1px solid #fca5a5`, label color `#991b1b`, value color `#ef4444`
   - When `remainingPayable === 0`: background `#ffffff`, border `1px solid #e2e8f0`, label color `#64748b`, value color `#059669`

**No sub-label count note** on payable cards (no equivalent of "N POs" note). The note pattern on Row 2 cards is informational for procurement context; payable cards are Finance-oriented totals and do not need document count.

## Interaction Contract

### Modal open flow
No change to trigger, open animation, backdrop, or close behavior. Phase 69 only modifies the static HTML body of the modal.

### No new interactive elements
Row 3 is display-only. No click handlers, no expand/collapse, no tooltips.

### Tab navigation unaffected
The "By Category" / "Transport Fees" tab bar and all tab-switching logic remain unchanged. Row 3 is above the tab bar in the DOM.

### Async load behavior
Row 3 data is fetched in the same `showExpenseBreakdownModal` async function before modal HTML is built. The modal does not appear until the RFP query completes — consistent with existing PO/TR query pattern. No loading spinner needed.
