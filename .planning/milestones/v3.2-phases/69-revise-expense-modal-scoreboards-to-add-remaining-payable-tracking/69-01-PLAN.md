**Change 1 — RFP query (insert after line 45, after the existing PO/TR query block ends):**

Add a new block that fetches RFPs based on mode:

```javascript
// Fetch RFPs for payable tracking
let rfpsForPayable = [];
if (mode === 'service') {
    const rfpSnap = await getDocs(
        query(collection(db, 'rfps'), where('service_code', '==', identifier))
    );
    rfpSnap.forEach(d => rfpsForPayable.push(d.data()));
} else {
    const projectCode = project?.project_code || '';
    if (projectCode) {
        const rfpSnap = await getDocs(
            query(collection(db, 'rfps'), where('project_code', '==', projectCode))
        );
        rfpSnap.forEach(d => rfpsForPayable.push(d.data()));
    }
}
let totalRequested = 0;
let totalPaid = 0;
rfpsForPayable.forEach(rfp => {
    totalRequested += parseFloat(rfp.amount_requested || 0);
    totalPaid += (rfp.payment_records || []).reduce((s, r) => s + parseFloat(r.amount || 0), 0);
});
const remainingPayable = totalRequested - totalPaid;
```

CRITICAL: In project mode, use `project.project_code` (the `project` variable from line 39), NOT `project_name`. RFP documents store `project_code`, not `project_name`. This is the #1 pitfall identified in RESEARCH.md.

CRITICAL: In service mode, use `identifier` directly with `service_code` field on rfps — same pattern as the PO/TR queries above.
