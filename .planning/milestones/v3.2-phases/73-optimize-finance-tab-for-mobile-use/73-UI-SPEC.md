## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (e.g. modal footer gap on mobile, filter toolbar gaps, card-header gap on mobile stack) |
| md | 8px | Alias for sm — use sm token; md = 8px (not 12px) |
| lg | 16px | Default element spacing (1rem), container padding on mobile |
| xl | 24px | Section padding (1.5rem), card-header gap default |
| 2xl | 32px | Layout gaps |
| 3xl | 64px | Page-level spacing |

Source: CLAUDE.md UI Design System ("Consistent spacing (0.75rem, 1rem, 1.5rem, 2rem)") and existing `components.css` / `views.css` values confirmed by codebase read.

Exceptions:
- Touch targets for mobile buttons must meet 44px minimum height. Finance card-header buttons and filter selects must be at least 44px tall on screens <= 768px. Implement via `min-height: 44px` on `.card-header .btn` and `.card-header select` in the Finance mobile block.
- Sub-tab bar buttons (Projects/Services/Recurring) maintain their inline height; the container adds `overflow-x: auto` for 320px safety.
- Filter toolbar gaps and card-header gap on mobile stack: use `gap: 0.5rem` (8px / sm token). Do NOT use `gap: 0.75rem` (12px).

## Interaction Contract

This section is specific to Phase 73 because it is an interaction/layout phase, not a feature phase.

### Breakpoints

| Breakpoint | Value | Applies To |
|------------|-------|------------|
| Mobile primary | 768px max-width | All Finance tab mobile fixes (card-headers, sub-tab bar) |
| Phone narrow | 480px max-width | Card-header selects and buttons go full-width (secondary block) |

Source: RESEARCH.md "Standard Stack" — project breakpoints are 768px (mobile), 1024px (tablet), 1400px (wide). Phase 73 uses only the 768px breakpoint as the primary fix target.

### Card-Header Mobile Layout (PRIMARY fix)

On screens <= 768px, Finance card-headers that contain a heading AND right-side controls must stack vertically:

```
[card-header flex-direction: column, align-items: flex-start, gap: 8px]
  [h2 heading — full width]
  [controls div — width: 100%, display: flex, flex-wrap: wrap, gap: 8px]
    [select — flex: 1, min-width: 120px, min-height: 44px]
    [button.btn — flex: 1, min-width: 0, min-height: 44px]
```

Note: card-header gap on mobile stack is 8px (`gap: 0.5rem`), not 12px.

Affected card-headers (confirmed from RESEARCH.md finance.js line audit):
1. Material PRs card-header: h2 + [status select, "View Details" button] — lines 1543–1554
2. Purchase Orders card-header: h2 + [status select, dept select, Refresh button, Export button] — lines 1612–1629
3. Transport Requests card-header: same pattern as Material PRs

Implementation target: Add to `styles/components.css` inside the existing `@media (max-width: 768px)` block (lines 1284–1360). This makes it global, matching the pattern for `.modal-footer`, `.pagination-container`, and `.form-actions` which are also globally stacked in that block.

Risk mitigation: The RESEARCH.md Pitfall 1 flags global `.card-header` side-effects. The planner must verify card-header behavior in Procurement, Projects, and Services views at 768px before shipping.

### Table Horizontal Scroll (CONFIRMED already working)

All Finance tables already use `.table-scroll-container`. No changes needed to the scroll container. The executor must NOT add a second `.table-scroll-container` wrapper.

| Table | Container | Status |
|-------|-----------|--------|
| Material PRs | `.table-scroll-container` | Confirmed OK |
| Transport Requests | `.table-scroll-container` | Confirmed OK |
| Purchase Orders | `.table-scroll-container` | Confirmed OK |
| RFP Processing | `.table-scroll-container` | Confirmed OK |
| PO Payment Summary | `.table-scroll-container` | Confirmed OK |
| Project/Service/Recurring expense tables | `<div style="overflow-x: auto;">` | NEEDS UPGRADE — see below |

### Bare `overflow-x` Upgrade (cosmetic fix)

Three render functions in `finance.js` wrap expense tables with `<div style="overflow-x: auto;">` instead of `<div class="table-scroll-container">`. These must be upgraded in `renderProjectExpensesTable()`, `renderServiceExpensesTable()`, and `renderRecurringExpensesTable()`.

Change: replace `style="overflow-x: auto;"` with `class="table-scroll-container"` in each function's `tableHTML` string.

### Sub-Tab Bar (Project List tab, 320px safety)

The Projects/Services/Recurring sub-tab bar uses `display: flex; gap: 0` inline with 3 buttons. On 320px devices these may clip. Fix: add a `.finance-expense-subtab-bar` class to the container div in the `render()` function, then add to `views.css` Finance mobile block:

```css
.finance-expense-subtab-bar {
    overflow-x: auto;
    -webkit-overflow-scrolling: touch;
}
.finance-expense-subtab-bar button {
    white-space: nowrap;
}
```

This is scoped to Finance only — no global side-effects.

### Modals

No modal changes needed. Existing global rules already handle Finance modals:
- `modal-content { width: 95%; max-height: 90vh; }` at 768px (components.css line 1302)
- `modal-footer { flex-direction: column-reverse; gap: 0.5rem; }` at 768px (components.css line 1349)
- Signature canvas uses `resizeCanvas()` — already adapts to container width

### Payables Filter Toolbars (already correct — no change)

Both Payables toolbars use `flex-wrap: wrap` inline. They already wrap on mobile. No fix needed; executor must not touch these.

### Stats Grid (already responsive — no change)

`grid-template-columns: repeat(auto-fit, minmax(250px, 1fr))` collapses under ~500px. No fix needed.
