    **Edit 8b — Dynamic legacy filter injection (NEW, addresses REVIEWS Concern 2):**
    Add a new helper `rebuildServiceStatusFilterOptions()` mirroring the Plan 81-01 pattern. It scans `servicesData` for legacy `project_status` values not in `UNIFIED_STATUS_OPTIONS` and injects them as a separate `<optgroup label="Other (legacy)">` at the bottom of `#serviceProjectStatusFilter`. Preserves the user's currently-selected value across rebuilds.

    Place adjacent to `applyServiceFilters()` within the same module scope:

    ```javascript
    function rebuildServiceStatusFilterOptions() {
        const select = document.getElementById('serviceProjectStatusFilter');
        if (!select) return;
        const previousValue = select.value;
        const legacyValues = new Set();
        for (const service of servicesData) {
            const v = service.project_status;
            if (v && !UNIFIED_STATUS_OPTIONS.includes(v)) {
                legacyValues.add(v);
            }
        }
        const unifiedHtml = UNIFIED_STATUS_OPTIONS
            .map(s => `<option value="${s}">${s}</option>`)
            .join('');
        const legacyHtml = legacyValues.size > 0
            ? `<optgroup label="Other (legacy)">${[...legacyValues].sort().map(v => `<option value="${escapeHTML(v)}">${escapeHTML(v)} (legacy)</option>`).join('')}</optgroup>`
            : '';
        select.innerHTML = `<option value="">All Statuses</option>${unifiedHtml}${legacyHtml}`;
        if (previousValue && [...select.options].some(o => o.value === previousValue)) {
            select.value = previousValue;
        }
    }
    ```

    Call `rebuildServiceStatusFilterOptions()` from inside `renderServicesTable()` (the function that re-renders on data updates) — invoke it as the first line of that function so the dropdown contains every value present in the current data set.
