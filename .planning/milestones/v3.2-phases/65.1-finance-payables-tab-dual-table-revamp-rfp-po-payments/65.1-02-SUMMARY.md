# Phase 65.1 Plan 02: PO Payment Summary Table (Full Implementation) Summary

## One-liner

Replaced renderPOSummaryTable stub with full PO-grouped implementation using buildPOMap + derivePOSummary — Finance now sees aggregated per-PO totals with expand-to-RFP sub-tables, independent filtering, and Record Payment access from both tables.

## What Was Built

### Task 1: buildPOMap and derivePOSummary helper functions

- Added `buildPOMap(rfps)` — pure function that groups an RFP array into a `Map<poId, {poId, supplier, deptLabel, isService, rfps[]}>` at render time; not stored as module-level state
- Added `derivePOSummary(rfpList)` — computes per-PO aggregates:
  - `totalAmount`: sum of `amount_requested` across all RFPs
  - `totalPaid`: sum of non-voided `payment_records.amount` across all RFPs
  - `remaining`: `totalAmount - totalPaid`
  - `currentTranche` (D-09): label of first non-Fully-Paid tranche sorted by `tranche_percentage` asc
  - `overallStatus` (D-13): Fully Paid (`remaining <= 0`) > Overdue (any RFP overdue) > Partially Paid (`totalPaid > 0`) > Pending
  - `sortedRFPs`: tranches sorted by `tranche_percentage` asc for sub-table rendering

### Task 2: Full renderPOSummaryTable implementation

Replaced the 3-line stub with a complete ~115-line implementation:

- Calls `buildPOMap(rfpsData)` to group the module-level RFP array by PO ID
- Calls `derivePOSummary(entry.rfps)` for each PO entry
- D-16: Applies independent `poSummaryStatusFilter` and `poSummaryDeptFilter` at PO level (not RFP level)
- D-21: Sorts PO entries alphabetically by PO ID via `localeCompare`
- Renders one `<tr>` per PO with 9 columns: Chevron | PO ID | Supplier | Dept | Current Active Tranche | Total Amount | Total Paid | Remaining | Overall Status
- `safePoId` replaces non-alphanumeric chars with `_` for safe `id="po-expand-..."` / `id="po-chevron-..."` DOM element IDs
- Each PO row has a hidden expand `<tr id="po-expand-${safePoId}">` with inline sub-table
- Sub-table 8 columns: RFP ID | Tranche | Amount | Paid | Balance | Due Date | Status | Actions
- D-15: Record Payment button appears in sub-table rows for non-Fully-Paid RFPs (Finance role only via `canEditTab`)
- Overdue tint (`background-color:#fef2f2`) on both PO rows and sub-table RFP rows
- Contextual empty states: filtered vs. no data at all
