## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Status row gap between label and count; internal hs-status-row padding |
| sm | 8px (0.5rem) | Between status rows within a section group; section heading bottom margin |
| md | 16px (1rem) | Stat card internal padding — horizontal; between section groups inside a card |
| lg | 24px (1.5rem) | Stat card internal padding — vertical (top/bottom); gap between cards in the `.quick-stats` grid |
| xl | 32px (2rem) | Gap between `.hs-type-section` blocks in the Services card |
| 2xl | 48px (3rem) | Not used in this phase |
| 3xl | 64px (4rem) | Not used in this phase |

Exceptions:
- Stat card `padding: 1.5rem` (24px all sides) — smaller than nav-card's `2.5rem 2rem` per D-04 "slightly smaller padding" decision (CONTEXT.md D-04).
- Services card `<hr>` divider between One-time and Recurring sections: `margin: 1rem 0` (16px top and bottom).
- `.hs-type-label` (One-time / Recurring): `margin-bottom: 0.5rem` (8px) before the first section group.

## Typography

All sizes map directly to existing tokens in `styles/main.css`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Card heading | 1rem (16px) = h5 | 600 (semibold) | 1.2 | `.hs-stat-card-title` — "Procurement", "Projects", "Services" |
| Section heading | 0.75rem (12px) | 600 (semibold) | 1.2 | `.hs-section-heading` — "Internal Status", "Project Status"; uppercase + letter-spacing creates hierarchy without heavier weight |
| Type label | 0.75rem (12px) | 600 (semibold) | 1.2 | `.hs-type-label` — "One-time", "Recurring"; consolidated from 0.7rem — uppercase + tighter letter-spacing differentiates from section heading |
| Status label | 0.875rem (14px) | 400 (regular) | 1.5 | `.hs-status-label` — status name text e.g. "For Inspection"; consolidated from 0.8125rem |
| Status count | 0.875rem (14px) | 600 (semibold) | 1.5 | `.hs-status-count` — numeric count value |
| Procurement stat label | 0.875rem (14px) = `.stat-label` | 400 (regular) | 1.5 | Reuse existing `.stat-label` class — "Pending MRFs", "Pending PRs", "Active POs" |
| Procurement stat value | 2.5rem (40px) = `.stat-value` | 600 (semibold) | 1.0 | Reuse existing `.stat-value` class — numeric count (monospace font) |

**Font sizes in use (4 total):** 0.75rem (12px), 0.875rem (14px), 1rem (16px), 2.5rem (40px).

**Font weights in use (2 total):** regular (400) for status labels and stat labels; semibold (600) for card titles, section headings, type labels, status counts, and stat values. Uppercase lettering and letter-spacing provide visual hierarchy between section headings and type labels without requiring a third weight.

## Card Layout Specification

### `.quick-stats` Container (replacing current flex layout)

```
display: grid
grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))
gap: 1.5rem (24px)
max-width: 1200px
width: 100%
padding: 0                (cards carry their own padding — remove old single-card background)
background: none          (remove old white container background)
box-shadow: none          (each card has its own shadow)
border-radius: 0
```

Mobile override (`@media (max-width: 768px)`):
```
grid-template-columns: 1fr    (stacked single column)
gap: 1.5rem
```

### `.hs-stat-card` (individual card)

Matches `.nav-card` visually but with these differences:
- `padding: 1.5rem` (not `2.5rem 2rem` — secondary/informational per D-04)
- `cursor: default` (not `pointer` — no click action)
- `border: none` (no hover border — informational only)
- No hover transform (no `translateY(-8px)` — not interactive)
- Same: `background: white`, `border-radius: 12px`, `box-shadow: 0 4px 16px rgba(0,0,0,0.08)`

### Status Row Layout (`.hs-status-row`)

Use a 2-column CSS grid inside each status section:
```
display: grid
grid-template-columns: 1fr auto
align-items: center
gap: 0.5rem (8px)
padding: 0.25rem 0 (4px top and bottom)
```

This gives a label-flush-left / count-flush-right layout for all 11 status rows in Projects and 22 in Services. No separate flex row needed.

## Card Content Specifications

### Procurement Card

- **Card title:** "Procurement" (`.hs-stat-card-title`)
- **No section headings** — 3 stat items displayed as existing `.stat-item` / `.stat-label` / `.stat-value` pattern, arranged horizontally inside the card using `display: flex; justify-content: space-around`
- **Stats (left to right):**
  - "Pending MRFs" → `id="stat-mrfs"` (existing element ID preserved)
  - "Pending PRs" → `id="stat-prs"` (existing element ID preserved)
  - "Active POs" → `id="stat-pos"` (existing element ID preserved)
- Loading state: reuse existing `.skeleton.skeleton-stat` inside `.stat-value` (already used in `projectsStatsHtml()`)
- Role filter: mode `'projects'` shows department-filtered MRF count; mode `'services'` NOT shown (Procurement card shown to all roles per D-05); mode `'both'` shows total across all departments

### Projects Card (rendered when `mode === 'projects'` or `mode === 'both'`)

- **Card title:** "Projects" (`.hs-stat-card-title`)
- **Section 1:** "Internal Status" heading, then 4 rows:
  - For Inspection | count
  - For Proposal | count
  - For Internal Approval | count
  - Ready to Submit | count
- **Section 2:** "Project Status" heading, then 7 rows:
  - Pending Client Review | count
  - Under Client Review | count
  - Approved by Client | count
  - For Mobilization | count
  - On-going | count
  - Completed | count
  - Loss | count
- All 11 rows **always shown**, including zero counts (stable layout per RESEARCH.md recommendation)
- Loading state: full-width `.skeleton` rows with `height: 16px; margin-bottom: 4px` × 4 before data arrives

### Services Card (rendered when `mode === 'services'` or `mode === 'both'`)

- **Card title:** "Services" (`.hs-stat-card-title`)
- **Structure:** Two stacked type sections separated by `<hr class="hs-divider">`

**One-time section** (`.hs-type-section`):
- Type label: "One-time" (`.hs-type-label`, uppercase, muted)
- Section group 1: "Internal Status" → 4 rows
- Section group 2: "Project Status" → 7 rows

**Recurring section** (`.hs-type-section`):
- Type label: "Recurring" (`.hs-type-label`, uppercase, muted)
- Section group 1: "Internal Status" → 4 rows
- Section group 2: "Project Status" → 7 rows

- All 22 rows **always shown**, including zero counts
- Loading state: `.skeleton` rows × 4 per section on first load

## Interaction Contract

This phase is **read-only display only**. No interactions are introduced.

| Interaction | Behavior |
|-------------|----------|
| Card click | None — `cursor: default`, no hover effect, no link |
| Status row click | None — rows are not interactive |
| Count value update | Smooth text swap via `renderStatusBreakdown()` — no animation needed |
| Procurement counts | Continue using existing `updateStatDisplay()` for text swap |
| Role change mid-session | Not supported — role is stable; destroy()+init() handles on navigation return |

**Real-time updates:** All counts update automatically via Firestore `onSnapshot` without user action. No manual refresh button is added in this phase.

## States

| State | What is shown |
|-------|--------------|
| First load — no cached data | Skeleton shimmer rows in status containers; skeleton-stat in Procurement counts |
| First load — cached data exists | Cached values shown immediately (stale-while-revalidate); `.stat-refreshing` class adds muted styling until fresh data arrives |
| Data loaded | Count numbers and status rows render with `renderStatusBreakdown()` |
| Zero counts | "0" shown in count position — row remains visible |
| Role: operations_admin/user | Procurement card + Projects card only |
| Role: services_admin/user | Procurement card + Services card only |
| Role: all others | All 3 cards |

| `.hs-stat-card-title` | `<h4>` inside each card | `font-size:1rem; font-weight:600; color:var(--gray-900); margin-bottom:1rem; line-height:1.2` |

| `.hs-divider` | `<hr>` between One-time and Recurring | `border:none; border-top:1px solid var(--gray-100); margin:1rem 0` |
| `.hs-status-row` | Each label+count row | `display:grid; grid-template-columns:1fr auto; align-items:center; gap:0.5rem; padding:0.25rem 0` |

| `.hs-status-count` | Count number span | `font-size:0.875rem; font-weight:600; color:var(--gray-900); line-height:1.5` |

**Note on `.hs-type-label` vs `.hs-section-heading`:** Both use `0.75rem / 600`. Visual separation comes from different `letter-spacing` values (`.08em` vs `.05em`) and context (type label appears above a section heading, not beside one).

**Classes being removed/replaced from `.quick-stats`:** The old `.stat-group`, `.stat-group-items`, `.stat-group-label`, `.stat-group-divider` usages inside `.quick-stats` are superseded by the card-based layout. The old classes remain in CSS (do not delete — other views may reference them) but are no longer used in `home.js`.
