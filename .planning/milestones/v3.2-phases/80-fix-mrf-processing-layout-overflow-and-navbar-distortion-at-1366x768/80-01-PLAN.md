<objective>
Fix the MRF Processing two-panel layout so the right-hand MRF Details card (header buttons + items table) fits within the viewport at 1366x768. Today the items-table inside the right panel has `min-width: 1000px` (intentional, for its own horizontal scroll), but because CSS Grid items default to `min-width: auto`, the `1fr` right column expands to its intrinsic content width, pushing the card-header buttons (Save / Generate PR / Reject MRF) past the viewport edge and forcing page-level horizontal + vertical scroll.

Purpose: Restore the intended split-panel layout at typical laptop viewports without hiding any controls. The items-table's own horizontal scroll already exists — we just need to keep the card itself bounded by its grid track.

Output: Updated `.dashboard-grid` rules in `styles/views.css` so grid children honour their track width and the items-table-wrapper properly contains its overflow.
</objective>

Root cause: CSS Grid items have `min-width: auto` by default, which means a track resolves to `max(track-size, content-min-width)`. The items-table's `min-width: 1000px` propagates up through `.items-table-wrapper > .card > .dashboard-grid` and forces the `1fr` track to be at least 1000px + card padding (48px) = ~1048px. At 1366px viewport with 320px left column + 16px gap + 64px section padding, only ~966px is available — the right column overflows by ~80px, clipping the card-header right edge and forcing page horizontal scroll.

Fix pattern: `.dashboard-grid > *` { `min-width: 0` } — this is the canonical CSS Grid overflow fix. It tells grid items to honour their track width, allowing the inner `overflow-x: auto` on `.items-table-wrapper` to do its job.
