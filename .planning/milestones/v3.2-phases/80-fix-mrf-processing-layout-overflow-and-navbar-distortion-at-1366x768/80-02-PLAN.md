Fix the top navigation bar so it stays on a single line at 1366x768 (laptop monitor) without wrapping to a second row or stacking the logo above the title. The current breakpoint goes straight from full desktop nav to mobile hamburger at 768px, leaving the 769–1366px range under-served. At 1366px the nav items barely don't fit and wrap.

Approach: keep desktop nav at 1366px (user expects it on a laptop) but tighten its spacing so it fits comfortably, AND lock both `.nav-links` and `.nav-brand` to `flex-wrap: nowrap` with `white-space: nowrap` on individual links so a one-pixel shortfall causes horizontal compression rather than line-wrapping.

Purpose: Restore expected desktop chrome on standard laptop viewports. Mobile hamburger behavior at <=768px is preserved untouched.

Output: Updated nav rules in `styles/components.css` — additive declarations + a new `@media (max-width: 1400px)` block.

Width budget at 1366px: `.top-nav-content` has `max-width: 1600px` and `padding: 0 2rem` (32px each side). Available row width = 1366 − 64 = 1302px.
- `.nav-brand`: img (40px) + gap 1rem (16px) + "CLMC Operations" (~155px @ 1.25rem 600 weight) ≈ 211px
- `.nav-links` (8 items + logout):
  - 7 plain links @ 0.5rem 1rem padding (16px L+R) + ~70px text avg ≈ 102px each → 714px
  - Admin dropdown trigger ≈ 80px
  - 7 gaps × 0.5rem (8px) ≈ 56px
  - Logout button: margin-left 1rem (16px) + padding 0.5rem 1rem + "Log Out" (~70px) ≈ 102px + 16px = 118px
  - Total .nav-links ≈ 714 + 80 + 56 + 118 = 968px
- Total demand: 211 + 968 = 1179px. Plus space-between gap. Tight fit at 1302px — any rendering rounding or font fallback pushes it over and triggers wrap.

Fix: at <= 1400px viewports, compress nav-link padding from `1rem` → `0.625rem` and gap from `0.5rem` → `0.25rem`, lock `flex-wrap: nowrap` on `.nav-links` and `.nav-brand`, and `white-space: nowrap` on `.nav-link` and `.nav-logout-btn`. This reduces total demand to roughly 1080px — comfortable at 1302px available width — without altering desktop chrome at >= 1401px or mobile (<= 768px) behavior.
