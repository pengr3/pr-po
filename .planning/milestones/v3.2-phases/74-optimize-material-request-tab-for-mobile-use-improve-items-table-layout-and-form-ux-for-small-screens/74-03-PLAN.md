<objective>
At viewport ≤768px on the Material Request > My Requests tab, replace the 8-column scrollable table with a vertical stack of compact MRF summary cards. Each card shows only the 3 most scannable fields (MRF ID, MRF Status, Date Needed) plus a 3-dot (⋮) action button that opens Edit/Cancel actions via a dropdown. Desktop behavior (table + right-click context menu) is unchanged — cards are hidden via CSS at ≥769px. Card HTML generation lives in `mrf-records.js` (per D-10); the action callback hooks into the existing `window._myRequestsEditMRF` / `window._myRequestsCancelMRF` handlers already set up in `mrf-form.js`.

Purpose: The 8-column My Requests table is unreadable on phones — users must horizontal-scroll to see status and date-needed, and there's no touch equivalent for right-click context menu. Cards + 3-dot menu make the mobile experience native.

Output: New `.mrf-req-card*` CSS classes, modified `createMRFRecordsController` that emits both table and card list via SHARED per-MRF display data (computed once per MRF via `mapMRFToDisplayData`), `onMobileAction` callback option, and a 3-dot dropdown wired to the existing Edit/Cancel handlers in `mrf-form.js`, with a scroll listener that auto-closes the dropdown so it can never float detached from its card.

### Review feedback addressed (from 74-REVIEWS.md)
- **[MEDIUM] Card data recomputed separately** — Introduce a single `mapMRFToDisplayData(mrf, subCache)` helper inside `createMRFRecordsController` that computes `{displayId, dateNeeded, mrfStatusHtml, type}` for one MRF. Both the existing row loop and the new card builder call this helper — no duplicate formatting logic. If status formatting changes in the future, only `mapMRFToDisplayData` needs to be updated.
- **[LOW] Floating 3-dot menu on scroll** — When the dropdown opens, attach a one-shot window `scroll` listener (stored in `_mrfMobileMenuScrollHandler`) that closes the menu and removes itself. When the menu is closed (via outside-click, menu-item click, or scroll), the listener is detached.
</objective>
