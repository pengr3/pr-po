# Phase 65.2 Plan 01: Remove Processed RFPs from Table 1 by Default Summary

**One-liner:** Default exclusion of Fully Paid RFPs from Table 1 (RFP Processing) with explicit opt-in via status dropdown.

## What Was Built

Added a default filter in `renderRFPTable()` that hides Fully Paid RFPs unless the user has explicitly selected "Fully Paid" from the status dropdown. Finance users now land on the Payables tab seeing only actionable payment requests (Pending, Overdue, Partially Paid). All historical paid RFPs remain accessible via the filter dropdown and via Table 2 (PO Payment Summary), which is completely unchanged.

## Changes Made

### app/views/finance.js

**Change 1 — renderRFPTable() exclusion guard (line 447):**
```javascript
// Default: hide Fully Paid RFPs unless explicitly filtered to show them
if (rfpStatusFilter !== 'Fully Paid') {
    displayed = displayed.filter(r => deriveRFPStatus(r) !== 'Fully Paid');
}
```
Placed before the existing `rfpStatusFilter` and `rfpDeptFilter` blocks. The guard short-circuits only when the user has selected "Fully Paid" in the dropdown.

**Change 2 — rfpStatusFilter dropdown (line 1479):**
Label changed from `All Statuses` to `Outstanding (default)` so users immediately understand the default view is scoped to actionable RFPs.

**Change 3 — Empty state message (line 477):**
Updated to: "No outstanding payment requests. Use the status filter to view Fully Paid RFPs, or check the PO Payment Summary below."

## Behavior Matrix

| User selection | Exclusion guard active | Result |
|---|---|---|
| (default / Outstanding) | Yes | Pending + Overdue + Partially Paid only |
| Pending | Yes (redundant, no change) | Pending only |
| Overdue | Yes (redundant, no change) | Overdue only |
| Partially Paid | Yes (redundant, no change) | Partially Paid only |
| Fully Paid | No (guard skipped) | Fully Paid only |
