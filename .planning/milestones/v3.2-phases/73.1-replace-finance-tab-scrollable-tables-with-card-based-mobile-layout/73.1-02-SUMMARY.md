# Phase 73.1 Plan 02: PO Card Layout + buildProofIndicator Extraction Summary

## Accomplishments

- Extracted proof indicator logic into reusable `buildProofIndicator(po)` helper — 3-state (proof-filled / proof-remarks / proof-empty), eliminates quote-heavy template duplication between table and card
- Added `buildPOCard(po)` helper producing full card HTML with PO ID, status badge, proof indicator, supplier, project, amount, PR ID, date issued, and full-width View PO button
- Modified `renderPOs()` to emit both `.table-scroll-container` (desktop) and `.fc-card-list` (mobile) in a single `container.innerHTML` assignment — CSS from Plan 01 handles show/hide
- Fixed the status mismatch that existed in the original table row: both `statusLabel` and `statusClass` now derive from the same `po.procurement_status || 'Pending Procurement'` fallback

## Decisions Made

- `statusValue` single-source fallback (`po.procurement_status || 'Pending Procurement'`) used for both badge text and CSS class — addresses REVIEWS Suggestion that the original table row had `|| 'Pending'` for text but `|| 'Pending Procurement'` for the class
- `buildPOCard` uses bare `promptPODocument` (no `window.` prefix) to match existing table row convention (both patterns work since the function is registered on window in attachWindowFunctions)
- `fc-card-list` inserted between `</div>` of table-scroll-container and the `filteredPOs.length > 20` footer note — preserves footer note below both table and cards at correct position
