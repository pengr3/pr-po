Add this sort:

```javascript
    // Sort by PO Ref then tranche percentage so tranches under the same PO group together
    displayed = [...displayed].sort((a, b) => {
        const poCompare = (a.po_id || '').localeCompare(b.po_id || '');
        if (poCompare !== 0) return poCompare;
        return (a.tranche_percentage || 0) - (b.tranche_percentage || 0);
    });
```

Key details:
- Spread into a new array (`[...displayed]`) to avoid mutating rfpsData when no filters are active (since `displayed` may still reference the original array).
- Primary sort: `po_id` ascending via localeCompare (groups all tranches for the same PO together).
- Secondary sort: `tranche_percentage` ascending (e.g. 30% DP before 70% Balance).
- This is a stable sort in modern JS engines so equal-key rows retain their relative order.
