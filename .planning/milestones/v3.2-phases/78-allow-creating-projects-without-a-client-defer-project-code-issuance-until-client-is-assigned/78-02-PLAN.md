<objective>
Wire the MRF form layer to support clientless projects: (1) in `app/views/mrf-form.js` (standalone Material Request route), include clientless projects in the project dropdown with a "(No code yet)" label suffix; (2) in `app/views/procurement.js` (Procurement > Create MRF), include clientless projects in the unified project/service dropdown with the same "(No code yet)" suffix; (3) in BOTH save paths, denormalize the Firestore project doc ID as `project_id` on every new MRF document so child records can be backfilled later by Plan 03's issuance flow.

Purpose: Clientless projects must be selectable as MRF targets (per D-01: TR-for-site-visit is the primary use case). Without `project_id` denormalization, the Plan-03 issuance backfill cannot find the MRF children to update.

Output: Requestors see clientless projects in MRF dropdowns with a "(No code yet)" hint and can submit MRFs against them. Every new MRF (clientless or coded) carries `project_id` so issuance backfill works.
</objective>
