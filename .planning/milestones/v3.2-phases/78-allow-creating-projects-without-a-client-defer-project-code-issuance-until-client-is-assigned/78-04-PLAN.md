The backfill in Plan 03 Task 3 runs `getDocs(query(collection(db, COL), where('project_id', '==', projectId)))` for COL in [mrfs, prs, pos, transport_requests, rfps] — these are ALL single-field equality queries.

    STEP B — Read the file and grep its `fieldOverrides` array for any entry referencing `project_id`:
    ```bash
    node -e "const o=JSON.parse(require('fs').readFileSync('firestore.indexes.json','utf8')); const overrides=o.fieldOverrides||[]; const projIdOverrides=overrides.filter(x => x.fieldPath === 'project_id' && ['mrfs','prs','pos','transport_requests','rfps'].includes(x.collectionGroup)); console.log(JSON.stringify({total_overrides: overrides.length, project_id_overrides_on_target_collections: projIdOverrides}, null, 2));"
    ```

    - **Case 1: `project_id_overrides_on_target_collections` is an empty array.** Auto-indexing is in effect for project_id on all 5 target collections. NO file change needed. Proceed to STEP D.

    - **Case 2: `project_id_overrides_on_target_collections` contains one or more entries with `indexes: []` or `indexes: [{queryScope: 'COLLECTION_GROUP'}]` only (i.e., the entry exempts COLLECTION-scope auto-indexing).** This WOULD break runCodeIssuance. Fix by either:
      (a) DELETING the offending fieldOverride entries from `firestore.indexes.json`, OR
      (b) ADDING explicit single-field indexes back (`indexes: [{order: 'ASCENDING', queryScope: 'COLLECTION'}]`).

      Use option (a) — simpler, restores default behavior. After deletion, re-run STEP B; the result should be an empty array.

    - **Case 3: Existing entries already provide explicit COLLECTION-scope indexes for project_id (e.g., `indexes: [{order: 'ASCENDING', queryScope: 'COLLECTION'}]`).** No change needed.
