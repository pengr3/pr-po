# Requirements: CLMC Procurement System

**Defined:** 2026-03-01
**Core Value:** Projects tab must work - it's the foundation where project name and code originate, and everything in the procurement system connects to it.

## v2.5 Requirements

Requirements for pre-launch security hardening and data migration preparation. Each maps to roadmap phases.

### Security Audit

- [x] **SEC-01**: Client-side code reviewed for XSS vulnerabilities (innerHTML, user input rendering)
- [x] **SEC-02**: Client-side code reviewed for injection risks (Firestore query manipulation, eval usage)
- [x] **SEC-03**: Client-side code reviewed for sensitive data exposure (API keys in source, PII in logs)
- [x] **SEC-04**: Firebase Security Rules audited for authorization gaps across all 12 collections
- [x] **SEC-05**: Auth flow edge cases reviewed (session handling, token expiry, role escalation)
- [x] **SEC-06**: CSP headers reviewed and hardened for production

### Database Safety

- [x] **DBS-01**: Firestore backup script exports all collections to local JSON files
- [x] **DBS-02**: Data integrity verification identifies orphaned references and schema inconsistencies
- [x] **DBS-03**: Backup restore procedure documented and verified

### Data Wipe

- [x] **WIP-01**: Standalone script clears all Firestore collections except users
- [x] **WIP-02**: Dry-run mode previews what will be deleted before execution
- [x] **WIP-03**: Confirmation safeguard prevents accidental wipe

### Data Migration

- [x] **MIG-01**: Import script reads projects data from CSV/JSON and writes to Firestore
- [x] **MIG-02**: Import script reads services data from CSV/JSON and writes to Firestore
- [x] **MIG-03**: Validation/preview mode shows data before writing to Firestore
- [x] **MIG-04**: Error reporting for invalid or malformed records

### Clickable Active Badge

- [x] **BADGE-01**: Active/Inactive badges in project and service list views are clickable to toggle status with confirmation
- [x] **BADGE-02**: View-only users see static (non-clickable) badges without cursor or hover effects
- [x] **BADGE-03**: Separate Activate/Deactivate button removed from Actions column in both list views

### Finance Expense List

- [x] **FINLIST-01**: Finance view #/finance/projects has sub-tabs for Projects, Services, and Recurring
- [x] **FINLIST-02**: Services sub-tab displays one-time services with budget, expense, and status columns matching the project list layout
- [x] **FINLIST-03**: Recurring sub-tab displays recurring services with the same column structure
- [x] **FINLIST-04**: Search bar on each sub-tab filters the list by name or code in real-time

## Future Requirements

### Activity Logging
- **ACT-01**: Structured activity entries on projects
- **ACT-02**: Activity types: Inspection, Meeting, Proposal Submitted, Site Visit, Other
- **ACT-03**: Visible to assigned personnel and permitted users

### Document Management
- **DOC-01**: Upload BOQ files to projects
- **DOC-02**: Upload contract documents
- **DOC-03**: View/download uploaded documents

### Payment Tracking
- **PAY-01**: PO payment status tracking (percentage paid)
- **PAY-02**: Payment terms on POs (Net 30, Net 60, custom)
- **PAY-03**: Payment milestones with trigger workflow

## Out of Scope

| Feature | Reason |
|---------|--------|
| User data migration | User handles user deletion manually; users collection preserved during wipe |
| MRF/PR/PO migration | Fresh start — no legacy procurement data being carried forward |
| Automated CI/CD security scanning | No build system — manual audit appropriate for static SPA |
| Penetration testing | Out of scope for code-level milestone; separate engagement |
| Firebase Admin SDK integration in app | Scripts use Admin SDK locally, not embedded in SPA |
