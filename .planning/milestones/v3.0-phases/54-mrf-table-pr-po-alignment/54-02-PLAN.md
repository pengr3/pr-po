<objective>
Fix the MRF Records table in procurement.js so each PR row shows its corresponding PO ID inline on the same visual line, with a null slot when no PO exists, and the Procurement Status dropdown on that same line.

Purpose: Procurement staff currently see PRs, POs, and Procurement Status in three separate columns — they cannot tell which status dropdown belongs to which PR/PO pair, especially when an MRF has multiple suppliers. The fix merges all three into a single "PRs / POs" column where each sub-row is one PR+PO+Status triplet.

Output: Modified `app/views/procurement.js` where `renderPRPORecords()` renders a 7-column table instead of 8, with a merged PRs/POs cell showing per-PR paired rows including the editable Procurement Status dropdown on the same row.
</objective>

                        <option value="Procured" ${currentStatus === 'Procured' ? 'selected' : ''}>Procured</option>
                        <option value="Delivered" ${currentStatus === 'Delivered' ? 'selected' : ''}>Delivered</option>
                    `;
                }

                const statusColors = {

                    ? ' <span style="background: #e0f2fe; color: #0369a1; padding: 2px 6px; border-radius: 4px; font-size: 0.65rem; font-weight: 600;">SUBCON</span>'

                    onclick="window.viewPODetails('${po.docId}')"

            ${prBadge}
            <span style="color: #94a3b8; font-size: 0.7rem;">→</span>

    </div>`;
