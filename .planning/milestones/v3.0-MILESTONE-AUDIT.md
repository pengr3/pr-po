## Tech Debt Summary (Non-blocking)

### Phase 54 — MRF Table PR/PO Alignment

1. **SUMMARY description inaccuracy**: Both SUMMARY files claim the table was "collapsed from 8 to 6 columns." The actual implementation retains 8 columns with PRs, POs, and Procurement Status as separate columns. Inline flex sub-rows achieve the pairing effect within cells. The thead and tbody are aligned at 8 columns — no rendering bug, but the SUMMARY description does not match the code. **Impact**: Documentation only. Requirements TABLE-01/02/03 are satisfied.

2. **Skeleton loading header**: `mrf-records.js` line 958 shows old 8-column skeleton during the ~100–200ms data fetch window. Replaced by the correct table before user interaction. **Impact**: Cosmetic/transient.

### Phase 55 — Finance Pending Approvals Fixes

3. **TR approval count stability**: `approvedTRsThisMonthCount` is loaded once in `init()` and not recalculated on subsequent PO snapshots. A TR approval during an active session will not update the scoreboard until the Finance view is reloaded. **Impact**: Minor UX staleness within a single session; by design per SUMMARY.

### All Phases — Documentation

4. **SUMMARY `requirements-completed` field absent**: All SUMMARY.md files lack the `requirements-completed` frontmatter field expected by the audit workflow's 3-source cross-reference. REQUIREMENTS.md traceability and VERIFICATION.md code evidence compensate. **Impact**: Workflow documentation gap only; all requirements confirmed satisfied via two remaining sources.

**Total tech debt items: 4 across 3 phases**
