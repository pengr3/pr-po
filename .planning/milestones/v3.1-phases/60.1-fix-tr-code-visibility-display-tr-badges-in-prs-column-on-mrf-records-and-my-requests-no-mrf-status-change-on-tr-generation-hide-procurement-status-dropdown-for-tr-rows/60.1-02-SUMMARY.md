# Phase 60.1 Plan 02: TR Status Decoupling and Rejected MRFs Section Summary

TR submission no longer silently mutates MRF status; rejected MRFs now appear in their own dedicated panel in MRF Processing, separate from the pending sections.

## What Was Built

### Task 1: Remove MRF status mutation from TR submission paths

Three targeted fixes in `app/views/procurement.js`:

1. **submitTransportRequest (~line 3613)** â€” Removed `status: 'TR Submitted'` from the `updateDoc(mrfRef, ...)` call. The MRF now stores `tr_id` + `items_json` without touching its status field. MRF stays in Pending Transportation Requests after TR generation.

2. **generatePRandTR (~line 4229)** â€” Changed `status: 'PR & TR Submitted'` to `status: 'PR Submitted'`. Per Phase 60 decision, the TR component of a combined PR+TR generation does not drive MRF status.

3. **loadMRFs statuses query (~line 734)** â€” Added `'TR Rejected'` to the `statuses` array so legacy MRF documents with `status: 'TR Rejected'` surface in the MRF Processing list (needed for the new Rejected MRFs panel to show them).

### Task 2: Dedicated Rejected MRFs section in renderMRFList

Refactored `renderMRFList(materialMRFs, transportMRFs)` to cleanly separate rejected from pending MRFs:

- **Split at top**: `isRejectedStatus()` helper filters each array into `pendingMaterialMRFs` + `rejectedMaterialMRFs` and `pendingTransportMRFs` + `rejectedTransportMRFs`. `allRejectedMRFs` combines both rejected arrays.
- **Pending sections cleaned**: Material Requests and Pending Transportation Requests sections now iterate `pendingMaterialMRFs` / `pendingTransportMRFs` only. The `isRejected` / `rejectionStyle` inline detection removed â€” pending items render without red borders.
- **New Rejected MRFs section**: Conditionally rendered (`if (allRejectedMRFs.length > 0)`) between the transport section and the existing Rejected Transport Requests section. Cards use red border + background, display a status badge (PR REJECTED / TR REJECTED / FINANCE REJECTED), rejection reason, and urgency-colored left border. `onclick="window.selectMRF()"` preserved so clicking still loads details.
- **Rejected Transport Requests section unchanged** â€” Finance TR rejections from `cachedRejectedTRs` still appear below.
