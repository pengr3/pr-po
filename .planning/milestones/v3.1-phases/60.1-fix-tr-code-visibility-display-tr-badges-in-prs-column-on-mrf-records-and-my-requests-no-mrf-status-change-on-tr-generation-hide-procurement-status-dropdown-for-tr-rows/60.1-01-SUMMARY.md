# Phase 60.1 Plan 01: Fix TR Code Visibility in MRF Records and My Requests Summary

**One-liner:** TR codes rendered as finance-status-colored badges in the PRs column for Transport rows in both My Requests (mrf-records.js) and MRF Records (procurement.js), with MRF ID column corrected to always show the true mrf_id.

## What Was Built

Transport MRF rows in both table views previously showed a dash in the PRs column (making TR tracking invisible) and incorrectly showed the tr_id in the MRF ID column. Both issues are now fixed:

1. **My Requests (mrf-records.js):** Transport rows now fetch `trDataArray` (array of `{ docId, tr_id, finance_status }`) from `transport_requests` collection. The array is cached in `_subDataCache`. A new `else if (type === 'Transport')` block renders each TR code as a `<span class="status-badge">` colored by `finance_status`. `displayId` now always uses `mrf.mrf_id`.

2. **MRF Records / Procurement tab (procurement.js `renderPRPORecords`):** Same pattern applied — `trDataArray` built during Transport fetch, stored in `_prpoSubDataCache`, rendered as status badges in `prHtml`. `displayId` corrected to always use `mrf.mrf_id`.
