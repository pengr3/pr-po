Make the Procurement and Finance views fluid at 1366×768 (standard laptop) by converting the fixed-width `.dashboard-grid` left panel from `350px` to a fluid `minmax()` column and adding a responsive breakpoint for laptop-sized screens.

<!-- Current CSS to change -->

From styles/views.css ~line 40:
```css
.dashboard-grid {
    display: grid;
    grid-template-columns: 350px 1fr;
    gap: 2rem;
    margin-top: 2rem;
}
```

The two-panel layout is: left panel (Pending MRFs list) = fixed 350px, right panel (MRF Details) = 1fr.

At 1366px viewport with 1600px max-width container and 2rem (32px) padding each side:
- Available content width = min(1366px, 1600px) - 64px = 1302px
- Left panel = 350px, gap = 32px, right panel = 1302 - 350 - 32 = 920px
- This actually fits — no overflow at page level
- But at very tight widths (e.g. 1280px browser window), 350px left + 32px gap leaves only 850px right

Root cause identified from CONTEXT.md: "Fixed-width panels/modals that don't shrink on smaller screens"
Strategy: "Fluid columns — tables and panels resize to fill available space, no hard-coded pixel widths"

Target fix for .dashboard-grid:
```css
.dashboard-grid {
    display: grid;
    grid-template-columns: minmax(280px, 320px) 1fr;
    gap: 1.5rem;
    margin-top: 2rem;
}
```
This reduces the left panel max from 350px to 320px and allows it to shrink to 280px, gaining ~60px.

Also add a responsive breakpoint for 1400px and below (covers 1366px laptops):
```css
@media (max-width: 1400px) {
    .dashboard-grid {
        gap: 1rem;
    }
}
```
Reducing the gap from 2rem to 1rem gains another 16px.

Additional check needed in views.css: Any other fixed-width elements used by Procurement/Finance panels?
