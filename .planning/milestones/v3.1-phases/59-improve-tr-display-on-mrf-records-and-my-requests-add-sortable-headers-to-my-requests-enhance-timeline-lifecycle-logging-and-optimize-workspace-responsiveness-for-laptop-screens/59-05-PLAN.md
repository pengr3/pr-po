<task type="auto">
  <name>Task 2: Add sub-data cache to eliminate Firestore re-fetch and loading flash on sort/filter/page-change</name>
  <files>app/views/mrf-records.js, app/views/procurement.js</files>
  <action>
The loading flash occurs because `render()` (mrf-records.js) and `renderPRPORecords()` (procurement.js) unconditionally show a loading placeholder and fire Firestore queries for PR/PO/TR sub-data on every call — including calls triggered by sort, filter, and page-change where the sub-data has not changed.

Fix: add a sub-data Map cache in each location. Cache key = `mrf.id` (the Firestore document ID). Cache value = the assembled per-row sub-data object. Skip the loading text and Firestore fetches when the cache is warm. Invalidate on a genuine fresh data load.


4. In `render()`, inside the `Promise.all(pageItems.map(async (mrf) => { ... }))` callback, wrap the existing Firestore fetch blocks with a cache check. Before the `if (type === 'Material') {` block that fires Firestore queries:
```javascript
// Check cache first — skip Firestore if sub-data already loaded
if (_subDataCache.has(mrf.id)) {
    const cached = _subDataCache.get(mrf.id);
    prDataArray = cached.prDataArray;
    poDataArray = cached.poDataArray;
    trFinanceStatus = cached.trFinanceStatus;
} else {
    // ... existing Firestore fetch blocks (Material PRs, Material POs, Transport TR) ...
    // After all fetches complete, store in cache:
```

The variables `prDataArray`, `poDataArray`, and `trFinanceStatus` must be declared (with `let`) before the cache check block so both branches can assign to them.

Only add the `container.innerHTML = '<div ...Loading...</div>'` loading text when the cache is empty (i.e., `_subDataCache.size === 0`). Add this guard just before the `Promise.all` call:
```javascript
if (_subDataCache.size === 0) {

}
```

The cached object must capture: `prDataArray`, `poDataArray`, `totalCost`, `prCount`, `prApprovedCount`, `trCost`, `trFinanceStatus` (all variables used for the row render after the fetches). Wrap all existing Firestore fetch blocks (`if (type === 'Material')` for PRs and POs, and the `if (type === 'Transport')` block for TR cost) inside an else branch of the cache check:

```javascript
let prDataArray = [];
let poDataArray = [];
let totalCost = 0;
let prCount = 0;
let prApprovedCount = 0;
let trCost = 0;
let trFinanceStatus = null;

if (_prpoSubDataCache.has(mrf.id)) {
```
