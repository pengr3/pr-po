# Changelog: v3.1 — PR/TR Routing Fix & Procurement Workflow Improvements

**Released:** 2026-03-10
**Phases:** 57-62.2 (11 phases, 22 plans)

---

## New Features

### Delivery By Supplier Category
- New item category **"DELIVERY BY SUPPLIER"** available in both MRF creation forms
- Items with this category route through the **PR/PO path** (not TR), so supplier delivery charges appear on the Purchase Order
- Supplier selection is required for these items (same validation as all other PR items)

### MRF Soft-Reject (replaces Delete)
- "Delete MRF" button replaced with **"Reject MRF"** in Procurement
- Rejected MRFs are preserved in Firestore with rejection reason, who rejected, and when
- Rejected MRFs appear in a dedicated **Rejected MRFs** panel in MRF Processing with red styling
- Full audit trail maintained — no data loss

### TR Details Modal
- TR ID badges in MRF Records and My Requests tables are now **clickable**
- Clicking opens a **TR Details modal** showing: TR ID, MRF reference, supplier, finance status, total amount, line items, and rejection reason (if any)

### Add/Delete Line Items on Rejected TRs
- Procurement can now **add new line items** to Finance-rejected TRs before resubmitting
- Per-row **delete buttons** allow removing unwanted items
- New items participate in subtotal/grand total calculations and are saved to Firestore

### MRF Rejection Timeline Event
- The Procurement Timeline now shows an **"MRF Rejected"** event (red) when an MRF has been rejected
- Displays rejection reason, who rejected it, and when
- Visible in both Procurement Timeline and My Requests Timeline

---

## Improvements

### TR Rejection Independence
- TR rejection **no longer cascades to MRF status** — the MRF stays Approved when a TR is rejected
- Rejected TRs appear in a dedicated panel in Procurement where users can:
  - View and edit item details (quantity, unit cost, supplier)
  - Save changes to Firestore
  - Resubmit to Finance for re-review
  - Delete the TR entirely
- Finance sees a **"Previously Rejected"** notice when reviewing resubmitted TRs

### TR Display in Tables
- TR codes now show as **colored badges in the PRs column** on both MRF Records and My Requests
- TR finance status badges (Pending/Approved/Rejected) visible in MRF Status column
- MRF ID column always shows the MRF ID (not TR ID)

### Alphabetical Dropdowns
- Project and service dropdowns in MRF creation now sort **A-Z by name** (previously sorted by recency)
- Applied to both standalone MRF form and Procurement Create MRF

### Project Code Format
- Project and service codes now use **dashes** instead of underscores: `CLMC-CLIENT-2026001` (was `CLMC_CLIENT_2026001`)

### Sortable My Requests Headers
- My Requests table headers are now **clickable for sorting** (MRF ID, Date Needed, Status)
- Default sort: Date Needed ascending

### Responsive Workspace
- Procurement and Finance views optimized for **1366px laptop screens** without requiring zoom
- My Requests table widened from 1200px to 1600px to match MRF Records tab proportions

### MRF Records Real-Time Rendering
- Newly created MRFs now appear **instantly** in both Procurement Records and My Requests tables
- No manual page refresh required
- Filter dropdown now includes Pending and Approved status options

### Timeline Lifecycle Events
- Procurement Timeline now shows the **full rejection/resubmission lifecycle** for PRs and TRs:
  - Submitted → Rejected (with reason and actor) → Resubmitted → Approved

## Bug Fixes

- **TR rejection visibility**: Finance-rejected TRs now correctly reappear in Procurement for resubmission
- **PR/TR Rejected records**: MRF Records tab now shows MRFs with PR Rejected and TR Rejected statuses
- **CSP header violations**: Added `https://www.gstatic.com` to CSP connect-src — Firebase source maps no longer blocked
- **Finance Project List**: Fixed error where incorrect field name caused empty project list; now shows all projects (active sorted first, then inactive)
- **MRF deletion permission**: Procurement role can now delete MRFs without permission errors
- **MRF submission permission**: Services users can now submit MRFs without permission errors
- **Soft-rejected MRF visibility**: Rejected MRFs now correctly appear in the Rejected MRFs panel (was invisible after rejection)
- **Window function cleanup**: viewTRDetails properly cleaned up when navigating away from My Requests
